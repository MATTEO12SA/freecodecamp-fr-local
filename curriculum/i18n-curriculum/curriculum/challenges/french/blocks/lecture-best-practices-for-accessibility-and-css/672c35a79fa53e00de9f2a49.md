# --interactive--

Masquer du contenu sur une page web est une pratique courante en développement web, mais il est essentiel de le faire d'une manière qui ne compromet pas l'accessibilité. Les différentes techniques de masquage peuvent avoir des impacts variés sur la façon dont les technologies d'assistance interprètent et présentent le contenu aux utilisateurs. Explorons quelques bonnes pratiques pour masquer du contenu tout en conservant l'accessibilité. Une méthode courante pour masquer du contenu consiste à utiliser `display: none`. Voici un exemple :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

Même si cela masque efficacement le contenu visuellement, cela le retire aussi de l'arbre d'accessibilité. L'arbre d'accessibilité est une structure utilisée par les technologies d'assistance, comme les lecteurs d'écran, pour interpréter et interagir avec le contenu d'une page web. Il représente le contenu et son sens sémantique d'une manière que les technologies d'assistance peuvent comprendre et présenter à l'utilisateur.

Utiliser `display: none` signifie que les lecteurs d'écran et les autres technologies d'assistance ne pourront pas accéder à ce contenu, car il n'est pas inclus dans l'arbre d'accessibilité. Il est donc important d'utiliser cette méthode uniquement lorsque tu veux supprimer complètement le contenu à la fois de la présentation visuelle et de l'accessibilité.

Une autre approche pour masquer du contenu consiste à utiliser `visibility: hidden` :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` masque le contenu visuellement, mais le garde dans le flux du document, ce qui signifie qu'il occupe toujours de l'espace sur la page. Comme `display: none`, `visibility: hidden` retire aussi le contenu de l'arbre d'accessibilité. Cela signifie que les technologies d'assistance, comme les lecteurs d'écran, ne pourront pas accéder au contenu masqué. Utilise `visibility: hidden` uniquement lorsque tu veux masquer le contenu pour tout le monde, y compris les personnes qui utilisent une technologie d'assistance.

Pour du contenu qui doit être masqué visuellement mais rester accessible aux lecteurs d'écran, tu peux utiliser une technique souvent appelée « visuellement masqué » ou « réservé aux lecteurs d'écran ». Voici un exemple utilisant la classe CSS `.sr-only`, une technique courante pour masquer visuellement du contenu tout en le gardant accessible aux lecteurs d'écran :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

Dans cet exemple, nous utilisons des propriétés comme `position`, `clip` et `white-space`, que tu découvriras dans de futures leçons. Pour l'instant, sache simplement que cette règle CSS masque efficacement le contenu visuellement tout en le gardant accessible aux lecteurs d'écran. Elle est utile pour fournir un contexte supplémentaire aux utilisateurs de lecteurs d'écran sans affecter la mise en page visuelle.

Pour basculer la visibilité du contenu, pense à utiliser l'attribut `hidden` :

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

L'attribut `hidden` est pris en charge par la plupart des navigateurs modernes et masque le contenu à la fois visuellement et dans l'arbre d'accessibilité. Il peut être facilement activé ou désactivé avec JavaScript. Enfin, fais attention quand tu masques du contenu important. Si une information est essentielle pour comprendre ou utiliser le site web, elle doit être visible et accessible à tous les utilisateurs. Masque du contenu uniquement lorsque cela améliore réellement l'expérience utilisateur. En suivant ces bonnes pratiques, tu peux t'assurer que ton contenu reste accessible à tous les utilisateurs, quelle que soit leur façon d'interagir avec ton site web.
