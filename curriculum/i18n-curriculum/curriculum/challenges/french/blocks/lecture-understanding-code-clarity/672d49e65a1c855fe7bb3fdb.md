# --description--

En programmation, les commentaires servent à apporter un contexte supplémentaire au code ou à laisser des notes pour toi-même et pour les autres.

Les commentaires sont des lignes ou des blocs de texte qui sont ignorés par le moteur JavaScript lorsque ton code est exécuté. Ils sont là uniquement pour les personnes qui lisent le code, que ce soit toi ou quelqu'un d'autre.

JavaScript propose deux façons d'ajouter des commentaires à ton code : les commentaires sur une seule ligne et les commentaires sur plusieurs lignes.

Les commentaires sur une seule ligne se créent à l'aide de deux barres obliques (`//`). Voici un exemple :

```js
// I am a single line comment in JavaScript
```

Ce type de commentaire est parfait pour de brèves explications ou clarifications.

Voici un exemple concret tiré des fichiers du projet de curriculum de freeCodeCamp :

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Ne t'inquiète pas de comprendre ce que fait réellement le code, car il est plus avancé que ce que tu as appris jusqu'à présent. Concentre-toi plutôt sur le commentaire laissé par le développeur. Ce commentaire apporte un contexte important sur la raison d'être de ce code.

Les commentaires de ce genre sont importants pour les personnes qui travaillent en équipe, et ce pour deux raisons :

1. Les autres développeurs qui travaillent sur le projet comprendront le but de ce code.

1. Cela aide à éviter les modifications ou suppressions inutiles sans consulter l'équipe, ce qui pourrait entraîner des bugs ou des problèmes.

Un autre type de commentaire est le commentaire sur plusieurs lignes. Voici la syntaxe de base :

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

Les commentaires sur plusieurs lignes sont utiles quand tu as besoin d'écrire des descriptions, des explications ou des notes plus longues dans ton code.

Jetons un nouveau coup d'œil aux fichiers du projet de curriculum de freeCodeCamp pour voir comment les commentaires sur plusieurs lignes peuvent être utilisés dans la pratique.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Comme précédemment, ignore tout le code JavaScript car il utilise des concepts qui n'ont pas encore été enseignés. Concentre-toi plutôt sur le commentaire laissé par le développeur.

Un développeur de l'équipe, ou même un nouveau contributeur qui travaille sur le projet, peut comprendre pourquoi ce morceau de code est là et disposer de tout le contexte avant de travailler sur cette partie du projet.

Même si les commentaires sont utiles en programmation, il est important d'éviter d'en abuser. Tu n'as pas besoin de commenter chaque ligne de code, surtout si le code est simple et explicite.

Voici un exemple de commentaires utilisés pour expliquer l'évidence :

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

Dans cette situation, il n'est pas nécessaire d'ajouter de commentaires ici car le code se comprend de lui-même. Le but est d'améliorer la lisibilité, pas d'encombrer le code avec des explications inutiles.

Si tu veux ajouter des commentaires à tes projets personnels pendant que tu apprends à coder, c'est très bien. Mais dès que tu commences à travailler sur des projets concrets avec d'autres développeurs, il est important de ne pas utiliser de commentaires pour du code qui se comprend de lui-même.

Il est également important de ne pas utiliser les commentaires pour masquer du code confus, trop compliqué ou mal écrit. Dans ces situations, le mieux est de refactoriser, c'est-à-dire de modifier, ton code pour que les autres développeurs comprennent mieux ce qui se passe.

Les commentaires sont des outils puissants pour documenter ton code et le rendre plus facile à comprendre. Tu devrais utiliser les commentaires pour apporter du contexte ou laisser des notes pour toi-même et pour les autres.
