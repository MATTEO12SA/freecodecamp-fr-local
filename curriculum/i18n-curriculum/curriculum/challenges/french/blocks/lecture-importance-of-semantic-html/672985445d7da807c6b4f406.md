# --interactive--

L'aspect le plus important de la création d'une hiérarchie structurelle est l'utilisation correcte des éléments de titre. Les éléments de titre sont numérotés `h1`, `h2`, `h3`, et ainsi de suite. Ces numéros représentent le niveau du titre pour cet élément.

Tout comme dans un document texte, tu ne veux pas utiliser les niveaux de titres dans le mauvais ordre. Ton élément `h1` est ton titre de premier niveau. Tu en auras rarement plus d'un sur une page, et il devrait généralement venir avant tout ton contenu.

Ton élément `h2` est ton sous-titre. Il doit toujours venir après ton `h1` et peut venir après un texte d'introduction. Contrairement à un élément `h1`, tu peux avoir plusieurs éléments `h2` sur ta page. Tu le feras souvent pour délimiter différentes sections de contenu. 

En suivant le même schéma, ton élément `h3` doit toujours venir après un élément `h2`. C'est-à-dire que tu ne devrais jamais sauter directement de `h1` à `h3`. Tu peux, cependant, avoir plusieurs éléments de titre au même niveau. Par exemple, ce code est correct :

```html
<section>
  <h1>freeCodeCamp</h1>
  <h2>Learn Front-End Development</h2>
  <h2>Learn Back-End Development</h2>
</section>
```

Mais ce code ne serait pas correct, parce que `h3` vient avant `h2`. 

Déplace l'élément `h2` au-dessus du `h3` pour qu'il soit sémantiquement correct. Pour interagir avec le code, tu dois activer l'éditeur interactif.

:::interactive_editor

```html
<section>
  <h1>freeCodeCamp</h1>
  <h3>Introduction to HTML</h3>
  <h2>Learn Front-End Development</h2>
</section>
```

:::

Il peut être tentant d'utiliser un élément de titre particulier à cause de son style, par exemple `h1` pour du texte de grande taille :

:::interactive_editor

```html
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>
```

:::

Au lieu de cela, tu devrais choisir l'élément approprié pour la structure de ton document et utiliser du CSS pour obtenir le style que tu veux.

Utiliser la bonne hiérarchie est important pour l'accessibilité. Les technologies d'assistance, comme les lecteurs d'écran, s'appuient sur la structure d'une page web pour déterminer comment l'analyser et l'annoncer à l'utilisateur. Utiliser un élément `h3` après un `h1` pourrait amener un utilisateur de lecteur d'écran à croire qu'il a accidentellement sauté du contenu important, à cause de l'absence d'un élément `h2`.

Une structure correcte est aussi importante pour le SEO. Les moteurs de recherche utilisent l'automatisation pour analyser le contenu de ta page web et déterminer quand et où elle doit apparaître dans les résultats. Si ta structure est mal formée, les moteurs de recherche peuvent ne pas être capables de bien te classer dans les résultats de recherche pertinents.

Enfin, selon le niveau d'incorrection de ta structure, ton HTML peut même ne pas être techniquement valide. Quand cela arrive, le navigateur web doit pratiquement deviner ce que tu voulais faire. Et ce qu'il devine peut ne pas être du tout ce que tu veux. 

Comme tu l'as appris aujourd'hui, il y a de nombreuses raisons d'utiliser la bonne hiérarchie structurelle pour ta page. Garde cela en tête quand tu construis de nouveaux projets.
