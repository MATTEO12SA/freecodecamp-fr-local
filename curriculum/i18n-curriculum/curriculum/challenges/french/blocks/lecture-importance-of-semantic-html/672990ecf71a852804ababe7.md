# --interactive--

Le HTML présentationnel se concentre sur l'apparence et le style du contenu. Aux débuts du HTML, les développeurs utilisaient des éléments comme `center`, `big` et `font`. Mais en développement web moderne, tu ne devrais pas utiliser ce type d'éléments, à cause de leurs limites et de leur impact négatif sur l'accessibilité et la maintenabilité.

Beaucoup d'éléments HTML présentationnels sont dépréciés, ce qui signifie qu'ils sont dépassés et ne sont plus recommandés. Il existe de meilleures façons d'obtenir les mêmes résultats. Cela dit, il est utile de savoir qu'ils existent, donc on va regarder quelques exemples.

L'élément `font` est un élément déprécié utilisé pour définir la taille et la couleur du texte. Voici un exemple d'élément `font`.

Active l'éditeur interactif et change la taille de 5 à 7 pour voir la police s'agrandir.

:::interactive_editor

```html
<font size="5" color="blue">This text is blue and large.</font>
```

:::

Cet exemple définit la couleur du texte à `blue` et la taille à une valeur de `5`. La plage de l'attribut `size` va de `1` à `7`, `1` étant la plus petite et `7` la plus grande. La valeur par défaut est `3`, si tu ne définis pas la valeur explicitement.

Bien que cet élément fonctionne encore, tu ne devrais pas l'utiliser parce que la taille et la couleur de police devraient toujours être définies en CSS, pas en HTML. 

L'élément `center` est un autre élément déprécié, utilisé pour centrer horizontalement le contenu dans son conteneur. Voici un exemple de l'élément `center` qui contient du texte et un élément paragraphe.

Active l'éditeur interactif et ajoute des balises `center` autour de `<p>Another example text.</p>` pour le voir centré sur la page.

:::interactive_editor

```html
<center>
  This text is centered.
  <p>HTML is awesome.</p>
</center>

<p>Another example text.</p>
```

:::

Ensuite, on a l'élément `big`. C'est un autre élément HTML présentationnel déprécié qui rend le texte qu'il contient d'un niveau plus grand que le texte qui l'entoure. Ici, on a un exemple qui définit un paragraphe en deux parties.

Active l'éditeur interactif et ajoute des balises `big` autour de `Some other text` et regarde les changements dans la fenêtre d'aperçu. 

:::interactive_editor

```html
<p>
  This text has a normal font size.
  <big>This text is larger.</big>
  Some other text.
</p>
```

:::

Souviens-toi que la taille de police devrait toujours être définie avec du CSS, donc tu ne devrais pas utiliser cet élément en HTML moderne.

C'étaient des exemples d'éléments HTML présentationnels. Mais ils sont tous dépréciés et ne sont plus recommandés. Alors qu'est-ce que tu devrais utiliser à la place ? Voyons ça.

Le HTML sémantique est maintenant la pratique recommandée. Il décrit le contenu des éléments, donc il est beaucoup plus facile à lire, à comprendre et à maintenir.

Les moteurs de recherche peuvent facilement comprendre ton site web quand tu utilises du HTML sémantique. C'est aussi utile pour l'accessibilité, parce que les lecteurs d'écran ont besoin d'informations sémantiques pour décrire ce qui est sur la page web.

Des exemples d'éléments HTML sémantiques incluent :

- L'élément `header` pour définir l'en-tête du document, ou d'une section.
- L'élément de section de navigation, `nav`, pour les sections contenant des liens de navigation.
- L'élément `section` pour regrouper des informations liées.
- L'élément `figure` pour les illustrations et les diagrammes.

Voici un exemple d'élément `header` qui contient un élément de section de navigation.

Active l'éditeur interactif et ajoute un `<a href="#">Products</a>` à l'intérieur du `nav` et regarde les changements dans la fenêtre d'aperçu.

:::interactive_editor

```html
<header>
  <nav>
    <a href="#">Home</a>
    <a href="#">About</a>
    <!--Add the products link here-->

    <a href="#">Contact</a>
  </nav>
</header>
```

:::

Les éléments sémantiques montrent clairement leur but au sein de la structure HTML. Il existe beaucoup d'éléments HTML sémantiques différents. Tu trouveras forcément celui qui correspond aux besoins de ton projet.

Beau travail. Maintenant tu connais la différence entre le HTML présentationnel et le HTML sémantique : le HTML sémantique décrit le contenu, alors que le HTML présentationnel se concentre sur l'apparence.
