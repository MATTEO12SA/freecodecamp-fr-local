# --interactive--

L'apparence et le comportement des éléments d'entrée comptent quand on construit des formulaires web. Un formulaire avec des entrées qui répondent aux actions de l'utilisateur va loin dans l'amélioration de l'expérience utilisateur, et il ne devrait pas causer de confusion ou de frustration.

Le CSS fournit plusieurs pseudo-classes d'entrée qui peuvent rendre tes formulaires plus intuitifs et conviviaux. Regardons ces pseudo-classes en détail.

La pseudo-classe `:focus` te permet de cibler un élément d'entrée quand un utilisateur le sélectionne ou clique dessus, attirant l'attention sur le champ d'entrée actif. Ça aide les utilisateurs à identifier facilement où ils tapent actuellement :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

Comme le nom l'implique, la pseudo-classe `:hover` est déclenchée quand l'utilisateur place le curseur sur un élément. Elle fournit un retour visuel avant que l'utilisateur n'interagisse avec l'entrée, l'alertant effectivement que l'entrée est prête pour l'action.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

Dans l'exemple ci-dessus, l'arrière-plan de l'élément d'entrée change en couleur orange quand l'utilisateur le survole, lui faisant savoir que le champ est prêt pour l'interaction.

La pseudo-classe `:checked` te permet de styliser un bouton radio ou une case à cocher différemment quand un utilisateur le sélectionne. De cette façon, l'utilisateur peut facilement voir quelle option il a choisie.

Voici un exemple avec une case à cocher pour accepter les conditions sur un site web.

**NOTE** : Une partie du CSS dans cet exemple utilise des propriétés qui n'ont pas encore été couvertes. C'est juste pour te donner une idée de comment créer une case à cocher personnalisée. Tu apprendras comment tout cela fonctionne dans les futures leçons et ateliers.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` cible les éléments d'entrée qui ont l'attribut `required`. Elle signale à l'utilisateur qu'il doit remplir le champ pour soumettre le formulaire.

Voici un exemple avec un formulaire qui a quelques champs d'entrée requis :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

Quand on valide des formulaires, tu peux utiliser la pseudo-classe `:valid` pour styliser les champs d'entrée qui respectent les critères de validation, et `:invalid` pour styliser les champs d'entrée qui ne respectent pas les critères. Généralement, tu utiliseras du vert pour une entrée valide et du rouge pour une entrée invalide.

Voici un exemple utilisant la pseudo-classe `:valid` :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

Voici un exemple d'utilisation de la pseudo-classe `:invalid` :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

La pseudo-classe `:disabled` te permet de sélectionner et styliser les éléments d'entrée qui sont désactivés. Ces éléments ont l'attribut `disabled`, qui empêche les utilisateurs d'interagir avec eux. Quand une entrée est désactivée, elle ne peut pas être cliquée, focalisée ou éditée.

Voici un exemple d'un label et d'un élément d'entrée désactivé.

Dans le CSS, on cible cette entrée désactivée et on lui donne une couleur d'arrière-plan `lightgray` et on change le pointeur de curseur en `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

La valeur de propriété CSS `cursor: not-allowed;` change l'apparence du curseur pour indiquer qu'une action n'est pas permise.

Quand appliquée à un élément, elle montre un curseur avec un symbole cercle-barre (généralement un cercle avec une ligne diagonale au travers) pour signaler aux utilisateurs que l'élément n'est pas interactif ou ne peut pas être cliqué ou interagi avec.

Voici quelques autres exemples de pseudo-classes d'entrée et ce qu'elles font :

- `:autofill` : applique des styles aux champs d'entrée que le navigateur remplit automatiquement avec des données sauvegardées.
- `:optional` : stylise les éléments d'entrée qui ne sont pas requis et peuvent être laissés vides.
- `:in-range` et `:out-of-range` : stylisent les éléments en fonction de si leurs valeurs sont à l'intérieur ou à l'extérieur des contraintes de plage spécifiées.
