## Pseudo-classes d'action utilisateur

- **Définition des pseudo-classes** : Ce sont des mots-clés CSS spéciaux qui te permettent de sélectionner un élément en fonction de son état ou de sa position spécifique.
- **Pseudo-classes d'action utilisateur** : Ce sont des mots-clés spéciaux qui te permettent de changer l'apparence des éléments en fonction des interactions utilisateur, améliorant l'expérience utilisateur globale.
- **Pseudo-classe `:active`** : Cette pseudo-classe te permet de sélectionner l'état actif d'un élément, comme cliquer sur un bouton.
- **Pseudo-classe `:hover`** : Cette pseudo-classe définit l'état hover d'un élément.
- **Pseudo-classe `:focus`** : Cette pseudo-classe applique des styles quand un élément reçoit le focus, généralement via la navigation au clavier ou quand un utilisateur clique sur un champ de saisie de formulaire.
- **Pseudo-classe `:focus-within`** : Cette pseudo-classe est utilisée pour appliquer des styles à un élément quand lui ou n'importe lequel de ses descendants a le focus.

## Pseudo-classes d'entrée

- **Pseudo-classes d'entrée** : Ces pseudo-classes sont utilisées pour cibler les éléments HTML `input` en fonction de l'état dans lequel ils se trouvent avant et après l'interaction utilisateur.
- **Pseudo-classe `:enabled`** : Cette pseudo-classe est utilisée pour cibler les boutons de formulaire ou autres éléments qui sont actuellement activés.
- **Pseudo-classe `:disabled`** : Cette pseudo-classe te permet de styliser un élément interactif en mode désactivé.
- **Pseudo-classe `:checked`** : Cette pseudo-classe est utilisée pour indiquer à l'utilisateur qu'il est coché.
- **Pseudo-classe `:valid`** : Cette pseudo-classe cible les champs d'entrée qui respectent les critères de validation.
- **Pseudo-classe `:invalid`** : Cette pseudo-classe cible les champs d'entrée qui ne respectent pas les critères de validation.
- **Pseudo-classes `:in-range` et `:out-of-range`** : Ces pseudo-classes appliquent des styles aux éléments en fonction de si leurs valeurs sont à l'intérieur ou à l'extérieur des contraintes de plage spécifiées.
- **Pseudo-classe `:required`** : Cette pseudo-classe cible les éléments `input` qui ont l'attribut `required`. Elle signale à l'utilisateur qu'il doit remplir le champ pour soumettre le formulaire.
- **Pseudo-classe `:optional`** : Cette pseudo-classe applique des styles aux éléments d'entrée qui ne sont pas requis et peuvent être laissés vides.
- **Pseudo-classe `:autofill`** : Cette pseudo-classe applique des styles aux champs d'entrée que le navigateur remplit automatiquement avec des données sauvegardées.

## Pseudo-classes de localisation

- **Pseudo-classes de localisation** : Ces pseudo-classes sont utilisées pour styliser les liens et les éléments qui sont ciblés dans le document actuel.
- **Pseudo-classe `:any-link`** : Cette pseudo-classe est une combinaison des pseudo-classes `:link` et `:visited`. Donc, elle correspond à n'importe quel élément d'ancrage avec un attribut href, qu'il soit visité ou non.
- **Pseudo-classe `:link`** : Cette pseudo-classe te permet de cibler tous les liens non visités sur une page web. Tu peux l'utiliser pour styliser les liens différemment avant que l'utilisateur ne clique dessus.
- **Pseudo-classe `:local-link`** : Cette pseudo-classe cible les liens qui pointent vers le même document. Elle peut être utile quand tu veux différencier les liens internes des externes.
- **Pseudo-classe `:visited`** : Cette pseudo-classe cible un lien que l'utilisateur a visité.
- **Pseudo-classe `:target`** : Cette pseudo-classe est utilisée pour appliquer des styles à un élément qui est la cible d'un fragment d'URL.

## Pseudo-classes structurelles d'arbre

- **Pseudo-classes structurelles d'arbre** : Ces pseudo-classes te permettent de cibler et styliser les éléments en fonction de leur position dans l'arbre du document.
- **Pseudo-classe `:root`** : Cette pseudo-classe est généralement l'élément `html` racine. Elle t'aide à cibler le plus haut niveau du document pour que tu puisses appliquer un style commun à tout le document.
- **Pseudo-classe `:empty`** : Les éléments vides, c'est-à-dire les éléments sans enfants autres que de l'espace blanc, sont aussi inclus dans l'arbre du document. C'est pour ça qu'il y a une pseudo-classe `:empty` pour cibler les éléments vides.
- **Pseudo-classe `:nth-child(n)`** : Cette pseudo-classe te permet de sélectionner les éléments en fonction de leur position dans un parent.
- **Pseudo-classe `:nth-last-child(n)`** : Cette pseudo-classe te permet de sélectionner les éléments en comptant à partir de la fin.
- **Pseudo-classe `:first-child`** : Cette pseudo-classe sélectionne le premier élément dans un élément parent ou le document.
- **Pseudo-classe `:last-child`** : Cette pseudo-classe sélectionne le dernier élément dans un élément parent ou le document.
- **Pseudo-classe `:only-child`** : Cette pseudo-classe sélectionne le seul élément dans un élément parent ou le document.
- **Pseudo-classe `:first-of-type`** : Cette pseudo-classe sélectionne la première occurrence d'un type d'élément spécifique dans son parent.
- **Pseudo-classe `:last-of-type`** : Cette pseudo-classe sélectionne la dernière occurrence d'un type d'élément spécifique dans son parent.
- **Pseudo-classe `:nth-of-type(n)`** : Cette pseudo-classe te permet de sélectionner un élément spécifique dans son parent en fonction de sa position parmi les frères et sœurs du même type.
- **Pseudo-classe `:only-of-type`** : Cette pseudo-classe sélectionne un élément s'il est le seul de son type dans son parent.

## Pseudo-classes fonctionnelles

- **Pseudo-classes fonctionnelles** : Les pseudo-classes fonctionnelles te permettent de sélectionner les éléments en fonction de conditions ou de relations plus complexes. Contrairement aux pseudo-classes régulières qui ciblent les éléments en fonction d'un état (par exemple, `:hover`, `:focus`), les pseudo-classes fonctionnelles acceptent des arguments.
- **Pseudo-classe `:is()`** : Cette pseudo-classe prend une liste de sélecteurs (ex. `ol`, `ul`) et sélectionne un élément qui correspond à l'un des sélecteurs dans la liste.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />

<p class="example">This text will change color.</p>
<p>This text will not change color.</p>
<p>This text will not change color.</p>
<p class="this-works-too">This text will change color.</p>
```

```css
p:is(.example, .this-works-too) {
    color: red;
}
```

:::

- **Pseudo-classe `:where()`** : Cette pseudo-classe prend une liste de sélecteurs (ex. `ol`, `ul`) et sélectionne un élément qui correspond à l'un des sélecteurs dans la liste. La différence entre `:is` et `:where` est que cette dernière aura une spécificité de 0.

```css
:where(h1, h2, h3) {
    margin: 0;
    padding: 0;
}
```

- **Pseudo-classe `:has()`** : Cette pseudo-classe est souvent surnommée le sélecteur `"parent"` parce qu'elle te permet de styliser les éléments qui contiennent des éléments enfants spécifiés dans la liste de sélecteurs.

```css
article:has(h2) {
    border: 2px solid hotpink;
}
```

- **Pseudo-classe `:not()`** : Cette pseudo-classe est utilisée pour sélectionner les éléments qui ne correspondent pas au sélecteur fourni.

```css
p:not(.example) {
  color: blue;
}
```

## Pseudo-éléments

- **Pseudo-élément `::before`** : Ce pseudo-élément utilise la propriété `content` pour insérer du contenu cosmétique comme des icônes juste avant l'élément.
- **Pseudo-élément `::after`** : Ce pseudo-élément utilise la propriété `content` pour insérer du contenu cosmétique comme des icônes juste après l'élément.
- **Pseudo-élément `::first-letter`** : Ce pseudo-élément cible la première lettre du contenu d'un élément, te permettant de la styliser.
- **Pseudo-élément `::marker`** : Ce pseudo-élément te permet de sélectionner le marqueur (puce ou numérotation) des éléments de liste pour les styliser.
