# --interactive--

## Importance du HTML sémantique

- **Hiérarchie structurelle des éléments de titre** : il est important d'utiliser le bon élément de titre pour conserver la hiérarchie structurelle du contenu. L'élément `h1` est le niveau de titre le plus élevé, et l'élément `h6` est le niveau le plus bas.
- **Éléments HTML de présentation** : éléments qui définissent l'apparence du contenu. Par exemple les anciens éléments dépréciés `center`, `big` et `font`.
- **Éléments HTML sémantiques** : éléments qui portent du sens et de la structure. Par exemple `header`, `nav`, `figure`.

## Éléments HTML sémantiques

- **Élément header** : utilisé pour définir l'en-tête d'un document ou d'une section.

:::interactive_editor

  ```html
  <header>
    <h1>CatPhotoApp</h1>
    <p>Bienvenue dans notre galerie de chats.</p>
  </header>
  ```

:::

- **Élément main** : utilisé pour contenir le contenu principal de la page web.

:::interactive_editor

  ```html
  <main>
    <section>
      <h2>Photos de chats</h2>
      <p>Parcours d'adorables photos de chats.</p>
    </section>
  </main>
  ```

:::

- **Élément section** : utilisé pour diviser le contenu en sections plus petites.

:::interactive_editor

  ```html
  <section>
    <h2>À propos de moi</h2>
    <p>Salut, je suis Jane Doe et je suis développeuse web.</p>
  </section>
  ```

:::

- **Élément de section de navigation (`nav`)** : représente une section avec des liens de navigation.

:::interactive_editor

  ```html
  <nav>
    <ul>
      <li><a href="#photos">Photos</a></li>
      <li><a href="#videos">Vidéos</a></li>
    </ul>
  </nav>
  ```

:::

- **Élément figure** : utilisé pour contenir des illustrations et des diagrammes.

:::interactive_editor

  ```html
  <figure>
    <img
      src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"
      alt="Deux chatons tigrés dormant ensemble sur un canapé."
    />
    <figcaption>Les chats <strong>détestent</strong> les autres chats.</figcaption>
  </figure>
  ```

:::

- **Élément d'emphase (`em`)** : marque un texte sur lequel on met l'accent.

:::interactive_editor

  ```html
  <p>
    N'abandonne jamais <em>tes</em> rêves.
  </p>
  ```

:::

- **Élément de texte idiomatique (`i`)** : utilisé pour mettre en évidence une voix ou une ambiance alternative, des expressions idiomatiques d'une autre langue, des termes techniques et des pensées.

:::interactive_editor

  ```html
  <p>
    Il y a un certain <i lang="fr">je ne sais quoi</i> dans l'air.
  </p>
  ```

:::

L'attribut `lang` dans la balise ouvrante `i` sert à préciser la langue du contenu. Dans ce cas, la langue serait le français. L'élément `i` n'indique pas si le texte est important ou non, il montre seulement qu'il est différent du texte qui l'entoure.

- **Élément d'importance forte (`strong`)** : marque un texte qui a une grande importance.

:::interactive_editor

  ```html
  <p>
    <strong>Attention :</strong> Ce produit peut provoquer des réactions allergiques.
  </p>
  ```

:::

- **Élément d'attention (`b`)** : utilisé pour attirer l'attention sur un texte qui n'est pas important pour le sens du contenu. Il est souvent utilisé pour mettre en évidence des mots-clés dans des résumés ou des noms de produits dans des avis.

:::interactive_editor

  ```html
  <p>
    Nous avons testé plusieurs produits, dont le <b>SuperSound 3000</b> pour la qualité audio, le <b>QuickCharge Pro</b> pour la charge rapide, et l'<b>Ecoclean Vacuum</b> pour le nettoyage. Les deux premiers ont bien fonctionné, mais l'<b>Ecoclean Vacuum</b> n'a pas répondu aux attentes.
  </p>
  ```

:::

- **Élément de liste de descriptions (`dl`)** : utilisé pour représenter une liste de groupes terme-description.
- **Élément de terme de description (`dt`)** : utilisé pour représenter le terme défini.
- **Élément de détails de description (`dd`)** : utilisé pour représenter la description du terme.

:::interactive_editor

  ```html
  <dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
  </dl>
  ```

:::

- **Élément de citation en bloc (`blockquote`)** : utilisé pour représenter une section citée depuis une autre source. Cet élément possède un attribut `cite`. La valeur de l'attribut `cite` est l'URL de la source.

:::interactive_editor

  ```html
  <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
    "Peux-tu imaginer ce que ce serait de devenir un développeur accompli ? D'avoir construit des systèmes logiciels sur lesquels les gens comptent ?"
  </blockquote>
  ```

:::

- **Élément de citation (`cite`)** : utilisé pour attribuer visuellement la source de l'oeuvre référencée. Il marque le titre de la référence.

:::interactive_editor

  ```html
  <div>
    <blockquote cite="https://www.freecodecamp.org/news/learn-to-code-book/">
      "Peux-tu imaginer ce que ce serait de devenir un développeur accompli ? D'avoir construit des systèmes logiciels sur lesquels les gens comptent ?"
    </blockquote>
    <p>
      -Quincy Larson, <cite>How to learn to Code and Get a Developer Job [Full Book].</cite>
    </p>
  </div>
  ```

:::

- **Élément de citation en ligne (`q`)** : utilisé pour représenter une courte citation en ligne.

:::interactive_editor

  ```html
  <p>
    Comme l'a dit Quincy Larson,
    <q cite="https://www.freecodecamp.org/news/learn-to-code-book/">
      L'élan est essentiel.
    </q>
  </p>
  ```

:::

- **Élément d'abréviation (`abbr`)** : utilisé pour représenter une abréviation ou un acronyme. Pour aider les utilisateurs à comprendre l'abréviation ou l'acronyme, tu peux afficher sa forme complète, une description lisible par un humain, avec l'attribut `title`.

:::interactive_editor

  ```html
  <p>
    <abbr title="HyperText Markup Language">HTML</abbr> est la base du web.
  </p>
  ```

:::

- **Élément d'adresse de contact (`address`)** : utilisé pour représenter les informations de contact.
- **Élément de date/heure (`time`)** : utilisé pour représenter une date et/ou une heure. L'attribut `datetime` sert à traduire les dates et heures dans un format lisible par les machines.

:::interactive_editor

  ```html
  <p>
    Les réservations sont pour <time datetime="20:00">20:00 </time>
  </p>
  ```

:::

- **Attribut de date ISO 8601 (`datetime`)** : utilisé pour représenter des dates et des heures dans un format lisible par les machines. Le format standard est `YYYY-MM-DDThh:mm:ss`, avec le `T` majuscule comme séparateur entre la date et l'heure.
- **Élément exposant (`sup`)** : utilisé pour représenter du texte en exposant. Les cas d'utilisation courants de l'élément `sup` incluent les exposants mathématiques, les lettres supérieures et les nombres ordinaux.

:::interactive_editor

  ```html
  <p>
    2<sup>2</sup> (2 au carré) vaut 4.
  </p>
  ```

:::

- **Élément indice (`sub`)** : utilisé pour représenter du texte en indice. Les cas d'utilisation courants de l'élément `sub` incluent les formules chimiques, les notes de bas de page et les indices de variables.

:::interactive_editor

  ```html
  <p>
    CO<sub>2</sub>
  </p>
  ```

:::

- **Élément de code en ligne (`code`)** : utilisé pour représenter un fragment de code informatique.
- **Élément de texte préformaté (`pre`)** : représente du texte préformaté.

:::interactive_editor

  ```html
  <pre>
    <code>
      body {
        color: red;
      }
    </code>
  </pre>
  ```

:::

- **Élément d'annotation non articulée (`u`)** : utilisé pour représenter un morceau de texte en ligne qui doit être rendu d'une façon indiquant qu'il a une annotation non textuelle.

:::interactive_editor

  ```html
  <p>
    Tu peux utiliser l'élément d'annotation non articulée pour mettre en évidence
    <u>des fotes</u> <u>d'ortografe</u> <u>ici</u>.
  </p>
  ```

:::

- **Élément d'annotation ruby (`ruby`)** : utilisé pour annoter du texte avec des explications de prononciation ou de sens. Un exemple d'utilisation concerne la typographie d'Asie de l'Est.
- **Élément parenthèse de fallback ruby (`rp`)** : utilisé comme solution de repli pour les navigateurs qui ne prennent pas en charge l'affichage des annotations ruby.
- **Élément de texte ruby (`rt`)** : utilisé pour indiquer le texte de l'annotation ruby. Il sert généralement à la prononciation ou aux détails de traduction en typographie d'Asie de l'Est.

:::interactive_editor

  ```html
  <ruby>
    明日 <rp>(</rp><rt>Ashita</rt><rp>)</rp>
  </ruby>
  ```

:::

- **Élément barré (`s`)** : utilisé pour représenter du contenu qui n'est plus exact ou pertinent.

:::interactive_editor

  ```html
  <p>
    <s>La randonnée de demain aura rendez-vous à midi.</s>
  </p>
  <p>
    En raison de conditions météo imprévues, la randonnée a été annulée.
  </p>
  ```

:::

- **Liens internes** : utilisés pour créer un lien vers une autre section de la page avec `href="#id"` sur un élément `a`, puis en donnant le même `id` à l'élément de destination. C'est souvent utilisé pour les liens d'évitement, les tables des matières ou les longues pages avec plusieurs sections.

:::interactive_editor

```html
<a href="#about-section">Aller à la section À propos</a>

<section id="about-section">
  <h2>À propos</h2>
  <p>Ceci est la section À propos de la page.</p>
</section>
```

:::
