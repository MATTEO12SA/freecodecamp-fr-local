# --description--

Découvrons le boilerplate HTML.

Qu'est-ce que le boilerplate HTML ? C'est comme un modèle prêt à l'emploi pour tes pages web. Imagine-le comme les fondations d'une maison. Un boilerplate contient la structure de base et les éléments essentiels dont chaque document HTML a besoin. Il te fait gagner du temps et t'aide à t'assurer que tes pages sont correctement configurées. Voici un exemple :

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
  </body>
</html>
```

Découpons les parties clés de ce boilerplate. D'abord, il y a la déclaration `DOCTYPE` :

```html
<!DOCTYPE html>
```

Elle indique aux navigateurs quelle version de HTML tu utilises. Ensuite vient la balise `html` :

```html
<!DOCTYPE html>
<html lang="en">
  <!--All other elements go inside here-->
</html>
```

Elle englobe tout ton contenu et peut préciser la langue de ta page. Dans la balise `html`, tu trouveras deux sections principales : un `head` et un `body` :

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!--Important metadata goes here-->
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

La section `head` contient des informations importantes en arrière-plan :

```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Document Title Goes Here</title>
  <link rel="stylesheet" href="./styles.css" />
</head>
```

Les métadonnées de ton site, contenues dans les éléments `meta`, donnent des détails sur des choses comme l'encodage des caractères et la façon dont des sites comme Twitter doivent prévisualiser le lien de ta page. Le titre de ton site, défini dans l'élément `title`, détermine le texte qui apparaît dans l'onglet ou la fenêtre du navigateur. Enfin, tu lieras généralement les feuilles de style externes de ta page dans la section `head` avec des éléments `link`.

La section `body` est l'endroit où va tout ton contenu :

```html
<body>
  <h1>I am a main title</h1>
  <p>Example paragraph text</p>
</body>
```

Pourquoi un boilerplate est-il important ? Il garantit que tes pages sont correctement structurées et fonctionnent bien dans différents navigateurs. Utiliser un boilerplate t'aide à éviter les erreurs courantes et à suivre les bonnes pratiques. C'est un excellent point de départ pour n'importe quel projet web. N'oublie pas que tu peux personnaliser ton propre boilerplate pour l'adapter à tes besoins. Avec l'expérience, tu pourrais ajouter tes propres éléments préférés ou balises `meta`. Plus tu amélioreras ton boilerplate personnel, plus tu verras qu'il te fait gagner du temps au démarrage de nouveaux projets.

La prochaine fois que tu commenceras un nouveau fichier HTML, pense à utiliser un boilerplate. Il te donnera une base solide sur laquelle construire.
