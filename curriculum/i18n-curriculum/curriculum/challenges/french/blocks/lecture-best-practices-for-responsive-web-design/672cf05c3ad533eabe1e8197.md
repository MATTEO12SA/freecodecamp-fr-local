# --interactive--

Les media queries sont un composant fondamental du design web responsive. Elles permettent aux développeurs d'appliquer différents styles selon les caractéristiques de l'appareil ou du navigateur de l'utilisateur. Elles fournissent un moyen d'adapter la présentation du contenu à une variété d'appareils sans modifier le contenu lui-même.

À la base, une media query se compose d'un type de média et d'une ou plusieurs expressions qui vérifient des conditions précises. Si ces conditions sont vraies, les styles correspondants sont appliqués. Ce mécanisme permet de créer des designs responsives qui s'adaptent à différentes tailles d'écran, résolutions et capacités d'appareil.

La syntaxe de base d'une media query en CSS ressemble à ceci :

```css
@media mediatype and (feature: value) {
  /* CSS rules go here */
}
```

Dans cette structure, `mediatype` précise le type de média auquel la requête s'applique, et la paire `feature: value` définit la condition qui doit être remplie pour que les styles soient appliqués.

Les types de média décrivent la catégorie générale d'un appareil. Parlons des types de média les plus couramment utilisés :

- `all` convient à tous les appareils. C'est la valeur par défaut si aucun type de média n'est précisé.

- `print` est destiné aux documents paginés et aux documents affichés à l'écran en mode aperçu avant impression.

- `screen` est principalement destiné aux écrans.

Par le passé, il existait davantage de types de média, comme `handheld` et `tv`, mais la plupart ont été dépréciés au profit des fonctionnalités media, qui permettent de cibler les appareils plus précisément.

Les fonctionnalités media décrivent des caractéristiques précises de l'agent utilisateur, du périphérique de sortie ou de l'environnement. Parlons de quelques-unes des fonctionnalités media les plus couramment utilisées. `width` et `height` font référence à la largeur et à la hauteur du viewport, et sont souvent utilisés avec les préfixes `min-` ou `max-` pour les requêtes par plage. Voici un exemple :

```css
@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
}
```

`aspect-ratio` décrit le ratio entre la largeur et la hauteur du viewport. Voici un exemple :

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

La fonctionnalité `orientation` indique si l'appareil est en orientation paysage ou portrait. Voici un exemple :

```css
@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
}
```

La fonctionnalité `resolution` décrit la résolution du périphérique de sortie en points par pouce (dpi) ou en points par centimètre (dpcm). Voici un exemple :

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

La fonctionnalité `hover` teste si le mécanisme d'entrée principal peut survoler des éléments. Voici un exemple :

```css
@media (hover: hover) {
  /* Styles for devices that support hover */
}
```

La fonctionnalité `prefers-color-scheme` détecte si l'utilisateur a demandé un thème de couleur clair ou sombre. Voici un exemple :

```css
@media (prefers-color-scheme: dark) {
  /* Styles for dark mode */
}
```

Les media queries peuvent aussi combiner plusieurs conditions avec des opérateurs logiques. L'opérateur `and` sert à combiner plusieurs fonctionnalités media, tandis que `not` et `only` peuvent servir à inverser ou isoler des media queries. Voici un exemple qui combine plusieurs fonctionnalités :

```css
@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
}
```

Il est aussi possible de cibler plusieurs requêtes dans une liste séparée par des virgules, ce qui fonctionne comme un opérateur « ou » :

```css
@media screen and (min-width: 768px), print {
  /* Styles for screens at least 768px wide OR for print */
}
```

Quand tu travailles avec des media queries, il est important de prendre en compte la cascade. Les media queries n'augmentent pas la spécificité : elles regroupent simplement des règles conditionnelles. Les règles normales de la cascade CSS s'appliquent toujours à l'intérieur de chaque media query.

En pratique, les media queries sont souvent utilisées pour créer des mises en page responsives. Un motif courant consiste à définir un style de base pour les appareils mobiles, puis à utiliser des media queries pour améliorer la mise en page sur les écrans plus grands :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="container">
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer nec odio. Praesent libero. Sed cursus ante dapibus diam.</p>
  <p>Sed nisi. Nulla quis sem at nibh elementum imperdiet. Duis sagittis ipsum. Praesent mauris. Fusce nec tellus sed augue semper porta.</p>
  <p>Mauris massa. Vestibulum lacinia arcu eget nulla. Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 15px;
}

/* Styles for tablets */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Styles for desktops */
@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::

Cette approche, connue sous le nom de design responsive « mobile-first », garantit que les styles de base conviennent aux appareils mobiles, avec des améliorations ajoutées pour les écrans plus grands.

Pour conclure, les media queries sont un outil puissant en CSS qui permet de créer des designs web responsives et adaptables. En comprenant comment utiliser différents types et fonctionnalités media, les développeurs peuvent créer des sites web offrant une expérience utilisateur optimale sur une grande variété d'appareils et de préférences. À mesure que les technologies web évoluent, rester à jour avec les nouvelles fonctionnalités media peut aider à créer des designs responsives plus nuancés et plus agréables à utiliser.
