# --description--

Les développeurs professionnels s'appuient sur une variété d'outils pour augmenter leur productivité et la qualité du code. Découvrons ces outils, y compris ceux qui semblent évidents.

Le premier de ces outils est l'ordinateur. Un ordinateur est l'environnement de développement principal. Ça peut être un gros desktop ou un laptop portable avec Windows, macOS ou Linux comme système d'exploitation.

Les développeurs professionnels choisissent souvent des ordinateurs avec une puissance de traitement rapide et beaucoup de RAM pour gérer les tâches gourmandes en ressources.

Après l'ordinateur, il y a un programme pour écrire et éditer le code directement sur cet ordinateur. C'est un éditeur de code ou un environnement de développement intégré (IDE).

Les IDE comme Visual Studio, IntelliJ IDEA, JetBrains et PyCharm offrent des fonctionnalités puissantes comme la complétion de code, le débogage et le support de terminal intégré.

Visual Studio Code (VS Code) est essentiellement un éditeur de code, mais il offre aussi ces fonctionnalités via sa riche bibliothèque d'extensions.

D'autres éditeurs de code sont Sublime Text, Atom, Notepad++ et Brackets.

Quand tu écris du code avec des éditeurs de code et des IDE, tu dois suivre les changements que tu y apportes. L'outil qui te permet de suivre ces changements est un système de contrôle de version.

Git est le système de contrôle de version le plus utilisé dans la communauté du développement.

Des plateformes comme GitHub, GitLab et Bitbucket offrent des services d'hébergement cloud pour tes dépôts Git. Ces plateformes permettent la collaboration avec d'autres développeurs, te permettent de travailler sur plusieurs branches et facilitent la fusion des changements de code.

Les gestionnaires de paquets sont un autre outil critique. Ils aident les développeurs à simplifier le processus d'ajout, de mise à jour et de suppression des bibliothèques et dépendances du projet.

Des exemples de gestionnaires de paquets populaires sont :

* NPM, Yarn et PNPM pour JavaScript
* PIP pour Python
* Composer pour PHP
* Maven pour Java

Après avoir écrit du code avec différents outils, les développeurs testent ce code pour s'assurer qu'il fonctionne comme prévu.

Pour ça, les développeurs utilisent des frameworks de test comme Cypress, Playwright, Selenium et d'autres. Il existe aussi des frameworks de test spécifiques au langage. Des exemples sont :

* Jest pour JavaScript
* pytest pour Python
* JUnit pour Java
* PHPUnit pour PHP

Tu n'as pas seulement besoin de tester le code pour t'assurer qu'il fonctionne comme prévu. Tu dois aussi tester l'aspect visuel du code pour les utilisateurs finaux. Les développeurs utilisent des navigateurs web pour ça.

Les navigateurs modernes comme Chrome, Firefox, Edge et Safari proposent des outils de développement pour inspecter le code HTML, CSS et JavaScript. Il y a aussi des outils pour le débogage et le profilage de performance.

Ces outils aident les développeurs à tester et optimiser leurs applications web sur différents navigateurs pour les utilisateurs finaux.
