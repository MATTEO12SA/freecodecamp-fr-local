# --interactive--

Les listes de description sont parfaites pour présenter des termes et des définitions dans un format organisé et facile à lire, comme dans un glossaire ou un vrai dictionnaire, où tu peux trouver des mots avec leurs définitions correspondantes.

Voici un exemple de liste de description en HTML avec deux termes et leurs détails correspondants.

Active l'éditeur interactif et essaie de décommenter le code pour voir le nouvel élément de détail apparaître dans la fenêtre d'aperçu.

:::interactive_editor

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
  <!-- <dt>JS</dt>
  <dd>JavaScript</dd> -->
</dl>
```

:::

Dans ce cas, les termes sont les acronymes HTML et CSS, et les détails sont leur signification développée. Les détails pourraient aussi être des définitions ou d'autres informations liées aux termes.

Tu auras besoin de trois éléments HTML pour définir une liste de description. D'abord, l'élément de liste de description, `dl`, qui est le conteneur pour toute la liste. Tu peux voir qu'il entoure tous les autres éléments de la liste de description dans l'exemple.

Ensuite, un élément de terme de description, `dt`, pour chaque terme. Dans ce cas, la liste de description a deux termes, HTML et CSS, donc elle a deux de ces éléments.

Et enfin, après chaque terme tu trouveras un élément de détails de description, `dd`, pour la description ou les détails associés à ce terme. Dans cet exemple, ce sont Hypertext Markup Language et Cascading Style Sheets.

Dans le navigateur, tu verrais chaque terme suivi de sa description correspondante. Par défaut, les descriptions sont légèrement plus en retrait vers la droite pour les distinguer visuellement.

Mais les listes de description ne sont pas limitées qu'aux termes et définitions. Elles sont beaucoup plus polyvalentes que ça. Ici, on a une recette avec deux ingrédients.

Active l'éditeur interactif et essaie de décommenter le code pour voir le nouvel élément de détail apparaître dans la fenêtre d'aperçu.

:::interactive_editor

```html
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  <!-- <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd> -->
</dl>
```

:::

Toute la liste de description est dans un élément de liste de description. Le premier ingrédient, `Flour`, est dans un élément de terme de description. Ensuite, tu peux voir la quantité dont tu auras besoin de cet ingrédient : `2 cups`. Ceci est dans un élément de détails de description placé directement après son ingrédient correspondant.

Et la même structure est répétée pour `Sugar`. Dans ce cas, la recette n'a que deux ingrédients, mais s'il y en avait plus, la même structure pourrait être répétée dans toute la liste de description.

Dans le navigateur, tu verrais les ingrédients alignés à gauche, et les mesures en retrait pour les séparer visuellement.

D'autres cas d'usage pour les listes de description incluent les spécifications de produits, les questions fréquentes, les informations de contact, et les métadonnées. En gros, quand tu as deux informations liées dans un format clé-valeur, où l'une fait office d'étiquette, la clé, et l'autre fait office d'information liée supplémentaire, la valeur, tu peux utiliser une liste de description.
