# --interactive--

Les animations CSS te permettent de créer des effets dynamiques et visuellement engageants sur les pages web sans avoir besoin de JavaScript ni de programmation complexe. Elles fournissent un moyen de faire passer progressivement des éléments entre différents styles pendant une durée donnée.

À la base, une animation CSS se compose de deux éléments principaux : la règle `@keyframes` et la propriété animation.

La règle `@keyframes` définit les étapes et les styles de l'animation. Elle précise les styles que l'élément doit avoir à différents moments de l'animation.

Voici un exemple :

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}
```

Cette règle `@keyframes`, nommée `slide-in`, définit une animation qui déplace un élément de gauche à droite. Les pourcentages représentent la progression de l'animation, `0%` étant le début et `100%` la fin.

La fonction `translateX` dans ton animation `@keyframes` contrôle la position horizontale d'un élément pendant qu'il apparaît à l'écran.

Pour appliquer cette animation à un élément, tu utilises la propriété `animation`. Cet exemple répète aussi l'animation à l'infini pour que tu puisses la voir en action (tu devras activer l'éditeur interactif pour voir les aperçus) :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="sliding-element">Hello, I slide in!</div>
```

```css
@keyframes slide-in {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}

.sliding-element {
  animation: slide-in 2s ease-out infinite;
}
```

:::

Cela applique l'animation `slide-in` à l'élément avec une durée de 2 secondes et une fonction de temporisation `ease-out`.

La propriété `animation` est en réalité un raccourci pour plusieurs propriétés individuelles :

`animation-name`, qui indique la règle `@keyframes` à utiliser.

`animation-duration`, qui définit le temps que l'animation doit prendre pour se terminer.

`animation-timing-function`, qui définit la manière dont l'animation progresse dans le temps, par exemple `ease`, `linear` ou `ease-in-out`.

`animation-delay`, qui indique un délai avant le démarrage de l'animation.

`animation-iteration-count`, qui définit combien de fois l'animation doit se répéter.

`animation-direction`, qui détermine si l'animation doit être jouée vers l'avant, vers l'arrière ou en alternance.

`animation-fill-mode`, qui indique comment l'élément doit être stylisé avant et après l'animation.

`animation-play-state`, qui te permet de mettre l'animation en pause et de la reprendre.

Tu peux utiliser ces propriétés individuellement pour un contrôle plus précis :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="complex-animation">Watch my colors change!</div>
```

```css
.complex-animation {
  animation-name: color-change;
  animation-duration: 3s;
  animation-timing-function: linear;
  animation-delay: 1s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}

@keyframes color-change {
  0% {
    background-color: red;
  }
  50% {
    background-color: blue;
  }
  100% {
    background-color: green;
  }
}
```

:::

Cela crée une animation qui fait passer en continu la couleur de fond d'un élément entre rouge, bleu et vert.

Les animations CSS peuvent être déclenchées par différents événements, comme le survol d'un élément :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Hover over me!</button>
```

```css
.button {
  background-color: blue;
  transition: background-color 0.3s;
}

.button:hover {
  background-color: red;
}
```

:::

Même si cet exemple utilise la propriété `transition`, qui est plus simple pour les effets de base, il montre comment CSS peut créer des éléments interactifs et animés.

Il est important de noter que, même si les animations CSS sont puissantes, elles doivent être utilisées avec modération. Un usage excessif des animations peut entraîner de mauvaises performances et peut distraire ou poser problème à des utilisateurs ayant certains besoins d'accessibilité. Pense toujours à fournir des options pour réduire ou désactiver les animations pour les utilisateurs qui préfèrent moins de mouvement.

Les animations CSS offrent un moyen de créer des expériences web engageantes et interactives sans dépendre de JavaScript. En comprenant les principes des propriétés `@keyframes` et `animation`, tu peux donner vie à tes designs web de manière performante et accessible.
