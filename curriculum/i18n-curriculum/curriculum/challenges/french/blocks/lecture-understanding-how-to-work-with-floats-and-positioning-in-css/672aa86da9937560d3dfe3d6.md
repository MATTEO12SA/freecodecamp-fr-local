# --interactive--

Les flottants en CSS sont une technique conçue à l'origine pour permettre au texte de s'enrouler autour d'un élément, comme une image. Avec le temps, les développeurs ont toutefois trouvé de nouvelles façons d'utiliser les flottants, en les appliquant de manière créative à la conception de mises en page. Même si les méthodes modernes de mise en page comme Flexbox et Grid sont maintenant plus courantes, comprendre les flottants reste important, surtout quand tu travailles avec du code ancien ou que tu dois produire des effets de mise en page précis.

Quand un élément est flottant, il est retiré du flux normal du document et poussé vers la gauche ou la droite de son conteneur. Le contenu qui suit s'enroule autour de l'élément flottant et remplit l'espace restant. Un usage classique consiste à faire s'enrouler du texte autour d'images : une image flotte d'un côté, et le texte l'entoure. Cette technique est encore largement utilisée, surtout dans les articles et les blogs où des images doivent être placées à côté du texte. Voici l'exemple de code :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
</div>
```

```css
.container {
  border: 1px solid black;
  padding: 10px;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

Dans l'exemple ci-dessus, le conteneur ne s'étend pas autour de l'image flottante. L'image est hors du flux normal du document, et le conteneur s'est réduit à une hauteur nulle parce qu'il ne « voit » pas les éléments enfants flottants.

Les flottants étaient aussi populaires pour créer des mises en page à plusieurs colonnes avant que Flexbox et Grid ne deviennent largement utilisés. En faisant flotter des éléments les uns à côté des autres, les développeurs pouvaient créer des colonnes alignées horizontalement. Cependant, quand tu utilises des flottants, il est important de gérer le problème des éléments parents qui s'effondrent lorsque leurs éléments enfants flottent. La technique du `clearfix` est appliquée à l'élément de classe `container` pour corriger ce problème.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <img src="https://placehold.co/150x150" alt="Placeholder Image">
  <p> This is an example of text flowing around a floated image.</p>
</div>
```

```css
.container {
  border: 1px solid black;
}

/* Clearfix CSS */
.container::after {
  content: "";
  display: block;
  clear: both;
}

img {
  float: left;
  margin-right: 20px;
}
```

:::

- `::after` est un pseudo-élément qui ajoute un bloc invisible après le contenu du conteneur.
- `content: ""` garantit que le pseudo-élément est présent sans afficher de contenu.
- `display: block` transforme le pseudo-élément en élément de niveau bloc.
- `clear: both` garantit que le pseudo-élément passe sous les éléments flottants situés au-dessus, des deux côtés.

Dans l'exemple ci-dessus, nous avons ajouté un nouvel élément paragraphe pour rendre l'effondrement plus visible. Comme le paragraphe reste dans le flux normal du document, le conteneur s'agrandit juste assez pour l'entourer. Ensuite, nous appliquons la technique du `clearfix` pour corriger l'effondrement et afficher correctement la bordure du conteneur. Tu peux essayer de supprimer la règle `clearfix` pour voir le conteneur s'effondrer de nouveau.

La technique du `clearfix` garantit que l'élément parent s'étend correctement autour de ses enfants flottants. Elle force le conteneur parent à « voir » les éléments enfants flottants en ajoutant une propriété `clear` après le contenu flottant.

Même si les flottants ne sont plus la méthode privilégiée pour les mises en page complexes grâce à des techniques plus modernes comme Flexbox et Grid, ils jouent encore un rôle essentiel dans certains scénarios. Que tu fasses s'enrouler du texte autour d'images ou que tu travailles sur des projets existants, comprendre le fonctionnement des flottants et savoir les nettoyer correctement est essentiel pour maintenir des mises en page bien structurées et responsives.
