## Bases du CSS

- **Qu'est-ce que le CSS ?** : Les Cascading Style Sheets (CSS) sont un langage de balisage utilisé pour appliquer des styles aux éléments HTML. Le CSS est utilisé pour les couleurs, les images d'arrière-plan, les mises en page et plus encore.
- **Anatomie de base d'une règle CSS** : Une règle CSS est composée de deux parties principales : un sélecteur et un bloc de déclaration. Un sélecteur est un motif utilisé en CSS pour identifier et cibler des éléments HTML spécifiques pour la mise en forme. Un bloc de déclaration applique un ensemble de styles pour un sélecteur ou des sélecteurs donnés.

Voici la syntaxe générale d'une règle CSS :

```css
selector {
    property: value;
}
```

- **Élément `meta name="viewport"`** : Cet élément `meta` donne au navigateur des instructions sur la façon de contrôler les dimensions et la mise à l'échelle de la page sur différents appareils, en particulier sur les téléphones mobiles et les tablettes.
- **Styles par défaut du navigateur** : Chaque élément HTML aura des styles par défaut du navigateur qui lui sont appliqués. Cela inclut généralement des éléments comme les margins et paddings par défaut.

## CSS inline, interne et externe

- **CSS inline** : Ces styles sont écrits directement dans un élément HTML en utilisant l'attribut `style`. La plupart du temps, tu n'utiliseras pas le CSS inline en raison de la séparation des préoccupations.

Voici un exemple de CSS inline :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p style="color: red;">This is a red paragraph.</p>
```

:::

- **CSS interne** : Ces styles sont écrits dans les balises `<style>` à l'intérieur de la section `head` d'un document HTML. Cela peut être utile pour créer de courts exemples de code, mais généralement tu n'auras pas besoin d'utiliser le CSS interne.
- **CSS externe** : Ces styles sont écrits dans un fichier CSS séparé et liés au document HTML en utilisant l'élément `link` dans la section `head`. Pour la plupart des projets, tu utiliseras un fichier CSS externe plutôt que du CSS interne ou inline.

## Travailler avec les propriétés `width` et `height`

- **Propriété `width`** : Cette propriété spécifie la largeur d'un élément. Si tu ne spécifies pas de largeur, alors la valeur par défaut est `auto`. Cela laisse le navigateur déterminer la largeur de l'élément en fonction de son contenu, de son parent et de son type d'affichage.
- **Propriété `min-width`** : Cette propriété spécifie la largeur minimale d'un élément.
- **Propriété `max-width`** : Cette propriété spécifie la largeur maximale d'un élément.
- **Propriété `height`** : Cette propriété spécifie la hauteur d'un élément. De manière similaire, la hauteur est `auto` par défaut, ce qui signifie qu'elle s'ajustera au contenu à l'intérieur.
- **Propriété `min-height`** : Cette propriété spécifie la hauteur minimale d'un élément.
- **Propriété `max-height`** : Cette propriété spécifie la hauteur maximale d'un élément.

## Différents types de combinateurs CSS

- **Combinateur descendant** : Ce combinateur est utilisé pour cibler les éléments qui sont des descendants d'un élément parent spécifié. L'exemple suivant ciblera tous les éléments `li` à l'intérieur des éléments `ul`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
</ul>
```

```css
ul li {
    background-color: yellow;
}
```

:::

- **Combinateur enfant (`>`)** : Ce combinateur est utilisé pour sélectionner les éléments qui sont des enfants directs d'un élément parent spécifié. L'exemple suivant ciblera tous les éléments `p` qui sont des enfants directs de la classe `container`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will get styled.</p>

  <div>
    <p>This will not get styled.</p>
  </div>
</div>
```

```css
.container > p {
  background-color: black;
  color: white;
}
```

:::

- **Combinateur de frère adjacent (`+`)** : Ce combinateur sélectionne un élément qui suit immédiatement un élément frère spécifié. L'exemple suivant sélectionnera l'élément paragraphe qui suit immédiatement l'élément `h2`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h2>I am a sub heading</h2>

<p>This paragraph element will get a red background.</p>
```

```css
h2 + p {
  background-color: red;
}
```

:::

- **Combinateur de frères suivants (`~`)** : Ce combinateur sélectionne tous les frères d'un élément spécifié qui viennent après lui. L'exemple suivant stylisera uniquement le second élément paragraphe parce qu'il est le seul qui est un frère de l'élément `ul` et partage le même parent.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>This will not get styled.</p>
  <ul>
    <li>Example item one</li>
    <li>Example item two</li>
    <li>Example item three</li>
  </ul>
  <p>This will get styled.</p>
</div>
<p>This will not get styled.</p>
```

```css
ul ~ p {
  background-color: green;
}
```

:::

## Éléments inline, block et inline-block

- **Éléments inline** : Les éléments inline ne prennent que la largeur dont ils ont besoin et ne commencent pas sur une nouvelle ligne. Ces éléments s'écoulent dans le contenu, permettant au texte et à d'autres éléments inline d'apparaître à côté d'eux. Les éléments inline courants sont les éléments `span`, `anchor` et `img`.
- **Éléments block** : Les éléments block-level commencent sur une nouvelle ligne et prennent par défaut toute la largeur disponible, s'étendant sur toute la largeur de leur conteneur. Quelques éléments block-level courants sont les éléments `div`, `paragraph` et `section`.
- **Éléments inline-block** : Tu peux définir un élément à `inline-block` en utilisant la propriété `display`. Ces éléments se comportent comme des éléments inline mais peuvent avoir une `width` et une `height` définies comme les éléments block-level.

## Margin et padding

- **Propriété `margin`** : Cette propriété est utilisée pour appliquer de l'espace à l'extérieur de l'élément, entre la bordure de l'élément et les éléments environnants.
- **Propriété `padding`** : Cette propriété est utilisée pour appliquer de l'espace à l'intérieur de l'élément, entre le contenu et sa bordure.
- **Raccourci `margin`** : Tu peux spécifier 1 à 4 valeurs pour définir les côtés de la margin. Une valeur s'applique aux quatre côtés ; deux valeurs définissent `top` et `bottom`, puis `right` et `left` ; trois valeurs définissent `top`, horizontal (`right` et `left`), puis `bottom` ; quatre valeurs définissent `top`, `right`, `bottom`, `left`.
- **Raccourci `padding`** : Tu peux spécifier 1 à 4 valeurs pour définir les côtés du padding. Une valeur s'applique aux quatre côtés ; deux valeurs définissent `top` et `bottom`, puis `right` et `left` ; trois valeurs définissent `top`, horizontal (`right` et `left`), puis `bottom` ; quatre valeurs définissent `top`, `right`, `bottom`, `left`.

## Spécificité CSS

- **Spécificité du CSS inline** : Le CSS inline a la spécificité la plus élevée parce qu'il est appliqué directement à l'élément. Il remplace tout CSS interne ou externe. La valeur de spécificité pour les styles inline est (1, 0, 0, 0).
- **Spécificité du CSS interne** : Le CSS interne est défini dans un élément `style` dans la section `head` du document HTML. Il a une spécificité plus faible que les styles inline. Le CSS interne et externe partagent le même niveau de spécificité, qui est déterminé par leurs sélecteurs, pas par leur emplacement.
- **Spécificité du CSS externe** : Le CSS externe est lié via un élément `link` dans la section `head` et est écrit dans des fichiers `.css` séparés. Comme le CSS interne, sa spécificité est déterminée par les sélecteurs utilisés. Quand deux règles ont une spécificité égale, l'ordre d'apparition détermine le gagnant — la règle qui apparaît plus tard dans le document a la priorité. Le CSS externe offre la meilleure maintenabilité pour les gros projets.
- **Sélecteur universel (`*`)** : un type spécial de sélecteur CSS qui correspond à n'importe quel élément du document. Il est souvent utilisé pour appliquer un style à tous les éléments de la page, ce qui peut être utile pour réinitialiser ou normaliser les styles entre différents navigateurs. Le sélecteur universel a la valeur de spécificité la plus faible de tous les sélecteurs. Il contribue 0 à toutes les parties de la valeur de spécificité (0, 0, 0, 0).
- **Sélecteurs de type** : Ces sélecteurs ciblent les éléments en fonction de leur nom de balise. Les sélecteurs de type ont une spécificité relativement faible par rapport aux autres sélecteurs. La valeur de spécificité pour un sélecteur de type est (0, 0, 0, 1).
- **Sélecteurs de classe** : Ces sélecteurs sont définis par un point (`.`) suivi du nom de la classe. La valeur de spécificité pour un sélecteur de classe est (0, 0, 1, 0). Cela signifie que les sélecteurs de classe peuvent remplacer les sélecteurs de type, mais peuvent être remplacés par les sélecteurs ID et les styles inline.
- **Sélecteurs ID** : Les sélecteurs ID sont définis par un symbole dièse (`#`) suivi du nom de l'ID. Les sélecteurs ID ont une très haute spécificité, plus élevée que les sélecteurs de type et les sélecteurs de classe, mais plus faible que les styles inline. La valeur de spécificité pour un sélecteur ID est (0, 1, 0, 0).
- **Mot-clé `!important`** : utilisé pour donner à une règle de style la priorité la plus élevée, lui permettant de remplacer toutes les autres déclarations pour une propriété. Quand il est utilisé, il force le navigateur à appliquer le style spécifié, indépendamment de la spécificité des autres sélecteurs. Tu devrais être prudent en utilisant `!important` parce que ça peut rendre ton CSS plus difficile à maintenir et déboguer.
- **Algorithme de cascade** : Un algorithme utilisé pour décider quelles règles CSS appliquer quand il y a plusieurs styles ciblant le même élément. Il garantit que les styles les plus appropriés sont utilisés, en se basant sur un ensemble de règles bien définies.
- **Héritage CSS** : Le processus par lequel les styles sont transmis des éléments parents à leurs enfants. L'héritage te permet de définir des styles à un niveau plus élevé dans l'arbre du document et de les avoir appliqués à plusieurs éléments sans les spécifier explicitement pour chaque élément.
