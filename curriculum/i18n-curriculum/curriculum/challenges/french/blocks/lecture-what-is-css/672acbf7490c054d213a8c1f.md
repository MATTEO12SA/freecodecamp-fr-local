# --interactive--

Le CSS peut être appliqué à une page web de trois façons principales : inline, interne ou externe.

Chaque méthode a son propre cas d'utilisation, ses avantages et ses limites, et savoir quand utiliser chacune est essentiel pour écrire du code propre, efficace et maintenable.

Décomposons les trois types de CSS et quand tu devrais les utiliser.

Le CSS inline est écrit directement à l'intérieur d'un élément HTML en utilisant l'attribut `style`. Il applique des styles à un élément spécifique.

Voici un exemple utilisant du CSS inline :

:::interactive_editor

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

:::

Dans cet exemple, on utilise l'attribut `style` pour mettre le texte du paragraphe en `green`.

Le CSS inline est généralement utilisé pour des styles rapides et ponctuels ou pour remplacer d'autres styles pour un élément spécifique.

Cependant, il devrait être évité dans la plupart des cas parce qu'il peut encombrer le HTML et rendre le code plus difficile à maintenir.

La plupart du temps, il vaut mieux utiliser du CSS interne ou externe pour garder tes styles organisés et maintenables.

Le CSS interne est écrit à l'intérieur des balises `style` dans la section `head` d'un document HTML. Il applique des styles à toute la page et est utile quand tu dois styliser un seul document.

Voici un exemple de CSS interne :

:::interactive_editor

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

:::

Dans l'exemple ci-dessus, le CSS interne applique du texte bleu à tous les éléments `p` de la page.

Le CSS interne est mieux utilisé quand tu dois appliquer des styles à une page spécifique plutôt qu'à plusieurs pages. C'est utile pour les sites web d'une seule page ou quand les styles n'ont pas besoin d'être réutilisés ailleurs.

Cependant, il y a certains inconvénients, comme le fait de ne pas favoriser la réutilisabilité sur plusieurs pages. De plus, comme le CSS inline, il mélange HTML et CSS, ce qui rend le code plus difficile à maintenir dans les projets plus gros.

Le CSS externe est écrit dans un fichier `.css` séparé et lié au document HTML en utilisant l'élément `link` dans la section `head`.

Il te permet de styliser plusieurs pages de manière cohérente et c'est la méthode préférée dans le développement web professionnel.

Voici un exemple de mise en forme de tous les éléments paragraphe :

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

:::

Dans une leçon précédente, tu as appris que l'élément `link` a un attribut `rel` qui spécifie la relation entre le document actuel et la ressource liée, comme le fait de lier à une feuille de style ou à une ressource externe.

Par ailleurs, l'attribut `href` spécifie l'URL de la ressource liée, indiquant d'où la ressource doit être récupérée.

Le CSS externe est idéal pour les gros projets où tu veux maintenir un style cohérent sur plusieurs pages.

Il favorise la séparation des préoccupations en ayant le HTML qui gère la structure et le CSS qui gère la mise en forme, ce qui rend le code plus maintenable et évolutif.

Comprendre quand utiliser chaque type de CSS est essentiel pour un développement web efficace et performant.

Dans la plupart des cas, le CSS externe devrait être ton approche par défaut, surtout pour les projets plus gros et plus complexes.
