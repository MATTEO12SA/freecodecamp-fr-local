# --interactive--

Quand tu travailles avec le CSS, tu rencontres souvent trois types différents de comportements d'affichage pour les éléments : `inline`, `block` et `inline-block`.

Chacune de ces propriétés affecte la façon dont les éléments sont positionnés et la façon dont ils interagissent avec d'autres éléments sur la page.

Dans cette leçon, on va se concentrer sur la façon dont la propriété `inline-block` fonctionne et en quoi elle diffère à la fois des éléments inline et block-level.

Les éléments block-level se comportent comme de grands conteneurs ou « blocs » qui prennent toute la largeur de leur conteneur parent. Ils commencent toujours sur une nouvelle ligne, et leur hauteur et largeur peuvent être ajustées.

Les éléments inline ne prennent que l'espace dont ils ont besoin. Ils s'écoulent dans le contenu environnant et ne sautent pas à une nouvelle ligne.

La propriété `inline-block` est un hybride de ces deux comportements. Comme les éléments inline, les éléments `inline-block` restent dans le flux du texte sans commencer sur une nouvelle ligne.

Cependant, contrairement aux éléments inline, tu peux ajuster la largeur et la hauteur d'un élément `inline-block`, comme tu le ferais avec les éléments block-level.

En bref, la différence clé entre `inline` et `inline-block` est que les éléments `inline` ne peuvent pas avoir leur taille contrôlée, tandis que les éléments `inline-block` permettent un contrôle complet sur les dimensions tout en restant inline avec les autres contenus.

Regardons un exemple.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Inline-Block</span>
  <span class="inline-block-element element2">Inline-Block</span>
</div>
```

```css
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

Dans l'exemple ci-dessus, on a un `div` avec une classe `container`. À l'intérieur de cet élément `div`, on a deux éléments `span`.

Chacun des éléments span est défini à `display:inline-block` et a aussi une largeur et une hauteur définies.

Les éléments inline-block apparaîtront côte à côte parce qu'ils se comportent comme des éléments inline, mais ils ont aussi une largeur et une hauteur spécifiées, ce qui leur donne des propriétés de type block.

Mais, si tu enlèves la propriété `display: inline-block`, ni la `height` ni la `width` ne seront appliquées même si tu les as définies clairement.

Voici le CSS révisé :

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<div class="container">
  <span class="inline-block-element element1">Span element</span>
  <span class="inline-block-element element2">Span element</span>
</div>
```

```css
.inline-block-element {
  width: 150px;
  height: 100px;
}

.element1 {
  background-color: lightblue;
}

.element2 {
  background-color: lightgreen;
}
```

:::

Dans ce code, on a enlevé la propriété `display: inline-block;` mais gardé tout le reste intact. Ici, les éléments `span` reviennent à leur comportement par défaut en tant qu'éléments inline.

Par conséquent, la largeur et la hauteur spécifiées sont ignorées, et les éléments ne prennent que l'espace nécessaire pour leur contenu.

Comprendre comment fonctionne `inline-block` est utile parce que tu peux l'utiliser pour créer des mises en page qui demandent à la fois un contrôle d'alignement et de dimension dans un flux de texte continu.
