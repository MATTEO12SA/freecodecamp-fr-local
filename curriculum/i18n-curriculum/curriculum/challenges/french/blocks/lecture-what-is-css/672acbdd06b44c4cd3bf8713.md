# --interactive--

Quand tu commences à travailler avec le HTML et le CSS, tu remarqueras que certains styles sont appliqués à tes pages web avant même que tu n'écrives du CSS. Ces styles sont appelés « styles par défaut du navigateur » ou « styles user-agent ».

Ces styles par défaut du navigateur fournissent une mise en forme de base pour s'assurer que les éléments HTML sont affichés de manière lisible dans tous les navigateurs.

Cependant, ces styles peuvent varier légèrement d'un navigateur à l'autre, c'est pourquoi il est important de les comprendre quand on conçoit une apparence cohérente pour ton site web.

Regardons quelques styles par défaut du navigateur courants appliqués à divers éléments HTML.

Les titres sont parmi les éléments HTML les plus couramment utilisés et sont stylisés par défaut avec des tailles et des graisses variables.

Par exemple, `h1` (le titre de plus haut niveau) est généralement en gras et plus grand en taille de police par rapport aux titres de niveau inférieur comme `h2`, `h3`, etc.

Ces balises de titre aident à définir la hiérarchie et la structure de ton contenu.

L'exemple suivant montre les six éléments de titre, de `h1` à `h6`.

:::interactive_editor

```html
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```

:::

Un autre élément avec des styles par défaut est l'élément `hr`, qui crée une règle horizontale souvent utilisée pour séparer visuellement des sections de contenu. Les navigateurs appliquent généralement un style de ligne simple à cet élément.

Pour mieux le comprendre, regardons cet exemple de code :

:::interactive_editor

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<hr>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

:::

La règle horizontale apparaît comme une fine ligne avec un espacement au-dessus et en dessous du texte pour distinguer les sections de contenu.

Ensuite, nous allons regarder l'élément `blockquote`.

Les blockquotes sont utilisés pour indiquer une section de texte qui est une citation d'une autre source. Les navigateurs ajoutent généralement une indentation et parfois mettent le texte en italique.

L'indentation aide à distinguer les blockquotes du reste du texte, ce qui rend clair que ce contenu est cité d'une autre source.

Regardons cet exemple de code :

:::interactive_editor

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<blockquote>I think, therefore I am. (Rene Descartes)</blockquote>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

:::

Dans l'exemple ci-dessus, l'élément `blockquote` produira une indentation sur le texte, appliquée par les styles par défaut du navigateur.

Un autre élément avec des styles par défaut est l'élément d'ancrage (`<a>`), qui est stylisé avec une couleur bleue par défaut et un soulignement pour le rendre reconnaissable en tant que lien.

Regardons l'exemple HTML suivant :

:::interactive_editor

```html
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<a href="https://freecodecamp.org/">Visit the freeCodeCamp learn page</a>
<p>Paragraph 3</p>
<p>Paragraph 4</p>
```

:::

Le code ci-dessus a quatre éléments paragraphe avec un élément d'ancrage au milieu. L'élément d'ancrage est stylisé en bleu avec un soulignement pour indiquer aux utilisateurs d'aller sur la page d'apprentissage de freeCodeCamp.

Enfin, nous allons regarder les éléments de liste non ordonnée et ordonnée.

Les navigateurs ajoutent une mise en forme de base aux listes, comme une indentation et des puces ou des nombres, selon que tu utilises une liste non ordonnée (`ul`) ou une liste ordonnée (`ol`).

Regardons un exemple :

:::interactive_editor

```html
<p>Sample Paragraph</p>
<ol>
  <li>item 1</li>
  <li>item 2</li>
  <li>item 3</li>
</ol>
<ul>
  <li>item</li>
  <li>another item</li>
  <li>yet another item</li>
</ul>
<p>Ending Paragraph</p>
```

:::

Dans l'exemple de code ci-dessus, on utilise une liste non ordonnée et une liste ordonnée. Ces deux listes seront légèrement indentées vers la droite par défaut.

Pour tous ces exemples, tu as vu comment le navigateur a appliqué un espacement par défaut, des tailles de police variables et des graisses à ces éléments HTML.

Dans les leçons suivantes, tu apprendras comment réinitialiser certains de ces styles par défaut en utilisant un reset CSS.
