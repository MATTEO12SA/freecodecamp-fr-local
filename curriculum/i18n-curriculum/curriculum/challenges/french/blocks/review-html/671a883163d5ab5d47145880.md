## Les bases du HTML

- **Rôle du HTML** : HTML (Hypertext Markup Language) est le fondement de la structure du web, il définit les éléments d'une page web.
- **Éléments HTML** : utilisés pour représenter le contenu sur la page. La plupart d'entre eux sont formés d'une balise ouvrante et d'une balise fermante (ex. `<h1></h1>`, `<p></p>`).
- **Structure HTML** : le HTML est composé d'un `head` et d'un `body`, où les métadonnées, les styles et le contenu sont structurés.
- **Éléments HTML courants** : titres (`<h1>` - `<h6>`), paragraphes (`<p>`) et conteneurs div (`<div>`).
- **Éléments `div`** : l'élément `div` est un élément HTML générique qui n'a aucun sens sémantique. Il est utilisé comme conteneur générique pour contenir d'autres éléments HTML.
- **Éléments vides** : n'ont pas de balise fermante (ex. `<img>`).
- **Attributs** : ajoutent des métadonnées et du comportement aux éléments.

## Identifiants et regroupement

- **IDs** : identifiants uniques d'éléments.
- **Classes** : regroupement d'éléments pour le style et le comportement.

## Caractères spéciaux et liens

- **Entités HTML** : utilisation de caractères spéciaux comme `&amp;` et `&lt;`.
- **Élément `link`** : pour lier des feuilles de style externes.
- **Élément `script`** : pour intégrer des fichiers JavaScript externes.

## Boilerplate et encodage

- **Boilerplate HTML** : structure de base d'une page web, qui inclut les éléments `DOCTYPE`, `html`, `head` et `body`. Il doit servir de point de départ pour un document HTML.
- **Encodage de caractères UTF-8** : garantit un affichage universel des caractères.

## SEO et partage social

- **Balises meta (`description`)** : fournissent une courte description de la page web et ont un impact sur le SEO.
- **Balises Open Graph** : améliorent le partage sur les réseaux sociaux.

## Éléments médias et optimisation

- **Éléments remplacés** : contenu embarqué (ex. images, iframes).
- **Optimisation des médias** : techniques pour améliorer les performances des médias.
- **Formats d'image et licences** : comprendre les droits d'usage et les types.
- **SVG** : graphiques vectoriels redimensionnables pour des visuels nets.

## Intégration multimédia

- **Éléments audio et vidéo HTML** : intégration multimédia.
- **Intégration avec `<iframe>`** : intégration de contenu vidéo externe.

## Chemins et comportement des liens

- **Types d'attribut target** : contrôle du comportement des liens.
- **Chemins absolus vs relatifs** : navigation entre les répertoires.
- **Syntaxe des chemins** : comprendre `/`, `./`, `../` pour la navigation des fichiers.
- **États des liens** : gestion des différentes interactions de lien (hover, active).
- **Liens internes** : faire un lien vers une autre section de la page en utilisant `href="#id"` sur un élément `a` et en donnant à l'élément de destination le même `id`.

## Importance du HTML sémantique

- **Hiérarchie structurelle des éléments de titre** : il est important d'utiliser le bon élément de titre pour maintenir la hiérarchie structurelle du contenu. L'élément `h1` est le niveau de titre le plus élevé et l'élément `h6` est le niveau de titre le plus bas.
- **Éléments HTML présentationnels** : éléments qui définissent l'apparence du contenu, par exemple les éléments dépréciés `center`, `big` et `font`.
- **Éléments HTML sémantiques** : ces éléments donnent du sens à la structure du contenu. Des exemples incluent :
  - `<header>` : représente du contenu d'introduction.
  - `<nav>` : contient des liens de navigation.
  - `<article>` : représente du contenu autonome.
  - `<aside>` : utilisé pour les barres latérales ou le contenu lié.
  - `<section>` : regroupe du contenu lié dans un document.
  - `<footer>` : définit le pied de page d'une section ou d'un document.

## Éléments HTML sémantiques

- **Élément emphase (`em`)** : marque un texte qui a une emphase d'insistance.
- **Élément texte idiomatique (`i`)** : utilisé pour mettre en avant une voix ou un ton différent, des expressions idiomatiques d'une autre langue, des termes techniques et des pensées.
- **Élément d'importance forte (`strong`)** : marque un texte qui a une grande importance.
- **Élément « attirer l'attention » (`b`)** : utilisé pour attirer l'attention sur un texte qui n'est pas important pour le sens du contenu.
- **Élément liste de description (`dl`)** : utilisé pour représenter une liste de groupes terme-description.
- **Élément terme de description (`dt`)** : utilisé pour représenter le terme qui est défini.
- **Élément détails de description (`dd`)** : utilisé pour représenter la description du terme.
- **Élément citation en bloc (`blockquote`)** : utilisé pour représenter une section citée d'une autre source.
- **Élément citation en ligne (`q`)** : utilisé pour représenter une courte citation en ligne.
- **Élément abréviation (`abbr`)** : utilisé pour représenter une abréviation ou un acronyme.
- **Élément adresse de contact (`address`)** : utilisé pour représenter les coordonnées de contact. 
- **Élément (date/heure) `time`** : utilisé pour représenter une date et/ou une heure.
- **Élément exposant (`sup`)** : utilisé pour représenter du texte en exposant.
- **Élément indice (`sub`)** : utilisé pour représenter du texte en indice.
- **Élément code en ligne (`code`)** : utilisé pour représenter un fragment de code informatique.
- **Élément annotation non articulée (`u`)** : utilisé pour représenter une portion de texte en ligne qui doit être rendue d'une façon qui indique qu'elle a une annotation non textuelle.
- **Élément annotation ruby (`ruby`)** : utilisé pour représenter le texte d'une annotation ruby.
- **Élément barré (`s`)** : utilisé pour représenter du contenu qui n'est plus exact ou pertinent.

## Éléments et attributs de formulaire HTML

### Formulaires

- **Élément `form`** : utilisé pour créer un formulaire HTML pour la saisie utilisateur.
- **Types d'input courants** : 
  - `text`, `email`, `password`, `radio`, `checkbox`, `number`, `date`.
- **Attribut `action`** : utilisé pour préciser l'URL où les données du formulaire doivent être envoyées.
- **Attribut `method`** : utilisé pour préciser la méthode HTTP à utiliser pour envoyer les données du formulaire. Les méthodes les plus courantes sont `GET` et `POST`.

```html
<form method="value-goes-here" action="url-goes-here">
  <!-- inputs go inside here -->
</form>
```

- **Élément `input`** : utilisé pour créer un champ de saisie pour l'utilisateur.
- **Attribut `type`** : utilisé pour préciser le type de champ de saisie, ex. `text`, `email`, `number`, `radio`, `checkbox`.
- **Attribut `placeholder`** : utilisé pour afficher un indice à l'utilisateur pour lui montrer ce qu'il faut entrer dans le champ.
- **Attribut `value`** : utilisé pour préciser la valeur de l'input. Si l'input est de type `button`, l'attribut `value` peut servir à définir le texte du bouton.
- **Attribut `name`** : utilisé pour assigner un nom à un champ de saisie, qui sert de clé quand les données du formulaire sont soumises. Pour les boutons radio, leur donner le même `name` les groupe ensemble, pour qu'une seule option du groupe puisse être sélectionnée à la fois.
- **Attribut `size`** : utilisé pour définir le nombre de caractères qui doivent être visibles pendant que l'utilisateur tape dans le champ.
- **Attribut `min`** : peut être utilisé avec des types d'input comme `number` pour préciser la valeur minimum autorisée dans le champ.
- **Attribut `max`** : peut être utilisé avec des types d'input comme `number` pour préciser la valeur maximum autorisée dans le champ.
- **Attribut `minlength`** : utilisé pour préciser le nombre minimum de caractères requis dans un champ de saisie.
- **Attribut `maxlength`** : utilisé pour préciser le nombre maximum de caractères autorisés dans un champ de saisie.
- **Attribut `required`** : utilisé pour préciser qu'un champ de saisie doit être rempli avant d'envoyer le formulaire.
- **Attribut `disabled`** : utilisé pour préciser qu'un champ de saisie doit être désactivé.
- **Attribut `readonly`** : utilisé pour préciser qu'un champ de saisie est en lecture seule.

:::interactive_editor

```html
<!-- Text input -->
<input 
  type="text"
  id="name"
  name="name"
  placeholder="e.g. Quincy Larson"
  size="20"
  minlength="5"
  maxlength="30"
  required
/>

<!-- Number input -->
<input 
  type="number"
  id="quantity"
  name="quantity"
  min="2"
  max="10"
  disabled
/>

<!-- Button -->
<input type="button" value="Show Alert" />
```

:::

- **Élément `label`** : utilisé pour créer un libellé pour un champ de saisie.
- **Attribut `for`** : utilisé pour préciser à quel champ de saisie le libellé est destiné.
- **Association implicite avec un formulaire** : les inputs peuvent être associés aux libellés en encapsulant le champ de saisie dans l'élément `label`.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

- **Association explicite avec un formulaire** : les inputs peuvent être associés aux libellés en utilisant l'attribut `for` sur l'élément `label`.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

- **Élément `button`** : utilisé pour créer un bouton cliquable. Un bouton peut aussi avoir un attribut `type`, qui contrôle le comportement du bouton quand il est activé, ex. `submit`, `reset`, `button`.

:::interactive_editor

```html
<button type="button">Show Form</button>
<button type="submit">Submit Form</button>
<button type="reset">Reset Form</button>
```

:::

- **Élément `fieldset`** : utilisé pour regrouper des inputs liés.
- **Élément `legend`** : utilisé pour ajouter une légende qui décrit le groupe d'inputs.

:::interactive_editor

```html
<!-- Radio group -->
<fieldset>
  <legend>Was this your first time at our hotel?</legend>

  <input id="yes-option" type="radio" name="hotel-stay" value="yes" />
  <label for="yes-option">Yes</label>

  <input id="no-option" type="radio" name="hotel-stay" value="no" />
  <label for="no-option">No</label>
</fieldset>

<!-- Checkbox group -->
<fieldset>
  <legend>
    Why did you choose to stay at our hotel? (Check all that apply)
  </legend>

  <input type="checkbox" id="location" name="location" value="location" />
  <label for="location">Location</label>

  <input type="checkbox" id="price" name="price" value="price" />
  <label for="price">Price</label>
</fieldset>
```

:::

- **État focused** : c'est l'état d'un champ de saisie quand il est sélectionné par l'utilisateur.

## Travailler avec les éléments et attributs des tableaux HTML

- **Élément tableau** : utilisé pour créer un tableau HTML.
- **Élément en-tête de tableau (`thead`)** : utilisé pour regrouper le contenu d'en-tête dans un tableau HTML.
- **Élément ligne de tableau (`tr`)** : utilisé pour créer une ligne dans un tableau HTML.
- **Élément en-tête (`th`)** : utilisé pour créer une cellule d'en-tête dans un tableau HTML.
- **Élément corps de tableau (`tbody`)** : utilisé pour regrouper le contenu du corps dans un tableau HTML.
- **Élément cellule de données (`td`)** : utilisé pour créer une cellule de données dans un tableau HTML.
- **Élément pied de tableau (`tfoot`)** : utilisé pour regrouper le contenu de pied dans un tableau HTML.
- **Élément `caption`** : utilisé pour ajouter un titre à un tableau HTML.
- **Attribut `colspan`** : utilisé pour préciser le nombre de colonnes qu'une cellule de tableau doit couvrir.

:::interactive_editor

```html
<table>
  <caption>Exam Grades</caption>

  <thead>
    <tr>
      <th>Last Name</th>
      <th>First Name</th>
      <th>Grade</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Davis</td>
      <td>Alex</td>
      <td>54</td>
    </tr>

    <tr>
      <td>Doe</td>
      <td>Samantha</td>
      <td>92</td>
    </tr>

    <tr>
      <td>Rodriguez</td>
```

## Outils HTML

- **Validateur HTML** : un outil qui vérifie la syntaxe du code HTML pour s'assurer qu'il est valide.
- **Inspecteur DOM** : un outil qui te permet d'inspecter et de modifier la structure HTML d'une page web.
- **Devtools** : un ensemble d'outils pour développeurs web intégrés directement dans le navigateur qui t'aide à déboguer, profiler et analyser les pages web.

## Introduction à l'accessibilité

- **Web Content Accessibility Guidelines** : Les Web Content Accessibility Guidelines (WCAG) sont un ensemble de recommandations pour rendre le contenu web plus accessible aux personnes en situation de handicap. Les quatre principes des WCAG sont POUR, qui signifie Perceptible, Opérable, Compréhensible et Robuste.

## Technologies d'assistance pour l'accessibilité

- **Lecteurs d'écran** : des logiciels qui lisent à voix haute le contenu d'un écran d'ordinateur. Ils sont utilisés par les personnes aveugles ou malvoyantes pour accéder au web.
- **Claviers à gros caractères ou en braille** : utilisés par les personnes avec des déficiences visuelles pour accéder au web.
- **Loupes d'écran** : des logiciels qui agrandissent le contenu d'un écran d'ordinateur. Ils sont utilisés par les personnes avec une vision faible pour accéder au web.
- **Périphériques de pointage alternatifs** : utilisés par les personnes avec des handicaps moteurs pour accéder au web. Cela inclut des appareils comme les joysticks, les trackballs et les pavés tactiles.
- **Reconnaissance vocale** : utilisée par les personnes avec des handicaps moteurs pour accéder au web. Elle permet aux utilisateurs de contrôler un ordinateur avec leur voix.

## Outils d'audit d'accessibilité

- **Outils d'accessibilité courants** : Google Lighthouse, Wave, IBM Equal Accessibility Checker et axe DevTools sont quelques-uns des outils d'accessibilité courants utilisés pour auditer l'accessibilité d'un site web.

## Bonnes pratiques d'accessibilité

- **Bonne structure des niveaux de titres** : tu devrais utiliser les bons niveaux de titres pour créer une structure logique pour ton contenu. Cela aide les technologies d'assistance à comprendre le contenu de ton site.
- **Accessibilité et tableaux** : quand tu utilises des tableaux, tu devrais utiliser l'élément `th` pour définir les cellules d'en-tête et l'élément `td` pour définir les cellules de données. Cela aide les technologies d'assistance à comprendre la structure du tableau.
- **Importance que les inputs aient un libellé associé** : tu devrais utiliser l'élément `label` pour associer des libellés aux champs de formulaire. Cela aide les technologies d'assistance à comprendre le rôle du champ.
- **Importance d'un bon texte `alt`** : tu devrais utiliser l'attribut `alt` pour fournir une alternative textuelle aux images. Cela aide les technologies d'assistance à comprendre le contenu de l'image.
- **Importance d'un bon texte de lien** : tu devrais utiliser un texte de lien descriptif pour aider les utilisateurs à comprendre le but du lien.
- **Bonnes pratiques pour rendre les contenus audio et vidéo accessibles** : tu devrais fournir des sous-titres et des transcriptions pour les contenus audio et vidéo afin de les rendre accessibles aux personnes avec des déficiences auditives. Tu devrais aussi fournir des descriptions audio pour les contenus vidéo afin de les rendre accessibles aux personnes avec des déficiences visuelles.
- **Attribut `tabindex`** : utilisé pour rendre les éléments focusables et définir l'ordre relatif dans lequel ils doivent être parcourus au clavier. Il est important de ne jamais utiliser l'attribut `tabindex` avec une valeur supérieure à 0. À la place, tu devrais utiliser une valeur de 0 ou -1.
- **Attribut `accesskey`** : utilisé pour définir un raccourci clavier pour un élément. Cela peut aider les utilisateurs avec des handicaps moteurs à naviguer plus facilement sur le site.

## WAI-ARIA, rôles et attributs

- **WAI-ARIA** : signifie Web Accessibility Initiative - Accessible Rich Internet Applications. C'est un ensemble d'attributs qui peuvent être ajoutés aux éléments HTML pour améliorer l'accessibilité. Il fournit des informations supplémentaires aux technologies d'assistance sur le but et la structure du contenu.
- **Rôles ARIA** : un ensemble de rôles prédéfinis qui peuvent être ajoutés aux éléments HTML pour définir leur but. Cela aide les technologies d'assistance à comprendre le contenu du site. Des exemples incluent `role="tab"`, `role="menu"` et `role="alert"`.
- **Attributs `aria-label` et `aria-labelledby`** : ces attributs servent à donner à un élément un nom programmatique (ou accessible), ce qui aide les technologies d'assistance (comme les lecteurs d'écran) à comprendre le rôle de l'élément. Ils sont souvent utilisés quand le libellé visuel d'un élément est une image ou un symbole plutôt que du texte. `aria-label` te permet de définir le nom directement dans l'attribut, alors que `aria-labelledby` te permet de référencer un texte existant sur la page.
- **Attribut `aria-hidden`** : utilisé pour cacher un élément aux technologies d'assistance comme les lecteurs d'écran. Par exemple, cela peut servir à cacher des images décoratives qui ne fournissent aucun contenu significatif.
- **Attribut `aria-describedby`** : utilisé pour fournir des informations supplémentaires sur un élément en l'associant à un autre élément qui contient l'information. Cela aide les technologies d'assistance à comprendre le rôle de l'élément.
