# --interactive--

Les propriétés personnalisées CSS, aussi appelées variables CSS, sont des entités définies par les auteurs CSS qui contiennent des valeurs précises à réutiliser dans tout un document. C'est une fonctionnalité puissante qui permet de créer des feuilles de style plus efficaces, maintenables et flexibles.

La syntaxe pour déclarer une propriété personnalisée est simple. Elle commence par deux tirets (`--`) suivis du nom de la propriété :

```css
:root {
  --main-color: #3498db;
}
```

Dans cet exemple, on déclare une propriété personnalisée nommée `--main-color` avec la valeur `#3498db`. La pseudo-classe `:root` est souvent utilisée pour déclarer des propriétés personnalisées globales, car elle représente le parent de plus haut niveau dans l'arbre DOM.

Pour utiliser une propriété personnalisée, tu emploies la fonction `var()` :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
:root {
  --main-color: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

Cela définit la couleur d'arrière-plan des éléments avec la classe `button` sur la valeur stockée dans `--main-color`.

L'une des fonctionnalités clés des propriétés personnalisées est qu'elles suivent la cascade CSS. Cela signifie que tu peux les redéfinir pour des éléments ou des contextes précis :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="alert">This is an alert message.</div>
```

```css
.alert {
  --main-color: #e74c3c;
  background-color: var(--main-color);
}
```

:::

Dans ce cas, les éléments avec la classe `alert` utiliseront une valeur différente pour `--main-color`, qui remplace la définition globale.

Les propriétés personnalisées prennent aussi en charge les valeurs de fallback. Si une propriété personnalisée n'est pas définie ou est invalide, tu peux fournir une valeur de secours :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="text">This is some text.</div>
```

```css
:root {
  --text-color: green;
}

.text {
  color: var(--text-color, green);
}
```

:::

Ici, si `--text-color` n'est pas définie, `color` prendra par défaut la valeur `green`.

Les propriétés personnalisées sont particulièrement utiles pour créer des designs thématisables. Tu peux définir un ensemble de propriétés pour différents thèmes :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="light-theme">
  <p>This is some light themed text.</p>
</div>
```

```css
:root {
  --bg-color: black;
}

body {
  background-color: var(--bg-color);
}

.light-theme {
  --bg-color: white;
  --text-color: black;
  background-color: var(--bg-color);
  color: var(--text-color);
  padding: 10px;
}

```

:::

Changer de thème devient aussi simple qu'ajouter ou retirer une classe sur l'élément `body`.

Les propriétés personnalisées peuvent aussi être utilisées avec des media queries pour créer des designs responsives :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="card">
  <h2>Responsive Design</h2>
  <p>Resize the window to see the card adapt!</p>
</div>

```

```css
:root {
  --card-width: 90%;
  --card-bg: #f0f0f0;
  --card-padding: 1rem;
  --text-color: #333;
}

/* Tablet screens and up */
@media (min-width: 600px) {
  :root {
    --card-width: 70%;
    --card-bg: #e8f5e9;
    --card-padding: 1.5rem;
  }
}

/* Desktop screens and up */
@media (min-width: 1024px) {
  :root {
    --card-width: 50%;
    --card-bg: #d0f0ff;
    --card-padding: 2rem;
  }
}

body {
  font-family: system-ui, sans-serif;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
  background-color: #fafafa;
}

.card {
  width: var(--card-width);
  background-color: var(--card-bg);
  padding: var(--card-padding);
  color: var(--text-color);
  border-radius: 8px;
  box-shadow: 0 2px 6px rgba(0,0,0,0.1);
  text-align: center;
  transition: all 0.3s ease;
}

```

:::

Cette approche permet de créer des mises en page responsives plus maintenables, car tu peux changer les valeurs à un seul endroit au lieu de les modifier partout dans ta feuille de style. Cela peut réduire fortement les répétitions dans tes feuilles de style, surtout pour les valeurs complexes souvent utilisées.

Les propriétés personnalisées peuvent aussi faire référence à d'autres propriétés personnalisées :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
:root {
  --primary-color: #3498db;
  --button-bg: var(--primary-color);
}

button {
  background-color: var(--button-bg);
}

```

:::

Cela permet de créer des relations entre différents aspects du style, ce qui facilite le maintien de thèmes cohérents sur ton site.

Même si les propriétés personnalisées offrent de nombreux avantages, il est important de tenir compte de leur prise en charge par les navigateurs. Elles sont bien prises en charge dans les navigateurs modernes, mais les navigateurs plus anciens peuvent ne pas les reconnaître. Prévois toujours un plan de fallback quand tu utilises des fonctionnalités récentes.

Pour conclure, les propriétés personnalisées CSS fournissent un moyen puissant de créer des feuilles de style plus dynamiques, flexibles et maintenables. Elles permettent de créer des designs thématisables, de simplifier les mises en page responsives et de manipuler les styles au moment de l'exécution.

À mesure que le développement web évolue, les propriétés personnalisées deviennent un outil de plus en plus important dans la boîte à outils d'un développeur, en offrant de nouvelles possibilités pour créer du CSS adaptable et efficace.
