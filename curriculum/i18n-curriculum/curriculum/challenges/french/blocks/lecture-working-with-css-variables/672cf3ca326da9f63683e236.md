# --interactive--

La règle `@property` est une fonctionnalité CSS puissante qui permet aux développeurs de définir des propriétés personnalisées avec un contrôle plus précis sur leur comportement, notamment leur animation et leurs valeurs initiales.

Cette règle fournit un moyen d'améliorer les fonctionnalités des propriétés personnalisées CSS et offre plus de flexibilité dans leur utilisation.

La syntaxe de base de la règle `@property` est la suivante :

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

`--property-name` est le nom de la propriété personnalisée que tu définis. Comme toutes les propriétés personnalisées, elle doit commencer par deux tirets.

`syntax` définit le type de la propriété, qui peut être par exemple `<color>`, `<length>`, `<number>`, `<percentage>`, ou des types plus complexes.

`inherits` précise si la propriété doit hériter sa valeur de son élément parent.

`initial-value` définit la valeur par défaut de la propriété.

Voici un exemple pratique d'utilisation de la règle `@property` :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

.button {
  background-color: var(--main-color);
}
```

:::

Dans cet exemple, on définit la propriété personnalisée `--main-color` comme une valeur de type `color`, on indique qu'elle n'hérite pas, et on lui donne la valeur initiale `#3498db`.

L'un des grands avantages de la règle `@property` est qu'elle permet d'animer des propriétés personnalisées, ce qui n'était pas possible auparavant :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="gradient-box"></div>
```

```css
@property --gradient-angle {
  syntax: '<angle>';
  inherits: false;
  initial-value: 0deg;
}

.gradient-box {
  width: 100px;
  height: 100px;
  background: linear-gradient(var(--gradient-angle), red, blue);
  transition: --gradient-angle 0.5s;
}

.gradient-box:hover {
  --gradient-angle: 90deg;
}
```

:::

Le code ci-dessus crée un dégradé qui s'anime fluidement quand l'élément est survolé, ce qui n'était pas réalisable avec les propriétés personnalisées standard.

Parlons maintenant de la façon dont la règle `@property` fonctionne avec les fallbacks.

Les fallbacks sont essentiels en CSS pour garantir que les styles se dégradent proprement dans les navigateurs qui ne prennent pas en charge certaines fonctionnalités. Avec `@property`, les fallbacks fonctionnent à deux niveaux : pour la règle elle-même et pour l'utilisation de la propriété personnalisée.

Pour les navigateurs qui ne prennent pas en charge la règle `@property`, tu peux fournir un fallback en déclarant la propriété personnalisée de façon traditionnelle :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
```

```css
:root {
  --main-color: #3498db;
}

@property --main-color {
  syntax: '<color>';
  inherits: false;
  initial-value: #3498db;
}

body {
  background-color: var(--main-color);
}
```

:::

Dans ce cas, les navigateurs qui prennent en charge `@property` utiliseront la version plus strictement définie, tandis que les autres se rabattront sur la déclaration standard de propriété personnalisée.

Quand tu utilises la propriété personnalisée, tu peux fournir une valeur de fallback avec la fonction `var()`, comme tu le ferais avec des propriétés personnalisées standard :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button class="button">Click Me</button>
```

```css
.button {
  background-color: var(--main-color, #3498db);
}
```

:::

Cela garantit que même si la propriété `--main-color` n'est pas définie ou est invalide, le bouton aura quand même une couleur d'arrière-plan.

La règle `@property` permet aussi des scénarios de fallback plus complexes. Par exemple, tu peux l'utiliser pour fournir des fallbacks typés :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="box">Colored box</div>
```

```css
@property --padding {
  syntax: '<length>';
  inherits: false;
  initial-value: 10px;
}

.box {
  width: 100px;
  height: 100px;
  background-color: darkred;
  color: white;
  padding: var(--padding);
}
```

:::

Dans ce cas, si quelqu'un essaie de définir `--padding` avec une valeur invalide, comme une couleur, le navigateur se rabattra sur la valeur initiale `10px`, ce qui préserve la sécurité de type.

La règle CSS `@property` est considérée comme largement disponible dans tous les grands moteurs de navigateurs modernes (comme Chrome, Edge, Opera, Firefox et Safari) depuis début 2024. Si tu veux prendre en charge des navigateurs plus anciens, tu dois fournir du CSS de fallback comme expliqué ci-dessus.

Pour conclure, la règle `@property` représente une avancée importante dans la façon de travailler avec les propriétés personnalisées en CSS. Elle offre plus de contrôle et ouvre de nouvelles possibilités, comme l'animation des propriétés personnalisées. Cependant, il est important de l'utiliser avec discernement et de toujours fournir des fallbacks adaptés pour garantir une bonne expérience sur tous les navigateurs. Comme avec toute technologie web récente, l'idée est d'améliorer l'expérience des navigateurs qui la prennent en charge tout en garantissant les fonctionnalités de base pour ceux qui ne la prennent pas en charge.
