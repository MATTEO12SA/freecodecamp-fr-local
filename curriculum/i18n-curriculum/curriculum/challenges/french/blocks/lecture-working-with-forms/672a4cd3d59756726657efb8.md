# --interactive--

L'élément `button` sert à effectuer une action particulière quand il est activé. Voici un exemple d'élément `button` avec le texte de bouton `Start Game`.

```html
<button>Start Game</button>
```

D'autres exemples d'utilisation de l'élément `button` incluent la soumission d'un formulaire, l'affichage d'une modale ou l'ouverture et la fermeture d'un menu latéral. L'élément `button` possède un attribut `type` qui contrôle le comportement du bouton quand il est activé. La première valeur possible pour l'attribut `type` serait le type `button`. Voici un exemple d'utilisation de l'élément `button` avec le type `button` et le texte `Show Alert` :

```html
<button type="button">Show Alert</button>
```

Par défaut, le bouton ne fera rien quand il sera activé. Cependant, tu peux ajouter du code JavaScript pour le rendre interactif, par exemple en affichant une alerte dans ce cas.

Clique sur le bouton `Show Alert` pour voir une alerte apparaître à l'écran. Pour interagir avec l'exemple, tu dois activer l'éditeur interactif.

**NOTE** : cet exemple interactif utilise JavaScript, mais tu n'as pas besoin de comprendre le code JavaScript pour l'instant. Tu en apprendras plus sur JavaScript dans de futurs modules.

:::interactive_editor

```html
<button type="button">Show Alert</button>
<script src="index.js"></script>
```

```js
const btn = document.querySelector("button");
btn.addEventListener("click", () => alert("You clicked on the alert button"));
```

:::

Une autre valeur possible pour l'attribut `type` est la valeur `submit`. Voici un exemple d'utilisation d'un élément `button` avec le type `submit`.

:::interactive_editor

```html
<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="submit">Submit form</button>
</form>
```

:::

Dans cet élément `form`, il y a un élément `label` et un élément `input` pour l'adresse e-mail de l'utilisateur. Quand l'utilisateur clique sur le bouton de soumission, ses données sont envoyées au serveur et traitées. La troisième valeur possible pour l'attribut `type` est la valeur `reset`. Voici un exemple d'élément `form` avec des boutons de réinitialisation et de soumission.

Active l'éditeur interactif et interagis avec l'input e-mail dans la fenêtre d'aperçu en fournissant une fausse adresse e-mail. Clique ensuite sur le bouton de réinitialisation pour voir ton e-mail disparaître du champ.

:::interactive_editor

```html
<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="reset">Reset form</button>
  <button type="submit">Submit form</button>
</form>
```

:::

Dans cet exemple modifié, un élément `label` et un élément `input` sont utilisés pour collecter l'adresse e-mail de l'utilisateur. Quand l'utilisateur clique sur le bouton de réinitialisation, toutes ses données saisies sont effacées. Il est important de noter que les boutons de réinitialisation ne sont généralement pas une bonne idée, car ils peuvent amener les utilisateurs à effacer accidentellement leurs données. De plus, plusieurs boutons dans ton formulaire peuvent encombrer l'interface utilisateur.

Une autre façon de créer des boutons en HTML consiste à utiliser l'élément `input`. L'élément `input` possède aussi un attribut `type` avec les valeurs possibles `submit`, `reset` et `button`. Voici un exemple d'utilisation de l'élément `input` avec le `type` défini sur `button` :

:::interactive_editor

```html
<input class="start-btn" type="button" value="Start Game" />
<script src="index.js"></script>
```

```js
document.addEventListener("DOMContentLoaded", () => {
  const btn = document.querySelector(".start-btn");
  btn.addEventListener("click", () => {
    const paraEl = document.createElement("p");
    const bodyEl = document.querySelector("body");

    bodyEl.appendChild(paraEl);
    paraEl.textContent = "The game has started!!!";
  });
});
```

:::

L'attribut `value` sert à afficher le texte du bouton. Alors, quelle est la différence entre les éléments `input` et `button` ? Les éléments `input` sont des éléments vides, ce qui signifie qu'ils ne peuvent pas avoir de noeuds enfants, comme du texte, et qu'ils ne peuvent avoir qu'une balise de début. En revanche, l'élément `button` offre plus de flexibilité, car tu peux y imbriquer du texte, des images et des icônes.
