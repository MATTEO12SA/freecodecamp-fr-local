# --interactive--

L'élément `form` en HTML sert à collecter des informations utilisateur, comme des noms et des adresses e-mail. Voici un exemple d'élément `form` :

```html
<form action="url-goes-here">
  <!-- input elements go here -->
</form>
```

L'attribut `action` indique où les données du formulaire seront envoyées lors de la soumission. Pour collecter des informations précises, comme des noms et des adresses e-mail, tu utiliserais l'élément `input`. Voici un exemple d'utilisation d'un élément `input`.

Active l'éditeur interactif et interagis avec l'élément `input` dans la fenêtre d'aperçu en tapant ton nom dans le champ.

:::interactive_editor

```html
<form action="">
  <input type="text" />
</form>
```

:::

Les éléments `input` sont des éléments vides et n'ont pas de balise fermante. L'attribut `type` définit le type de données attendu de l'utilisateur. Dans ce cas, les données seraient du texte brut. Pour ajouter un libellé à l'input, tu utiliserais un élément `label`. Voici un exemple d'utilisation d'un élément `label` avec le texte `Full Name:`.

Clique sur le texte `Full Name:` dans la fenêtre d'aperçu et tu verras l'input recevoir le focus. Pour interagir avec l'exemple, tu dois activer l'éditeur interactif.

:::interactive_editor

```html
<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
```

:::

En imbriquant un `input` dans un élément `label`, tu crées une association implicite entre le `label` et le champ `input`. Le terme "implicite" désigne quelque chose qui est compris ou déduit sans avoir besoin d'être explicitement indiqué ou défini avec des attributs ou éléments supplémentaires. Pour associer explicitement un `label` à un `input`, tu peux utiliser l'attribut `for`. Voici un exemple d'utilisation de l'attribut `for` pour un label d'adresse e-mail.

Interagis avec l'élément `input` dans la fenêtre d'aperçu en tapant une fausse adresse e-mail comme `jane@example.com`. Pour interagir avec l'exemple, tu dois activer l'éditeur interactif.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" />
</form>
```

:::

Quand tu utilises une association explicite, les valeurs de l'attribut `for` et de l'attribut `id` doivent être identiques. Dans ce cas, les deux valeurs sont définies sur `email`. Le type `email` dans l'input fournit une validation de base pour les adresses e-mail correctement formatées. Si tu veux afficher des indications supplémentaires aux utilisateurs sur l'entrée attendue, tu peux utiliser l'attribut `placeholder`. Voici un exemple avec l'attribut `placeholder` dans l'input e-mail.

Active l'éditeur interactif, clique sur l'input e-mail et commence à taper une adresse e-mail : tu verras le texte du placeholder disparaître.

:::interactive_editor

```html
<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>
```

:::

Pour le texte du placeholder, tu dois fournir un court texte utile qui montre le format et le type de données attendues de tes utilisateurs. Dans ce cas, le texte du placeholder, `example@email.com`, indique à l'utilisateur qu'il doit saisir une adresse e-mail correctement formatée.
