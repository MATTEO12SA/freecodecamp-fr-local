# --interactive--

Quand un utilisateur remplit un formulaire sur ton site web, il est important qu'il remplisse toutes les informations nécessaires au bon format. Les contrôles de formulaire HTML, comme les inputs, ont beaucoup de validations intégrées que tu peux utiliser pour détecter les données invalides. Cela aide à s'assurer que l'utilisateur corrige ces erreurs avant que les informations soient envoyées et traitées par le serveur.

Le terme "côté client" désigne tout ce qui se passe sur l'ordinateur ou l'appareil de l'utilisateur, comme la partie d'un site web ou d'une application avec laquelle tu interagis directement. Cela inclut la mise en page, le design et toutes les fonctionnalités interactives.

Le terme "côté serveur" désigne tout ce qui se passe sur le serveur, l'ordinateur ou le système qui héberge le site web ou l'application. Cela inclut le traitement des données, l'exécution des applications et la gestion des requêtes venant de l'appareil de l'utilisateur.

Même si la validation côté client est importante, tu as aussi besoin d'une validation côté serveur pour plus de sécurité. Les utilisateurs malveillants peuvent contourner les vérifications côté client, donc des mesures solides côté serveur sont essentielles. Tu en apprendras plus à ce sujet dans un module ultérieur. Pour l'instant, voyons quelques exemples de validation de formulaire côté client.

Un exemple courant de validation de formulaire intégrée consiste à utiliser l'attribut `required` dans les inputs. L'attribut `required` indique que l'utilisateur doit remplir cette partie du formulaire avant qu'il soit soumis. Voici un exemple d'utilisation de l'attribut `required` dans un input e-mail.

Clique sur le bouton `Submit Form` sans fournir d'adresse e-mail et tu verras apparaître un message te demandant de remplir le champ.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

Chaque navigateur aura ses propres styles pour afficher ce message d'alerte.

Un autre avantage de l'input e-mail est que les inputs e-mail ont une validation de base pour vérifier les adresses e-mail correctement formatées. Par exemple, si tu tapes des mots au hasard et que tu cliques sur soumettre, le navigateur affichera une alerte indiquant qu'un signe `@` manque.

Tape `abc` dans le champ e-mail et clique sur le bouton de soumission. Tu devrais voir un message apparaître parce que ce n'est pas une adresse e-mail valide.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

:::

Il est important de noter que les navigateurs effectuent seulement une validation de base pour les adresses e-mail standard. C'est à toi d'ajouter des couches supplémentaires de validation, ce que tu apprendras dans des modules ultérieurs.

D'autres formes de validation pour les inputs e-mail consistent à utiliser les attributs `minlength` et `maxlength`. Voici un exemple avec cette validation supplémentaire.

Tape `b@m` dans le champ et clique sur le bouton de soumission. Tu verras apparaître un message parce que cela ne respecte pas la longueur minimale requise.

:::interactive_editor

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input
    required
    type="email"
    name="email"
    id="email"
    minlength="4"
    maxlength="64"
  />
  <button type="submit">Submit Form</button>
</form>
```

:::

Les attributs `minlength` et `maxlength` servent à définir la longueur minimale et maximale, en caractères, pour l'input e-mail. Si tu n'atteins pas la longueur minimale ou si tu dépasses la longueur maximale de caractères, le navigateur affichera un message d'alerte.
