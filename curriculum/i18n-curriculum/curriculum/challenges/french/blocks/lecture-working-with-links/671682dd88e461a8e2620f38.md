# --description--

Un chemin est une chaîne de caractères qui indique l'emplacement d'un fichier ou d'un dossier dans un système de fichiers. En développement web, les chemins permettent aux développeurs de lier des ressources comme des images, des feuilles de style, des scripts et d'autres pages web. Il existe des chemins absolus et des chemins relatifs, et les deux sont essentiels pour indiquer l'emplacement de fichiers dans un système de fichiers. Voyons les deux pour que tu puisses choisir lequel utiliser et quand.

Un chemin absolu est un lien complet vers une ressource. Il commence à la racine, inclut tous les autres dossiers, puis enfin le nom du fichier et son extension. Le "dossier racine" désigne le dossier ou répertoire le plus haut dans une hiérarchie.

Si tu lies une ressource sur ta machine locale, utilise un chemin absolu, qui inclut l'emplacement complet du fichier. Voici comment créer un lien vers le fichier `about.html` avec un chemin absolu :

```html
<p>
  Read more on the
  <a
    href="/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html"
    >About Page</a
    >
</p>
```

Cela ressemble à ça parce que l'on part de la racine et que l'on entre dans un dossier appelé `Users`, puis dans un dossier appelé `user`, puis dans un dossier appelé `Desktop`, puis dans un dossier appelé `fCC`, puis dans un dossier appelé `script-code`, puis dans un dossier appelé `absolute-vs-relative-paths`, puis dans un dossier appelé `pages` pour enfin atteindre le fichier `about.html`.

Une URL absolue est une adresse complète utilisée pour accéder à une ressource. Elle inclut le protocole, qui peut être `http`, `https` ou `file`, ainsi que le nom de domaine si la ressource est sur le web. Voici un exemple d'URL absolue qui pointe vers le logo freeCodeCamp :

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

Dans cet exemple, le protocole est `https`, le nom de domaine est `design-style-guide.freecodecamp.org`, et le nom du fichier est `fcc_secondary_small.svg`.

Voici à quoi ressemble l'URL absolue dans la barre d'adresse du navigateur :

```sh
file:///Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/about.html
```

L'URL inclut le protocole `file://`. Elle inclut aussi le chemin, qui ressemble à ceci : `/Users/user/Desktop/fCC/script-code/absolute-vs-relative-paths/pages/`, et qui représente la série de dossiers menant au fichier. Enfin, elle inclut aussi `about.html`, qui est le nom du fichier et son extension.

Un chemin absolu montre l'emplacement complet d'un fichier dans un système de fichiers et est couramment utilisé pour des ressources sur une machine locale. Une URL absolue inclut des informations d'accès, comme le protocole et, pour les ressources web, le nom de domaine, ce qui indique au navigateur comment et où récupérer la ressource.

Voyons maintenant le chemin relatif. Un chemin relatif indique l'emplacement d'un fichier par rapport au dossier du fichier actuel. Il n'inclut pas le protocole ni le nom de domaine, ce qui le rend plus court et plus flexible pour les liens internes d'un même site web. Voici un exemple de lien vers la page `about.html` depuis la page `contact.html`, les deux étant dans le même dossier :

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

Imagine donc que tu es sur la page `contact.html`, et comme la page `about.html` se trouve au même endroit, tu utilises simplement le nom du fichier. C'est un exemple d'utilisation d'un chemin de fichier relatif.

Alors, que dois-tu utiliser et quand : un chemin absolu, une URL absolue ou un chemin relatif ? Voici les règles à suivre :

- Utilise des chemins absolus quand tu veux référencer une ressource depuis un emplacement fixe, comme la racine de ton site ou un dossier connu sur ta machine locale.

- Utilise une URL absolue quand tu crées un lien vers une ressource hébergée sur un site web externe.

- Utilise des chemins relatifs quand tu crées des liens vers des ressources du même site web.

- Utilise des chemins relatifs si tu veux garder ton code plus propre et plus facile à maintenir pendant le développement.

- Utilise des chemins relatifs pendant les tests locaux pour t'assurer que les liens fonctionnent sans connexion internet.
