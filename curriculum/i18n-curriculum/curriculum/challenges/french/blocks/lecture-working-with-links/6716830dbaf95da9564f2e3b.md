# --description--

Tu as peut-être déjà vu des liens comme `/public/logo.png`, `./script.js` ou `../styles.css`. Mais que signifient ces types spéciaux de liens ? On les appelle des chemins de fichiers. Il y a trois syntaxes clés à connaître. La première est le slash, qui peut être un antislash (`\`) ou un slash (`/`) selon ton système d'exploitation. La deuxième est le point simple (`.`). Et enfin, il y a les deux points (`..`).

Le slash est appelé "séparateur de chemin". Il sert à indiquer une séparation dans le texte entre les noms de dossiers ou de fichiers. C'est comme ça que ton ordinateur sait que `naomis-files/` pointe vers un dossier de ce nom, tandis que `naomis/files/` pointe vers un dossier `files` dans le dossier `naomis`.

Un point simple pointe vers le dossier courant, et deux points pointent vers le dossier parent. Tu verras généralement un point simple utilisé pour s'assurer que le chemin est reconnu comme un chemin relatif. Souviens-toi que tu as appris la différence entre chemins relatifs et absolus dans une leçon précédente.

Les deux points, en revanche, sont beaucoup plus courants pour accéder à des fichiers situés en dehors du dossier de travail actuel.

Par exemple, avec cette arborescence de fichiers :

```sh
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

Si ton fichier `public/index.html` devait charger le fichier `favicon.ico`, tu utiliserais un chemin relatif avec un point simple pour accéder au dossier courant : `./favicon.ico`. Si ton fichier `public/index.html` devait charger le fichier `index.css`, tu utiliserais un chemin relatif avec deux points pour remonter d'abord au dossier parent `my-app`, puis aller dans le dossier `src`, et enfin vers ton fichier : `../src/index.css`.
