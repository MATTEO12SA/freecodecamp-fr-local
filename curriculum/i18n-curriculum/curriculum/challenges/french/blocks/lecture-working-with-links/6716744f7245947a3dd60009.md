# --interactive--

Tu as peut-être déjà vu l'attribut `target` sur des éléments d'ancre, ou liens. Cet attribut important indique au navigateur où ouvrir l'URL de l'élément d'ancre.

Active l'éditeur interactif, clique sur le lien et tu seras dirigé vers la page d'accueil de freeCodeCamp dans un nouvel onglet du navigateur.

:::interactive_editor

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

:::

Il existe quatre valeurs importantes possibles pour cet attribut. Note que chaque valeur commence par un underscore.

La première valeur est `_self`, qui est la valeur par défaut. Elle ouvre le lien dans le contexte de navigation actuel. Dans la plupart des cas, ce sera l'onglet ou la fenêtre actuelle.

La deuxième valeur est `_blank`, qui ouvre le lien dans un nouveau contexte de navigation. Généralement, cela ouvre un nouvel onglet. Mais certains utilisateurs peuvent configurer leur navigateur pour ouvrir une nouvelle fenêtre à la place.

La troisième valeur est `_parent`, qui ouvre le lien dans le parent du contexte actuel. Par exemple, si ton site web contient une `iframe`, une valeur `_parent` dans cette `iframe` s'ouvrirait dans l'onglet ou la fenêtre de ton site, pas dans le cadre intégré.

La quatrième valeur est `_top`, qui ouvre le lien dans le contexte de navigation le plus haut, comme "le parent du parent". C'est similaire à `_parent`, mais le lien s'ouvrira toujours dans l'onglet ou la fenêtre complète du navigateur, même avec des cadres intégrés imbriqués.

Il existe une cinquième valeur, appelée `_unfencedTop`, actuellement utilisée pour l'API expérimentale FencedFrame. Au moment de cette leçon, tu n'auras probablement pas encore de raison de l'utiliser.

Choisir la bonne valeur de `target` pour contrôler où tes utilisateurs arrivent est une considération importante quand tu crées un site web.
