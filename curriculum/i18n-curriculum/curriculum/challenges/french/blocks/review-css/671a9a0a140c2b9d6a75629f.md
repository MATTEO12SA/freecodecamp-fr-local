## Bases du CSS

- **Qu'est-ce que CSS ?**: Cascading Style Sheets (CSS) est un langage de balisage utilisé pour appliquer des styles aux éléments HTML. CSS sert à gérer les couleurs, les images d'arrière-plan, les mises en page et plus encore.
- **Anatomie de base d'une règle CSS**: Une règle CSS est composée de deux parties principales : un sélecteur et un bloc de déclarations. Un sélecteur est un motif utilisé en CSS pour identifier et cibler des éléments HTML précis afin de les styliser. Un bloc de déclarations applique un ensemble de styles à un ou plusieurs sélecteurs donnés.
- **Élément `meta name="viewport"`**: Cet élément `meta` donne au navigateur des instructions sur la manière de contrôler les dimensions et la mise à l'échelle de la page sur différents appareils, en particulier les téléphones et les tablettes.
- **Styles par défaut du navigateur**: Chaque élément HTML reçoit des styles par défaut du navigateur. Cela inclut généralement des éléments comme des marges et des espacements internes par défaut.

## CSS inline, interne et externe

- **CSS inline**: Ces styles sont écrits directement dans un élément HTML avec l'attribut `style`. La plupart du temps, tu n'utiliseras pas de CSS inline à cause de la séparation des responsabilités.
- **CSS interne**: Ces styles sont écrits dans les balises `<style>` à l'intérieur de la section `head` d'un document HTML. Cela peut être utile pour créer de courts exemples de code, mais en général tu n'utiliseras pas de CSS interne.
- **CSS externe**: Ces styles sont écrits dans un fichier CSS séparé et liés au document HTML avec l'élément `link` dans la section `head`. Pour la plupart des projets, tu utiliseras un fichier CSS externe plutôt que du CSS interne ou inline.

## Travailler avec les propriétés `width` et `height`

- **Propriété `width`**: Cette propriété indique la largeur d'un élément. Si tu ne définis pas de largeur, la valeur par défaut est `auto`. Cela laisse le navigateur déterminer la largeur de l'élément en fonction de son contenu, de son parent et de son type d'affichage.
- **Propriété `min-width`**: Cette propriété indique la largeur minimale d'un élément.
- **Propriété `max-width`**: Cette propriété indique la largeur maximale d'un élément.
- **Propriété `height`**: Cette propriété indique la hauteur d'un élément. De la même façon, la hauteur vaut `auto` par défaut, ce qui signifie qu'elle s'adapte au contenu qu'elle contient.
- **Propriété `min-height`**: Cette propriété indique la hauteur minimale d'un élément.
- **Propriété `max-height`**: Cette propriété indique la hauteur maximale d'un élément.

## Différents types de combinateurs CSS

- **Combinateur descendant (` `)&nbsp;\*un seul espace entre les sélecteurs**: Ce combinateur sert à cibler les éléments qui sont des descendants d'un élément parent précis.
- **Combinateur enfant (`>`)**: Ce combinateur sert à sélectionner les éléments qui sont des enfants directs d'un élément parent précis.
- **Combinateur de frère adjacent (`+`)**: Ce combinateur sélectionne un élément qui suit immédiatement un élément frère précis.

- **Combinateur de frères suivants (`~`)**: Ce combinateur sélectionne tous les frères d'un élément précis qui viennent après lui.

## Éléments inline, block et inline-block

- **Éléments inline**: Les éléments inline prennent seulement la largeur dont ils ont besoin et ne commencent pas sur une nouvelle ligne. Ces éléments suivent le flux du contenu, ce qui permet au texte et aux autres éléments inline d'apparaître à côté d'eux. Les éléments `span`, `anchor` et `img` sont des exemples courants d'éléments inline.
- **Éléments block**: Les éléments de niveau block prennent par défaut toute la largeur disponible, en s'étendant sur la largeur de leur conteneur. Les éléments `div`, `p` et `section` sont des exemples courants d'éléments de niveau block.
- **Éléments inline-block**: Tu peux définir un élément en `inline-block` avec la propriété `display`. Ces éléments se comportent comme des éléments inline, mais peuvent recevoir une `width` et une `height` comme les éléments de niveau block.

## Margin et padding

- **Propriété `margin`**: Cette propriété sert à appliquer de l'espace à l'extérieur de l'élément, entre la bordure de l'élément et les éléments qui l'entourent.
- **Raccourci `margin`**: Tu peux définir 1 à 4 valeurs pour régler les côtés de la marge. Une valeur s'applique aux quatre côtés ; deux valeurs règlent `top` et `bottom`, puis `right` et `left` ; trois valeurs règlent `top`, l'horizontal (`right` et `left`), puis `bottom` ; quatre valeurs règlent `top`, `right`, `bottom`, `left`.
- **Propriété `padding`**: Cette propriété sert à appliquer de l'espace à l'intérieur de l'élément, entre le contenu et sa bordure.
- **Raccourci `padding`**: Tu peux définir 1 à 4 valeurs pour régler les côtés du padding. Une valeur s'applique aux quatre côtés ; deux valeurs règlent `top` et `bottom`, puis `right` et `left` ; trois valeurs règlent `top`, l'horizontal (`right` et `left`), puis `bottom` ; quatre valeurs règlent `top`, `right`, `bottom`, `left`.

## Spécificité CSS

- **Spécificité du CSS inline**: Le CSS inline a la spécificité la plus élevée, car il est appliqué directement à l'élément. Il remplace tout CSS interne ou externe. La valeur de spécificité des styles inline est (1, 0, 0, 0).
- **Spécificité du CSS interne**: Le CSS interne est défini dans un élément `style` de la section `head` du document HTML. Il a une spécificité plus faible que les styles inline. Le CSS interne et le CSS externe partagent le même niveau de spécificité, déterminé par leurs sélecteurs et non par leur emplacement.
- **Spécificité du CSS externe**: Le CSS externe est lié avec un élément `link` dans la section `head` et écrit dans des fichiers `.css` séparés. Comme le CSS interne, sa spécificité est déterminée par les sélecteurs utilisés. Quand deux règles ont la même spécificité, l'ordre des sources détermine la gagnante : la règle qui apparaît plus tard dans le document prend le dessus. Le CSS externe offre la meilleure maintenabilité pour les projets plus grands.
- **Sélecteur universel (`*`)**: C'est un type spécial de sélecteur CSS qui correspond à n'importe quel élément du document. Il est souvent utilisé pour appliquer un style à tous les éléments de la page, ce qui peut être utile pour réinitialiser ou normaliser les styles entre différents navigateurs. Le sélecteur universel a la valeur de spécificité la plus faible de tous les sélecteurs. Il contribue 0 à toutes les parties de la valeur de spécificité (0, 0, 0, 0).
- **Sélecteurs de type**: Ces sélecteurs ciblent les éléments selon leur nom de balise. Les sélecteurs de type ont une spécificité relativement basse par rapport aux autres sélecteurs. La valeur de spécificité d'un sélecteur de type est (0, 0, 0, 1).
- **Sélecteurs de classe**: Ces sélecteurs sont définis par un point (`.`) suivi du nom de la classe. La valeur de spécificité d'un sélecteur de classe est (0, 0, 1, 0). Cela signifie que les sélecteurs de classe peuvent remplacer les sélecteurs de type, mais peuvent être remplacés par les sélecteurs d'ID et les styles inline.
- **Sélecteurs d'ID**: Les sélecteurs d'ID sont définis par un dièse (`#`) suivi du nom de l'ID. Les sélecteurs d'ID ont une très forte spécificité, plus élevée que celle des sélecteurs de type et de classe, mais plus faible que celle des styles inline. La valeur de spécificité d'un sélecteur d'ID est (0, 1, 0, 0).
- **Mot-clé `!important`**: Utilisé pour donner la priorité la plus élevée à une règle de style, ce qui lui permet de remplacer toute autre déclaration pour une propriété. Quand il est utilisé, il force le navigateur à appliquer le style indiqué, quelle que soit la spécificité des autres sélecteurs. Tu dois utiliser `!important` avec prudence, car il peut rendre ton CSS plus difficile à maintenir et à déboguer.
- **Algorithme de cascade**: Un algorithme utilisé pour décider quelles règles CSS appliquer quand plusieurs styles ciblent le même élément. Il garantit que les styles les plus appropriés sont utilisés, selon un ensemble de règles bien définies.
- **Héritage CSS**: Le processus par lequel les styles sont transmis des éléments parents à leurs enfants. L'héritage te permet de définir des styles à un niveau plus haut dans l'arbre du document et de les appliquer à plusieurs éléments sans les préciser explicitement pour chacun.

## Terminologie du design

- **Layout**: C'est la façon dont les éléments visuels sont disposés sur une page ou un écran pour communiquer un message. Ces éléments peuvent inclure du texte, des images et de l'espace blanc.
- **Alignement**: C'est la façon dont les éléments sont placés les uns par rapport aux autres. Bien utiliser l'alignement aide à rendre le design propre et organisé.
- **Composition**: C'est l'action de disposer des éléments pour créer un design harmonieux. Elle détermine comment les images, le texte et les formes interagissent entre eux et contribuent au design d'une manière artistique.
- **Équilibre**: C'est la façon dont le poids visuel est réparti dans une composition. Les designers cherchent à créer un équilibre avec des arrangements symétriques ou asymétriques.
- **Échelle**: Cela consiste à comparer les dimensions ou la taille d'un élément à celles d'un autre.
- **Hiérarchie**: Cela établit l'ordre d'importance des éléments dans un design. L'idée est de s'assurer que l'information la plus importante est remarquée en premier.
- **Contraste**: C'est le processus qui consiste à créer des distinctions nettes entre les éléments. Tu peux le faire avec des variations de couleur, de taille, de forme, de texture ou de toute autre caractéristique visuelle. Un contraste fort aide aussi à améliorer la lisibilité.
- **Espace blanc (espace négatif)**: C'est l'espace vide dans un design. C'est la zone qui entoure les éléments.
- **UI (User Interface)**: L'UI comprend les éléments visuels et interactifs que les utilisateurs peuvent voir sur leurs écrans, comme les icônes, images, textes, menus, liens et boutons.
- **UX (User Experience)**: L'UX concerne ce que ressentent les utilisateurs lorsqu'ils utilisent un produit ou un service. Une application avec une expérience utilisateur bien conçue est intuitive, facile à utiliser, efficace, accessible et agréable.
- **Brief de design**: C'est un document qui décrit les objectifs, buts et exigences d'un projet. C'est une feuille de route qui guide le processus de design et garantit que le produit final répond aux besoins du client.
- **Design vectoriel**: Cela consiste à créer de l'art numérique avec des formules mathématiques pour définir les lignes, les formes et les couleurs.
- **Prototypage**: Cela désigne le processus de création d'un modèle interactif d'un produit ou d'une interface utilisateur.

## Fondamentaux du design UI

- **Bon contraste entre les couleurs d'arrière-plan et de premier plan**: Il est important de s'assurer que les couleurs d'arrière-plan et de premier plan ont un bon contraste pour rendre le texte lisible. Les Web Content Accessibility Guidelines (WCAG) recommandent un ratio de contraste minimum de 4.5:1 pour le texte normal et de 3:1 pour le grand texte.
- **Bonne hiérarchie visuelle dans le design**: Une hiérarchie visuelle forte peut fournir un chemin clair que l'œil peut suivre, afin que l'information que tu transmets soit consommée dans l'ordre voulu.
- **Images responsives**: Les images responsives sont des images qui s'adaptent à la taille de l'écran sur lequel elles sont affichées. C'est important parce que cela garantit que tes images restent correctes sur tous les appareils, des ordinateurs de bureau aux téléphones.
- **Amélioration progressive**: C'est une approche de design qui garantit que tous les utilisateurs, quel que soit leur navigateur ou appareil, peuvent accéder au contenu et aux fonctionnalités essentiels d'une application.
- **Design centré utilisateur**: C'est une approche qui donne la priorité à l'utilisateur final, de ses besoins à ses préférences et limites. Le but du design centré utilisateur est de concevoir une page web intuitive, efficace à utiliser et agréable pour tes utilisateurs.
- **Recherche utilisateur**: C'est l'étude systématique des personnes qui utilisent ton produit. Le but est de mesurer les besoins, comportements et points de friction des utilisateurs.
- **Test utilisateur**: Cela désigne la pratique qui consiste à recueillir des données auprès des utilisateurs lorsqu'ils interagissent avec ton application.
- **Test A/B**: C'est le processus qui consiste à livrer une nouvelle fonctionnalité à un sous-ensemble choisi aléatoirement de ta base d'utilisateurs. Tu peux ensuite exploiter les données d'analyse pour déterminer si la fonctionnalité est bénéfique.
- **Exigences utilisateur**: Cela désigne les user stories ou la grille que ton application doit suivre. Les exigences utilisateur peuvent être définies par la recherche utilisateur ou par des standards du secteur. Elles peuvent même être définies par les retours des parties prenantes.
- **Divulgation progressive**: C'est un modèle de design qui consiste à montrer seulement aux utilisateurs le contenu pertinent selon leur activité actuelle et à masquer le reste. Cela réduit la charge cognitive et rend l'expérience utilisateur plus intuitive.
- **Inscription différée/lazy registration**: C'est un modèle de design UI qui permet aux utilisateurs de parcourir ton application et d'interagir avec elle sans avoir à s'inscrire.

## Bonnes pratiques de design

- **Mode sombre**: C'est une fonctionnalité spéciale des applications web qui permet de remplacer le jeu de couleurs clair par défaut par un jeu de couleurs sombre. Tu dois utiliser des couleurs désaturées en mode sombre. Les couleurs désaturées sont des couleurs moins intenses, avec un niveau de saturation plus faible.
- **Fil d'Ariane**: C'est une aide à la navigation qui montre à l'utilisateur où il se trouve dans la hiérarchie du site. Il est préférable de placer le fil d'Ariane en haut de la page pour que les utilisateurs le trouvent facilement. Tu dois aussi t'assurer qu'il est assez grand pour être lu facilement, mais pas au point d'occuper trop d'espace sur la page.
- **Composant carte**: Ton composant carte doit avoir un design simple, sans surcharge visuelle ni trop d'informations. Pour les médias, choisis des images et vidéos de haute qualité afin d'améliorer l'expérience utilisateur.
- **Défilement infini**: C'est un modèle de design qui charge plus de contenu à mesure que l'utilisateur fait défiler la page. Tu devrais envisager d'utiliser un bouton de chargement supplémentaire, car il donne à l'utilisateur le contrôle sur le moment où il veut voir plus de contenu. Tu peux aussi ajouter un bouton retour pour que les utilisateurs puissent revenir à la page précédente sans devoir faire défiler toute la page vers le haut.
- **Boîte de dialogue modale**: C'est un type de fenêtre contextuelle qui s'affiche au-dessus du contenu existant de la page. En général, le contenu en arrière-plan reçoit un voile sombre pour aider l'utilisateur à mieux se concentrer sur le contenu de la modale. Il est aussi toujours conseillé de permettre à l'utilisateur de cliquer en dehors de la modale pour la fermer. Quand tu utilises l'élément HTML `dialog`, tu obtiens beaucoup de fonctionnalités et d'avantages d'accessibilité intégrés.
- **Indication de progression pour l'inscription par formulaire**: C'est une façon de montrer aux utilisateurs où ils en sont dans un processus. Elle peut être utilisée dans les formulaires, inscriptions et processus de configuration. Ton design doit être simple, facile à trouver et permettre de revenir aux étapes précédentes.
- **Panier d'achat**: Les paniers sont l'endroit où l'utilisateur voit les articles qu'il a déjà sélectionnés sur une plateforme e-commerce. Tes paniers doivent toujours être visibles pour l'utilisateur, utiliser une icône courante comme un panier, un sac ou un cabas, et avoir un bouton d'appel à l'action clair pour passer au paiement.

## Unités absolues

- **`px` (pixels)**: Cette unité absolue est une unité de mesure de taille fixe en CSS, qui offre un contrôle précis des dimensions. Cela signifie que 1px est toujours égal à 1/96e de pouce.
- **`in` (pouce)**: Cette unité absolue est égale à 96px.
- **`cm` (centimètres)**: Cette unité absolue est égale à 25.2/64 de pouce.
- **`mm` (millimètres)**: Cette unité absolue est égale à 1/10e de centimètre.
- **`q` (quarts de millimètre)**: Cette unité absolue est égale à 1/40e de centimètre.
- **`pc` (picas)**: Cette unité absolue est égale à 1/6e de pouce.
- **`pt` (points)**: Cette unité absolue est égale à 1/72e de pouce.

## Unités relatives

- **Pourcentages**: Ces unités relatives te permettent de définir des tailles, dimensions et autres propriétés comme une proportion de leur élément parent. Par exemple, si tu définis `width: 50%;` sur un élément, il occupera la moitié de la largeur de son conteneur parent.
- **Unité `em`**: Ces unités sont relatives à la taille de police de l'élément. Si tu utilises des `ems` pour la propriété `font-size`, la taille du texte sera relative à la taille de police de l'élément parent.
- **Unité `rem`**: Ces unités sont relatives à la taille de police de l'élément racine, qui est l'élément `html`.
- **Unité `vh`**: `vh` signifie `"viewport height"` et `1vh` est égal à 1% de la hauteur du viewport.
- **Unité `vw`**: `vw` signifie `"viewport width"` et `1vw` est égal à 1% de la largeur du viewport.

## Fonction `calc`

- **Fonction `calc()`**: Avec la fonction `calc()`, tu peux effectuer des calculs directement dans tes feuilles de style afin de déterminer dynamiquement les valeurs des propriétés. Cela signifie que tu peux créer des interfaces utilisateur flexibles et responsives en calculant des dimensions selon la taille du viewport ou d'autres éléments.

## Pseudo-classes d'action utilisateur

- **Définition des pseudo-classes**: Ce sont des mots-clés CSS spéciaux qui te permettent de sélectionner un élément selon son état ou sa position précise.
- **Pseudo-classes d'action utilisateur**: Ce sont des mots-clés spéciaux qui te permettent de modifier l'apparence des éléments selon les interactions de l'utilisateur, améliorant ainsi l'expérience utilisateur globale.
- **Pseudo-classe `:active`**: Cette pseudo-classe te permet de sélectionner l'état actif d'un élément, comme lors d'un clic sur un bouton.
- **Pseudo-classe `:hover`**: Cette pseudo-classe définit l'état de survol d'un élément.
- **Pseudo-classe `:focus`**: Cette pseudo-classe applique des styles lorsqu'un élément reçoit le focus, généralement via la navigation au clavier ou lorsqu'un utilisateur clique dans un champ de formulaire.
- **Pseudo-classe `:focus-within`**: Cette pseudo-classe sert à appliquer des styles à un élément lorsque lui-même ou l'un de ses descendants a le focus.

## Pseudo-classes d'input

- **Pseudo-classes d'input**: Ces pseudo-classes servent à cibler les éléments HTML `input` selon l'état dans lequel ils se trouvent avant et après l'interaction de l'utilisateur.
- **Pseudo-classe `:enabled`**: Cette pseudo-classe sert à cibler les boutons de formulaire ou autres éléments actuellement activés.
- **Pseudo-classe `:disabled`**: Cette pseudo-classe te permet de styliser un élément interactif en mode désactivé.
- **Pseudo-classe `:checked`**: Cette pseudo-classe sert à indiquer à l'utilisateur qu'un élément est coché.
- **Pseudo-classe `:valid`**: Cette pseudo-classe cible les champs de saisie qui respectent les critères de validation.
- **Pseudo-classe `:invalid`**: Cette pseudo-classe cible les champs de saisie qui ne respectent pas les critères de validation.
- **Pseudo-classes `:in-range` et `:out-of-range`**: Ces pseudo-classes appliquent des styles aux éléments selon que leurs valeurs se trouvent à l'intérieur ou à l'extérieur des contraintes de plage définies.
- **Pseudo-classe `:required`**: Cette pseudo-classe cible les éléments `input` qui possèdent l'attribut `required`. Elle signale à l'utilisateur qu'il doit remplir le champ pour envoyer le formulaire.
- **Pseudo-classe `:optional`**: Cette pseudo-classe applique des styles aux éléments input qui ne sont pas obligatoires et peuvent rester vides.
- **Pseudo-classe `:autofill`**: Cette pseudo-classe applique des styles aux champs de saisie que le navigateur remplit automatiquement avec des données enregistrées.

## Pseudo-classes de localisation

- **Pseudo-classes de localisation**: Ces pseudo-classes servent à styliser les liens et les éléments ciblés dans le document actuel.
- **Pseudo-classe `:any-link`**: Cette pseudo-classe combine les pseudo-classes `:link` et `:visited`. Elle correspond donc à n'importe quel élément d'ancrage avec un attribut href, qu'il ait été visité ou non.
- **Pseudo-classe `:link`**: Cette pseudo-classe te permet de cibler tous les liens non visités d'une page web. Tu peux l'utiliser pour styliser les liens différemment avant que l'utilisateur clique dessus.
- **Pseudo-classe `:local-link`**: Cette pseudo-classe cible les liens qui pointent vers le même document. Elle peut être utile quand tu veux distinguer les liens internes des liens externes.
- **Pseudo-classe `:visited`**: Cette pseudo-classe cible un lien que l'utilisateur a visité.
- **Pseudo-classe `:target`**: Cette pseudo-classe sert à appliquer des styles à un élément qui est la cible d'un fragment d'URL.

## Pseudo-classes structurelles

- **Pseudo-classes structurelles**: Ces pseudo-classes te permettent de cibler et de styliser des éléments selon leur position dans l'arbre du document.
- **Pseudo-classe `:root`**: Cette pseudo-classe correspond généralement à l'élément racine `html`. Elle t'aide à cibler le niveau le plus haut du document afin d'appliquer un style commun à tout le document.
- **Pseudo-classe `:empty`**: Les éléments vides, c'est-à-dire les éléments sans enfants autres que des espaces blancs, font aussi partie de l'arbre du document. C'est pourquoi il existe une pseudo-classe `:empty` pour cibler les éléments vides.
- **Pseudo-classe `:nth-child(n)`**: Cette pseudo-classe te permet de sélectionner des éléments selon leur position dans un parent.
- **Pseudo-classe `:nth-last-child(n)`**: Cette pseudo-classe te permet de sélectionner des éléments en comptant depuis la fin.
- **Pseudo-classe `:first-child`**: Cette pseudo-classe sélectionne le premier élément dans un élément parent ou dans le document.
- **Pseudo-classe `:last-child`**: Cette pseudo-classe sélectionne le dernier élément dans un élément parent ou dans le document.
- **Pseudo-classe `:only-child`**: Cette pseudo-classe sélectionne l'unique élément dans un élément parent ou dans le document.
- **Pseudo-classe `:first-of-type`**: Cette pseudo-classe sélectionne la première occurrence d'un type d'élément précis dans son parent.
- **Pseudo-classe `:last-of-type`**: Cette pseudo-classe sélectionne la dernière occurrence d'un type d'élément précis dans son parent.
- **Pseudo-classe `:nth-of-type(n)`**: Cette pseudo-classe te permet de sélectionner un élément précis dans son parent selon sa position parmi ses frères du même type.
- **Pseudo-classe `:only-of-type`**: Cette pseudo-classe sélectionne un élément s'il est le seul de son type dans son parent.

## Pseudo-classes fonctionnelles

- **Pseudo-classes fonctionnelles**: Les pseudo-classes fonctionnelles te permettent de sélectionner des éléments selon des conditions ou relations plus complexes. Contrairement aux pseudo-classes classiques qui ciblent les éléments selon un état (par exemple `:hover`, `:focus`), les pseudo-classes fonctionnelles acceptent des arguments.
- **Pseudo-classe `:is()`**: Cette pseudo-classe prend une liste de sélecteurs (ex. `ol`, `ul`) et sélectionne un élément qui correspond à l'un des sélecteurs de la liste.
- **Pseudo-classe `:where()`**: Cette pseudo-classe prend une liste de sélecteurs (ex. `ol`, `ul`) et sélectionne un élément qui correspond à l'un des sélecteurs de la liste. La différence entre `:is` et `:where` est que cette dernière a une spécificité de 0.
- **Pseudo-classe `:has()`**: Cette pseudo-classe est souvent appelée sélecteur `"parent"`, car elle te permet de styliser les éléments qui contiennent des éléments enfants indiqués dans la liste de sélecteurs.

## Pseudo-éléments

- **Pseudo-élément `::before`**: Ce pseudo-élément utilise la propriété `content` pour insérer du contenu cosmétique, comme des icônes, juste avant l'élément.
- **Pseudo-élément `::after`**: Ce pseudo-élément utilise la propriété `content` pour insérer du contenu cosmétique, comme des icônes, juste après l'élément.
- **Pseudo-élément `::first-letter`**: Ce pseudo-élément cible la première lettre du contenu d'un élément, ce qui te permet de la styliser.
- **Pseudo-élément `::marker`**: Ce pseudo-élément te permet de sélectionner le marqueur (puce ou numérotation) des éléments de liste pour le styliser.

## Théorie des couleurs

- **Définition de la théorie des couleurs**: C'est l'étude de la façon dont les couleurs interagissent entre elles et influencent notre perception. Elle couvre les relations entre les couleurs, l'harmonie des couleurs et l'impact psychologique des couleurs.
- **Couleurs primaires**: Ces couleurs, qui sont le jaune, le bleu et le rouge, sont les teintes fondamentales dont toutes les autres couleurs dérivent.
- **Couleurs secondaires**: Ces couleurs résultent du mélange de quantités égales de deux couleurs primaires. Le vert, l'orange et le violet sont des exemples de couleurs secondaires.
- **Couleurs tertiaires**: Ces couleurs résultent de la combinaison d'une couleur primaire avec une couleur secondaire voisine. Le jaune-vert, le bleu-vert et le bleu-violet sont des exemples de couleurs tertiaires.
- **Couleurs chaudes**: Ces couleurs, qui incluent les rouges, oranges et jaunes, évoquent des sensations de confort, de chaleur et de convivialité.
- **Couleurs froides**: Ces couleurs, qui incluent les bleus, verts et violets, évoquent des sensations de calme, de sérénité et de professionnalisme.
- **Roue chromatique**: La roue chromatique est un diagramme circulaire qui montre comment les couleurs sont liées entre elles. C'est un outil essentiel pour les designers, car il les aide à choisir des combinaisons de couleurs.
- **Schémas de couleurs analogues**: Ces schémas de couleurs créent des expériences cohérentes et apaisantes. Ils utilisent des couleurs analogues, qui sont adjacentes sur la roue chromatique.
- **Schémas de couleurs complémentaires**: Ces schémas de couleurs créent un fort contraste et un impact visuel. Leurs couleurs se trouvent aux extrémités opposées de la roue chromatique les unes par rapport aux autres.
- **Schéma de couleurs triadique**: Ce schéma de couleurs utilise des couleurs vibrantes. Elles proviennent de couleurs approximativement équidistantes les unes des autres. Si elles sont reliées, elles forment un triangle équilatéral sur la roue chromatique.
- **Schéma de couleurs monochromatique**: Dans ce schéma, toutes les couleurs dérivent de la même couleur de base en ajustant sa luminosité, son obscurité et sa saturation. Cela évoque une sensation d'unité et d'harmonie tout en créant du contraste.

## Différentes façons de travailler avec les couleurs en CSS

- **Couleurs nommées**: Ce sont des noms de couleurs prédéfinis reconnus par les navigateurs. Par exemple `blue`, `darkred`, `lightgreen`.
- **Fonction `rgb()`**: RGB signifie Red, Green et Blue, les couleurs primaires de la lumière. Ces trois couleurs sont combinées à différentes intensités pour créer une grande variété de couleurs. La fonction `rgb()` te permet de définir des couleurs avec le modèle de couleur RGB.
- **Fonction `rgba()`**: Cette fonction ajoute une quatrième valeur, alpha, qui contrôle la transparence de la couleur.
- **Fonction `hsl()`**: HSL signifie Hue, Saturation et Lightness, trois composants clés qui définissent une couleur.
- **Fonction `hsla()`**: Cette fonction ajoute une quatrième valeur, alpha, qui contrôle l'opacité de la couleur.
- **Hexadécimal**: Un code hex (abréviation de code hexadécimal) est une chaîne de six caractères utilisée pour représenter des couleurs dans le modèle RGB. Le terme "hex" fait référence au système de numération en base 16, qui utilise les chiffres 0 à 9 et les lettres A à F.

## Propriété `box-shadow`

- **Définition**: La propriété `box-shadow` applique une ou plusieurs ombres autour d'un élément.
- **Valeurs d'offset**: Tu dois préciser des valeurs horizontales (`offset-x`) et verticales (`offset-y`). Un `offset-x` positif déplace l'ombre vers la droite, tandis qu'une valeur négative la déplace vers la gauche. Un `offset-y` positif déplace l'ombre vers le bas, tandis qu'une valeur négative la déplace vers le haut. Si une valeur vaut `0`, tu n'as pas besoin d'inclure d'unité.
- **Rayon de flou**: Cette valeur facultative contrôle à quel point l'ombre paraît floue. Si elle n'est pas incluse, elle vaut `0` par défaut, ce qui crée des bords nets. Plus la valeur est élevée, plus l'ombre est douce.
- **Rayon d'étalement**: Cette valeur facultative contrôle à quel point l'ombre s'étend ou se rétrécit. Si elle n'est pas incluse, elle vaut `0` par défaut.
- **Couleur de l'ombre**: Tu peux préciser la couleur avec des couleurs nommées, des valeurs hex, ou les fonctions `rgb()`, `rgba()`, `hsl()` ou `hsla()`.
- **Mot-clé `inset`**: Ajouter le mot-clé `inset` place l'ombre à l'intérieur de l'élément au lieu de l'extérieur.
- **Appliquer plusieurs ombres de boîte**: Tu peux appliquer plusieurs ombres en les séparant par des virgules. Les ombres sont empilées de l'avant vers l'arrière.

### Syntaxe

```css
box-shadow: offset-x offset-y blur-radius spread-radius color;
```

## Dégradés linéaires et radiaux

- **Dégradés linéaires**: Ces dégradés créent une transition progressive entre des couleurs le long d'une ligne droite. Tu peux contrôler la direction de cette ligne et les couleurs utilisées.
- **Dégradés radiaux**: Ces dégradés créent des dégradés circulaires ou elliptiques qui rayonnent depuis un point central.

## Bonnes pratiques pour styliser les inputs

- **Styliser les inputs**: Comme pour tous les éléments de texte, tu dois t'assurer que les styles que tu appliques aux inputs de texte sont accessibles. Cela signifie que la police doit avoir une taille suffisante et que la couleur doit avoir assez de contraste avec l'arrière-plan. Les éléments input peuvent aussi recevoir le focus. Quand tu modifies tes styles, veille à conserver un indicateur visible quand l'élément a le focus, comme une bordure épaisse.

## Utiliser `appearance: none` pour les inputs

- **`appearance: none`**: Les navigateurs appliquent un style par défaut à beaucoup d'éléments. La propriété CSS `appearance: none` te donne un contrôle complet sur le style, mais elle vient avec quelques réserves. Quand tu construis des styles personnalisés pour les éléments input, tu dois t'assurer que les indicateurs de focus et d'erreur sont toujours présents.

## Problèmes courants pour styliser les propriétés `datetime-local` et `color`

- **Problèmes courants**: Ces types spéciaux d'inputs s'appuient sur des pseudo-éléments complexes pour créer des éléments comme les sélecteurs de date et de couleur. Cela pose un défi important pour styliser ces inputs. L'un des défis est que le style par défaut dépend entièrement du navigateur, donc le CSS que tu écris pour obtenir l'apparence voulue du sélecteur peut être totalement différent dans un autre navigateur.

## Propriété CSS overflow

- **Définition**: Overflow désigne la manière dont les éléments gèrent le contenu qui dépasse, ou "overflow", la taille de l'élément conteneur. Overflow est bidimensionnel.
- **`overflow-x`**: L'axe x détermine le débordement horizontal.
- **`overflow-y`**: L'axe y détermine le débordement vertical.

## Propriété CSS transform

- **Définition**: Cette propriété te permet d'appliquer différentes transformations aux éléments, comme les faire pivoter, les mettre à l'échelle, les incliner ou les déplacer en espace 2D ou 3D.
- **Fonction `translate()`**: Cette fonction sert à déplacer un élément depuis sa position actuelle.
- **Fonction `scale()`**: Cette fonction te permet de changer la taille d'un élément.
- **Transformations et accessibilité**: Si tu utilises transform pour masquer ou révéler du contenu, assure-toi que le contenu reste accessible aux lecteurs d'écran et à la navigation au clavier. Le contenu masqué doit être réellement masqué, par exemple avec `display: none` ou `visibility: hidden`, plutôt que simplement déplacé visuellement hors de l'écran.

## Le modèle de boîte

- **Définition**: Dans le modèle de boîte CSS, chaque élément est entouré d'une boîte. Cette boîte se compose de quatre éléments : la zone de contenu, `padding`, `border`, `margin`.
- **Zone de contenu**: La zone de contenu est la partie la plus interne de la boîte. C'est l'espace qui contient le contenu réel d'un élément, comme du texte ou des images.
- **`padding`**: Le padding est la zone située immédiatement après la zone de contenu. C'est l'espace entre la zone de contenu et la bordure d'un élément.
- **`border`**: La bordure est le bord externe ou le contour d'un élément dans le modèle de boîte CSS. C'est la limite visuelle de l'élément.
- **`margin`**: La marge est l'espace à l'extérieur de la bordure d'un élément. Elle détermine la distance entre un élément et les autres éléments autour de lui.

## Fusion des marges

- **Définition**: Ce comportement se produit lorsque les marges verticales d'éléments adjacents se superposent, ce qui produit une seule marge égale à la plus grande des deux. Ce comportement s'applique uniquement aux marges verticales (`top` et `bottom`), pas aux marges horizontales (`left` et `right`).

## Les valeurs de propriété `content-box` et `border-box`

- **Propriété `box-sizing`**: Cette propriété sert à déterminer comment les valeurs finales de `width` et `height` sont calculées pour un élément HTML.
- **Valeur `content-box`**: Dans le modèle `content-box`, les valeurs `width` et `height` que tu définis pour un élément déterminent les dimensions de la zone de contenu, mais elles n'incluent pas le `padding`, la `border` ou la `margin`.
- **Valeur `border-box`**: Avec `border-box`, les valeurs `width` et `height` d'un élément incluent la zone de contenu, le `padding` et la `border`, mais elles n'incluent pas la `margin`.

## Reset CSS

- **Définition**: Un reset CSS est une feuille de style qui supprime tout ou partie du formatage par défaut appliqué par les navigateurs web aux éléments HTML. Parmi les options tierces de resets CSS, on trouve `sanitize.css` et `normalize.css`.

## Propriété CSS filter

- **Définition**: Cette propriété peut servir à créer différents effets comme le flou, le changement de couleur et l'ajustement du contraste.
- **Fonction `blur()`**: Cette fonction applique un flou gaussien à l'élément. La quantité est définie en pixels et représente le rayon du flou.
- **Fonction `brightness()`**: Cette fonction ajuste la luminosité de l'élément. Une valeur de 0% rend l'élément complètement noir, tandis que les valeurs supérieures à 100% augmentent la luminosité.
- **Fonction `grayscale()`**: Cette fonction convertit l'élément en niveaux de gris. La quantité est définie en pourcentage, où 100% correspond à des niveaux de gris complets et 0% laisse l'image inchangée.
- **Fonction `sepia()`**: Cette fonction applique une teinte sépia à l'élément. Comme `grayscale`, elle utilise une valeur en pourcentage.
- **Fonction `hue-rotate()`**: Cette fonction applique une rotation de teinte à l'élément. La valeur est définie en degrés et représente une rotation autour du cercle chromatique.

## Introduction à CSS Flexbox et au modèle flex

- **Définition**: CSS flexbox est un modèle de mise en page unidimensionnel qui te permet d'organiser des éléments en lignes et en colonnes dans un conteneur.
- **Modèle flex**: Ce modèle définit comment les éléments flex sont disposés dans un conteneur flex. Chaque conteneur flex possède deux axes : l'axe principal et l'axe transversal.

## La propriété `flex-direction`

- **Définition**: Cette propriété définit la direction de l'axe principal. La valeur par défaut de `flex-direction` est `row`, ce qui place tous les éléments flex sur la même ligne, dans la direction de la langue par défaut de ton navigateur (de gauche à droite ou de droite à gauche).
- **`flex-direction: row-reverse;`**: Cela inverse les éléments de la ligne.
- **`flex-direction: column;`**: Cela aligne les éléments flex verticalement au lieu d'horizontalement.
- **`flex-direction: column-reverse;`**: Cela inverse l'ordre des éléments flex verticalement.

## La propriété `flex-wrap`

- **Définition**: Cette propriété détermine comment les éléments flex passent à la ligne dans un conteneur flex pour tenir dans l'espace disponible. `flex-wrap` peut prendre trois valeurs possibles : `nowrap`, `wrap` et `wrap-reverse`.
- **`flex-wrap: nowrap;`**: C'est la valeur par défaut. Les éléments flex ne passent pas sur une nouvelle ligne, même si leur largeur dépasse celle du conteneur.
- **`flex-wrap: wrap;`**: Cette propriété fait passer les éléments à la ligne lorsqu'ils dépassent la largeur de leur conteneur.
- **`flex-wrap: wrap-reverse;`**: Cette propriété fait passer les éléments flex à la ligne dans l'ordre inverse.
- **Propriété `flex-flow`**: Cette propriété est un raccourci pour `flex-direction` et `flex-wrap`.

## La propriété `justify-content`

- **Définition**: Cette propriété aligne les éléments enfants le long de l'axe principal du conteneur flex.
- **`justify-content: flex-start;`**: Dans ce cas, les éléments flex sont alignés au début de l'axe principal. Celui-ci peut être horizontal ou vertical.
- **`justify-content: flex-end;`**: Dans ce cas, les éléments flex sont alignés à la fin de l'axe principal, horizontalement ou verticalement.
- **`justify-content: center;`**: Cela centre les éléments flex le long de l'axe principal.
- **`justify-content: space-between;`**: Cela répartit les éléments uniformément le long de l'axe principal.
- **`justify-content: space-around;`**: Cela répartit les éléments flex uniformément sur l'axe principal, en ajoutant un espace avant le premier élément et après le dernier.
- **`justify-content: space-evenly;`**: Cela répartit les éléments uniformément le long de l'axe principal.

## La propriété `align-items`

- **Définition**: Cette propriété sert à répartir les éléments le long de l'axe transversal. Rappelle-toi que l'axe transversal est perpendiculaire à l'axe principal.
- **`align-items: center;`**: Sert à centrer les éléments le long de l'axe transversal.
- **`align-items: flex-start;`**: Aligne les éléments au début de l'axe transversal.
- **`align-items: stretch;`**: Sert à étirer les éléments flex le long de l'axe transversal.

## Introduction à la typographie

- **Définition**: La typographie est l'art de choisir les bonnes polices et la bonne mise en forme pour rendre un texte visuellement agréable et facile à lire. "Type" désigne la façon dont les caractères individuels sont conçus et arrangés.
- **Définition de typeface**: Une typeface est le design et le style global d'un ensemble de caractères, chiffres et symboles. C'est comme un plan pour une famille de polices.
- **Définition de police**: Une police est une variation précise d'une typeface avec des caractéristiques spécifiques, comme la taille, l'épaisseur, le style et la largeur.
- **Typeface serif**: Cette typeface a un style classique avec de petites lignes au bout des caractères. Les typefaces serif sont souvent utilisées pour les supports imprimés, comme les livres.
- **Typeface sans serif**: Cette typeface a une apparence plus moderne, sans les petites lignes au bout des caractères. Les typefaces sans serif sont souvent utilisées dans le design numérique parce qu'elles sont faciles à lire à l'écran. Helvetica, Arial et Roboto en sont des exemples.
- **Épaisseur de police**: C'est l'épaisseur des caractères, qui peut être légère, normale, grasse ou noire.
- **Style de police**: C'est l'inclinaison et l'orientation des caractères, comme italic et oblique.
- **Largeur de police**: C'est l'espace horizontal occupé par les caractères, comme condensed et expanded.
- **Ligne de base**: C'est la ligne imaginaire sur laquelle reposent la plupart des caractères.
- **Hauteur de capitale**: C'est la hauteur des lettres majuscules, mesurée depuis la ligne de base jusqu'au sommet.
- **Hauteur d'x**: C'est la hauteur moyenne des lettres minuscules, sans les hampes ascendantes et descendantes.
- **Hampes ascendantes**: Ce sont les parties des lettres minuscules qui dépassent au-dessus de la hauteur d'x, comme le haut des lettres 'h', 'b', 'd' et 'f'.
- **Hampes descendantes**: Ce sont les parties des lettres minuscules qui descendent sous la ligne de base, comme les queues de 'y', 'g', 'p' et 'q'.
- **Crénage**: C'est l'ajustement de l'espace entre certaines paires de caractères pour améliorer leur lisibilité et leur esthétique. Par exemple, réduire l'espace entre les lettres A et V.
- **Approche**: C'est l'ajustement de l'espace entre tous les caractères d'un bloc de texte. C'est essentiellement la distance entre les caractères. Cela influence la densité et l'ouverture du texte.
- **Interlignage**: C'est l'espace vertical entre les lignes de texte. Il se mesure de la ligne de base d'une ligne à la ligne de base de la ligne suivante.
- **Bonnes pratiques avec la typographie**: Tu dois choisir des polices claires et simples pour rendre tes designs faciles à comprendre. C'est particulièrement important pour le texte principal de ton site. Les utilisateurs sont plus susceptibles d'interagir avec ton contenu si la police est facile à lire. Tu dois utiliser au maximum deux ou trois polices pour créer une cohérence visuelle. Utiliser trop de polices peut rendre le texte plus difficile à lire et affaiblir l'identité de ta marque. Cela peut aussi affecter l'expérience utilisateur en augmentant le temps de chargement du site. Tu peux utiliser la taille de police pour créer une hiérarchie visuelle pour les titres, sous-titres, paragraphes et autres éléments. Par exemple, le titre principal d'une page web doit avoir une police plus grande, suivi de sous-titres avec des tailles de police plus petites. Cela donne à chaque élément de la hiérarchie une taille de police précise qui aide les utilisateurs à naviguer visuellement dans la structure.

## Propriété CSS `font-family`

- **Définition**: Une famille de polices est un groupe de polices qui partagent un design commun. Toutes les polices qui appartiennent à la même famille reposent sur la même typeface principale, mais elles ont aussi des variations de style, d'épaisseur et de largeur. Tu peux indiquer plusieurs familles de polices par ordre de priorité, de la plus haute à la plus basse, en les séparant par des virgules. Ces polices alternatives serviront d'options de secours. Tu dois toujours inclure une famille de polices générique à la fin de la liste `font-family`.
- **Familles de polices courantes**: Voici quelques familles de polices courantes utilisées en CSS : serif, sans-serif, monospace, cursive, fantasy

## Polices web-safe

- **Définition**: Ces polices sont un sous-ensemble de polices très susceptibles d'être installées sur un ordinateur ou appareil. Quand le navigateur doit afficher une police, il cherche le fichier de police sur le système de l'utilisateur. Mais si la police n'est pas trouvée, il revient généralement à une police système par défaut.

- **Polices web-safe**:
  - Les polices sans-serif sont souvent utilisées pour le développement web parce qu'elles n'ont pas de petits "pieds" ou traits à l'extrémité des caractères, ce qui les rend faciles à lire à l'écran. Arial, Verdana et Trebuchet MS sont des exemples de polices sans-serif web-safe.
  - À l'inverse, les polices serif ont de petits "pieds" au bout des caractères, elles sont donc couramment utilisées pour l'impression traditionnelle. Les polices serif web-safe incluent Times New Roman et Georgia.

## At-rules et règle `@font-face`

- **Définition**: Les at-rules sont des instructions données au navigateur. Tu peux les utiliser pour définir différents aspects de la feuille de style, comme les media queries, les keyframes, les polices et plus encore.
- **`@font-face`**: Cela te permet de définir une police personnalisée en indiquant le fichier de police, le format et d'autres propriétés importantes, comme l'épaisseur et le style. Pour que la règle `@font-face` soit valide, tu dois aussi préciser la propriété `src`. Cette propriété contient des références aux ressources de police.
- **Formats de police**: Pour chaque ressource de police, tu peux aussi préciser le format. C'est facultatif. C'est une indication pour le navigateur sur le format de la police. Si le format est omis, la ressource sera téléchargée et le format sera détecté après le téléchargement. Si le format est invalide, la ressource ne sera pas téléchargée. Les formats de police possibles incluent `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` et `woff2`.
- **`woff` et `woff2`**: `woff` signifie "Web Open Font Format". C'est un format de police largement utilisé, développé par Mozilla en collaboration avec Type Supply, LettError et d'autres organisations. La différence entre `woff` et `woff2` est l'algorithme utilisé pour compresser les données.
- **OpenType**: C'est un format de polices informatiques vectorielles développé par Microsoft et Adobe qui permet aux utilisateurs d'accéder à des fonctionnalités supplémentaires dans une police. Il est largement utilisé sur les principaux systèmes d'exploitation.
- **`tech()`**: Sert à préciser la technologie de la ressource de police. C'est facultatif.

## Travailler avec des polices externes

- **Définition**: Une police externe est un fichier de police qui n'est pas inclus directement dans les fichiers de ton projet. Elles sont généralement hébergées sur un serveur séparé. Pour inclure ces polices externes dans ton projet, tu peux utiliser un élément `link` ou une instruction `@import` dans ton CSS.
- **Google Fonts**: C'est un service de Google qui propose une collection de polices, dont beaucoup sont conçues spécialement pour le développement web.
- **Font Squirrel**: C'est une autre ressource populaire que tu peux utiliser pour ajouter des polices externes personnalisées à tes projets.

## Propriété `text-shadow`

- **Définition**: Cette propriété sert à appliquer des ombres au texte. Tu dois indiquer les offsets X et Y, qui représentent respectivement la distance horizontale et verticale de l'ombre par rapport au texte. Ces valeurs sont obligatoires. Des valeurs positives pour les offsets X et Y déplacent l'ombre respectivement vers la droite et vers le bas, tandis que des valeurs négatives la déplacent vers la gauche et vers le haut.
- **Couleur de l'ombre**: Tu peux aussi personnaliser la couleur de l'ombre en indiquant cette valeur avant ou après l'offset. Si la couleur n'est pas précisée, le navigateur la détermine automatiquement, donc il est généralement préférable de la définir explicitement.
- **Rayon de flou**: Le rayon de flou est facultatif, mais il rend l'ombre beaucoup plus douce et subtile. La valeur par défaut du rayon de flou est zéro. Plus la valeur est élevée, plus le flou est grand, ce qui signifie que l'ombre devient plus légère.
- **Appliquer plusieurs ombres de texte**: Le texte peut avoir plusieurs ombres. Les ombres sont appliquées en couches, de l'avant vers l'arrière, avec la première ombre au-dessus.

## Arbre d'accessibilité

L'arbre d'accessibilité est une structure utilisée par les technologies d'assistance, comme les lecteurs d'écran, pour interpréter le contenu d'une page web et interagir avec lui.

## Fonction `max()`

La fonction `max()` renvoie la plus grande valeur d'un ensemble de valeurs séparées par des virgules :

```css
img {
  width: max(250px, 25vw);
}
```

Dans l'exemple ci-dessus, la largeur de l'image sera de 250px si la largeur du viewport est inférieure à 1000 pixels (puisque 250px est supérieur à 25% de cette largeur de viewport). Si la largeur du viewport est supérieure à 1000 pixels, la largeur de l'image sera de 25vw. C'est parce que 25vw est égal à 25% de la largeur du viewport.

## Bonnes pratiques avec CSS et l'accessibilité

- **`display: none;`**: Utiliser `display: none;` signifie que les lecteurs d'écran et autres technologies d'assistance ne pourront pas accéder à ce contenu, car il n'est pas inclus dans l'arbre d'accessibilité. Il est donc important d'utiliser cette méthode seulement lorsque tu veux retirer complètement le contenu à la fois de la présentation visuelle et de l'accessibilité.
- **`visibility: hidden;`**: Cette propriété et cette valeur masquent visuellement le contenu, mais le gardent dans le flux du document, ce qui signifie qu'il occupe toujours de l'espace sur la page. Ces éléments ne seront également plus lus par les lecteurs d'écran, car ils auront été retirés de l'arbre d'accessibilité.
- **Classe CSS `.sr-only`**: C'est une technique courante utilisée pour masquer visuellement du contenu tout en le gardant accessible aux lecteurs d'écran.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: Cette propriété et cette valeur activent un comportement de défilement fluide.
- **Fonctionnalité `prefers-reduced-motion`**: C'est une fonctionnalité média qui peut être utilisée pour détecter les préférences d'animation de l'utilisateur.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

Dans l'exemple ci-dessus, le défilement fluide est activé si l'utilisateur n'a pas défini de préférence d'animation sur son appareil.

## Masquer du contenu avec des attributs HTML

- **Attribut `aria-hidden`**: Utilisé pour masquer un élément aux personnes qui utilisent une technologie d'assistance comme les lecteurs d'écran. Par exemple, cela peut servir à masquer des images décoratives qui n'apportent aucun contenu significatif.
- **Attribut `hidden`**: Cet attribut est pris en charge par la plupart des navigateurs modernes et masque le contenu à la fois visuellement et dans l'arbre d'accessibilité. Il peut être facilement activé ou désactivé avec JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## Problème d'accessibilité de l'attribut `placeholder`

Utiliser du texte placeholder n'est pas idéal pour l'accessibilité. Trop souvent, les utilisateurs confondent le texte placeholder avec une vraie valeur d'input : ils pensent qu'une valeur est déjà présente dans l'input.

## Travailler avec différents sélecteurs d'attribut et les liens

- **Définition**: Le sélecteur `attribute` te permet de cibler des éléments HTML selon leurs attributs comme les attributs `href` ou `title`.
- **Attribut `title`**: Cet attribut fournit des informations supplémentaires sur un élément.

## Cibler des éléments avec les attributs `lang` et `data-lang`

- **Attribut `lang`**: Cet attribut est utilisé en HTML pour indiquer la langue du contenu dans un élément. Tu peux vouloir styliser les éléments différemment selon la langue dans laquelle ils sont écrits, en particulier sur un site multilingue.
- **Attribut `data-lang`**: Les attributs de données personnalisés comme l'attribut `data-lang` sont souvent utilisés pour stocker des informations supplémentaires dans les éléments, par exemple pour indiquer la langue utilisée dans une section précise de texte.

## Travailler avec le sélecteur d'attribut, les éléments de liste ordonnée et l'attribut `type`

- **Attribut `type`**: Quand tu travailles avec des listes ordonnées en HTML, l'attribut `type` te permet d'indiquer le style de numérotation utilisé, par exemple numérique, alphabétique ou en chiffres romains.

## Travailler avec les flottants

- **Définition**: Les flottants servent à retirer un élément de son flux normal sur la page et à le positionner à gauche ou à droite de son conteneur. Quand cela se produit, le texte s'enroule autour de ce contenu flottant.
- **Effacer les flottants**: La propriété `clear` sert à déterminer si un élément doit être déplacé sous le contenu flottant. Quand plusieurs éléments flottants sont empilés les uns à côté des autres, il peut y avoir des problèmes de chevauchement et d'effondrement dans les mises en page. Un hack `clearfix` a donc été créé pour résoudre ce problème.

## Positionnements statique, relatif et absolu

- **Positionnement statique**: C'est le flux normal du document. Les éléments sont positionnés de haut en bas et de gauche à droite, les uns après les autres.
- **Positionnement relatif**: Cela te permet d'utiliser les propriétés `top`, `left`, `right` et `bottom` pour positionner l'élément dans le flux normal du document. Tu peux aussi utiliser le positionnement relatif pour faire chevaucher des éléments avec d'autres éléments sur la page.
- **Positionnement absolu**: Cela te permet de retirer un élément du flux normal du document, ce qui le fait se comporter indépendamment des autres éléments.

## Positionnements fixe et sticky

- **Positionnement fixe**: Quand un élément est positionné avec `position: fixed`, il est retiré du flux normal du document et placé par rapport au viewport, ce qui signifie qu'il reste à la même position même quand l'utilisateur fait défiler la page. C'est souvent utilisé pour des éléments comme les en-têtes ou les barres de navigation qui doivent rester visibles en permanence.
- **Positionnement sticky**: Ce type de positionnement agit comme un hybride entre le positionnement relatif et fixe. Au départ, l'élément se comporte comme s'il était positionné relativement, en restant dans le flux du document. Mais une fois que l'utilisateur le fait défiler au-delà d'un certain point, il "colle" au viewport (généralement en haut) et se comporte comme s'il était fixe. C'est très pratique pour créer des éléments comme des barres de navigation sticky, qui ne deviennent fixes qu'une fois que l'utilisateur a fait défiler jusqu'à une certaine position.

## Travailler avec la propriété `z-index`

- **Définition**: La propriété `z-index` en CSS sert à contrôler l'ordre d'empilement vertical des éléments positionnés qui se chevauchent sur la page.

## Design web responsive

- **Définition**: Le principe central du design responsive est l'adaptabilité, c'est-à-dire la capacité d'un site web à ajuster sa mise en page et son contenu selon la taille de l'écran et les capacités de l'appareil sur lequel il est consulté.
- **Grilles fluides**: Elles utilisent des unités relatives comme les pourcentages plutôt que des unités fixes comme les pixels, ce qui permet au contenu de se redimensionner et de se réorganiser selon la taille de l'écran.
- **Images flexibles**: Elles sont configurées pour se redimensionner à l'intérieur de leurs éléments conteneurs, afin de ne pas déborder de leurs conteneurs sur les petits écrans.

## Media queries

- **Définition**: Cela permet aux développeurs d'appliquer différents styles selon les caractéristiques de l'appareil, principalement la largeur du viewport.
- **Type de média `all`**: Convient à tous les appareils. C'est la valeur par défaut si aucun type de média n'est précisé.
- **Types de média `print`**: Destiné aux documents paginés et aux documents affichés à l'écran en mode aperçu avant impression.
- **Types de média `screen`**: Destiné principalement aux écrans.
- **`aspect-ratio`**: Décrit le rapport entre la largeur et la hauteur du viewport.
- **`orientation`**: Sert à indiquer si l'appareil est en orientation paysage ou portrait.
- **`resolution`**: Sert à décrire la résolution du périphérique de sortie en points par pouce (dpi) ou points par centimètre (dpcm).
- **`hover`**: Sert à tester si le mécanisme de saisie principal peut survoler des éléments.
- **`prefers-color-scheme`**: Sert à détecter si l'utilisateur a demandé un thème de couleur clair ou sombre.
- **Media queries et opérateurs logiques**: L'opérateur `and` sert à combiner plusieurs fonctionnalités média, tandis que `not` et `only` peuvent servir à nier ou isoler des media queries.

## Breakpoints media courants

- **Définition**: Les breakpoints media sont des points précis dans le design d'un site web où la mise en page et le contenu s'ajustent pour s'adapter à différentes tailles d'écran. Il existe quelques breakpoints généraux que tu peux utiliser pour cibler les téléphones, tablettes et écrans de bureau. Mais il n'est pas judicieux d'essayer de couvrir chaque taille d'écran possible pour chaque appareil.
- **Petits appareils (smartphones)**: jusqu'à 640px
- **Appareils moyens (tablettes)**: de 641px à 1024px
- **Grands appareils (ordinateurs de bureau)**: 1025px et plus

## Approche mobile-first

- **Définition**: L'approche mobile-first est une philosophie de design et une stratégie de développement en design web responsive qui donne la priorité à la création de sites web pour les appareils mobiles avant de concevoir pour les écrans plus grands.

## Propriétés personnalisées CSS (variables CSS)

- **Définition**: Les propriétés personnalisées CSS, aussi appelées variables CSS, sont des entités définies par les auteurs CSS qui contiennent des valeurs précises à réutiliser dans tout un document. C'est une fonctionnalité puissante qui permet de créer des feuilles de style plus efficaces, maintenables et flexibles. Les propriétés personnalisées sont particulièrement utiles pour créer des designs thémables. Tu peux définir un ensemble de propriétés pour différents thèmes.

## La règle `@property`

- **Définition**: La règle `@property` est une fonctionnalité CSS puissante qui permet aux développeurs de définir des propriétés personnalisées avec plus de contrôle sur leur comportement, notamment leur animation et leurs valeurs initiales.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: C'est le nom de la propriété personnalisée que tu définis. Comme toutes les propriétés personnalisées, elle doit commencer par deux tirets.
**`syntax`**: Cela définit le type de la propriété, qui peut être par exemple `<color>`, `<length>`, `<number>`, `<percentage>` ou des types plus complexes.
- **`inherits`**: Cela indique si la propriété doit hériter sa valeur de son élément parent.
- **`initial-value`**: Cela définit la valeur par défaut de la propriété.
- **Fallbacks**: Quand tu utilises la propriété personnalisée, tu peux fournir une valeur de secours avec la fonction `var()`, comme tu le ferais avec des propriétés personnalisées classiques.

## Bases de CSS Grid

- **Définition**: CSS Grid est un système de mise en page bidimensionnel utilisé pour créer des mises en page complexes dans les pages web. Les grilles ont des lignes et des colonnes avec des espaces entre elles. Pour définir une mise en page en grille, tu dois définir la propriété `display` sur `grid`.
- **Unité `fr` (fraction)**: Cette unité représente une fraction de l'espace dans le conteneur de grille. Tu peux utiliser l'unité `fr` pour créer des grilles flexibles.
- **Créer des espaces entre les pistes**: Il existe trois façons de créer des espaces entre les pistes dans CSS Grid. Tu peux utiliser la propriété `column-gap` pour créer des espaces entre les colonnes. Tu peux utiliser la propriété `row-gap` pour créer des espaces entre les lignes. Tu peux aussi utiliser la propriété raccourcie `gap` pour créer des espaces entre les lignes et les colonnes.
- **Fonction `repeat()`**: Cette fonction sert à répéter des sections dans la liste des pistes. Au lieu d'écrire `grid-template-columns: 1fr 1fr 1fr;`, tu peux utiliser la fonction `repeat()`.
- **Grille explicite**: Tu peux indiquer le nombre de lignes ou de pistes avec les propriétés `grid-template-columns` ou `grid-template-rows`.
- **Grille implicite**: Quand des éléments sont placés en dehors de la grille, des lignes et colonnes sont créées automatiquement pour ces éléments hors grille.
- **Fonction `minmax()`**: Cette fonction sert à définir les tailles minimale et maximale d'une piste.
- **Placement automatique dans la grille**: Le navigateur place automatiquement les éléments avec l'algorithme de placement automatique. La propriété `grid-auto-flow` contrôle ce comportement, avec des valeurs comme `row`, `column` ou `dense` pour influencer la direction du placement et décider si les espaces vides sont remplis.
- **Placement basé sur les lignes**: Toutes les grilles ont des lignes. Pour indiquer où l'élément commence sur une ligne, tu peux utiliser les propriétés `grid-column-start` et `grid-row-start`. Pour indiquer où l'élément se termine sur la ligne, tu peux utiliser les propriétés `grid-column-end` et `grid-row-end`. Tu peux aussi choisir d'utiliser les propriétés raccourcies `grid-column` ou `grid-row`.
- **`grid-template-areas`**: Cette propriété sert à fournir un nom aux éléments que tu vas positionner dans la grille.
- **Alignement :** Utilise `justify-items` et `align-items` pour contrôler comment les éléments sont alignés dans leurs zones de grille. Utilise `justify-self` et `align-self` sur les éléments de grille individuels pour remplacer l'alignement défini sur le conteneur.

## Bases des animations CSS

- **Définition**: Les animations CSS te permettent de créer des effets dynamiques et visuellement engageants sur les pages web sans JavaScript ni programmation complexe. Elles fournissent un moyen de faire passer doucement les éléments entre différents styles sur une durée précise.
- **Règle `@keyframes`**: Cette règle définit les étapes et styles de l'animation. Elle indique quels styles l'élément doit avoir à différents moments de l'animation.
- **Propriété `animation`**: C'est la propriété raccourcie utilisée pour appliquer des animations.
- **`animation-name`**: Indique le nom de la règle `@keyframes` à utiliser.
- **`animation-duration`**: Définit la durée nécessaire à l'animation pour se terminer.
- **`animation-timing-function`**: Définit la progression de l'animation dans le temps (par exemple ease, linear, ease-in-out).
- **`animation-delay`**: Indique un délai avant le début de l'animation.
- **`animation-iteration-count`**: Définit combien de fois l'animation doit se répéter.
- **`animation-direction`**: Détermine si l'animation doit se jouer vers l'avant, vers l'arrière ou en alternance.
- **`animation-fill-mode`**: Indique comment l'élément doit être stylisé avant et après l'animation.
- **`animation-play-state`**: Te permet de mettre l'animation en pause et de la reprendre.

## Accessibilité et media query `prefers-reduced-motion`

- **Media query `prefers-reduced-motion`**: L'une des principales préoccupations d'accessibilité avec les animations est qu'elles peuvent provoquer une gêne ou même un dommage physique chez certains utilisateurs. Les personnes avec des troubles vestibulaires ou une sensibilité au mouvement peuvent ressentir des étourdissements, des nausées ou des maux de tête lorsqu'elles sont exposées à certains types de mouvements à l'écran. La media query `prefers-reduced-motion` permet aux développeurs web de détecter si l'utilisateur a demandé un minimum d'animations ou d'effets de mouvement au niveau du système.
