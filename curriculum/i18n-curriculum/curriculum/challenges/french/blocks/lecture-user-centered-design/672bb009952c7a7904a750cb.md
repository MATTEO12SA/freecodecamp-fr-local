Le défilement infini est un motif de design qui charge plus de contenu à mesure que l'utilisateur fait défiler la page vers le bas. Souvent, c'est utilisé sur les sites de réseaux sociaux comme Twitter. Par exemple, si tu es connecté et que tu veux voir plus de tweets, tu peux faire défiler vers le bas et plus de tweets se chargeront. C'est un exemple de défilement infini.

Le défilement infini est aussi utilisé comme substitut à la pagination. La pagination est un motif de design qui découpe le contenu en pages. C'est souvent utilisé quand il y a beaucoup de contenu à afficher. Un exemple de pagination est quand tu cherches quelque chose sur Google et que tu vois les résultats de recherche sur plusieurs pages. Avec la pagination, tu dois cliquer sur un bouton pour aller à la page suivante. Avec le défilement infini, tu continues juste à faire défiler vers le bas et plus de contenu se charge.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="pagination">
  <div class="post">Post 1</div>
  <div class="post">Post 2</div>
  <div class="post">Post 3</div>
  <!-- More posts will be on the next pages -->
</div>
<div class="pagination-controls">
  <button class="prev" disabled>Previous</button>
  <button class="next">Next</button>
</div>
<script src="index.js"></script>
```

```css
.pagination {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.post {
  padding: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
}

.pagination-controls {
  margin-top: 12px;
}

.pagination-controls button {
  padding: 8px 16px;
  margin-right: 8px;
}
```

```js
let currentPage = 1;
const postsPerPage = 3;
const totalPosts = 9; 
const totalPages = Math.ceil(totalPosts / postsPerPage);
const container = document.querySelector('.pagination');
const prevButton = document.querySelector('.prev');
const nextButton = document.querySelector('.next'); 

function renderPosts() {
  container.innerHTML = '';
  const start = (currentPage - 1) * postsPerPage;
  const end = start + postsPerPage;
  for (let i = start; i < end && i < totalPosts; i++) {
    const post = document.createElement('div');
    post.className = 'post';
    post.textContent = `Post ${i + 1}`;
    container.appendChild(post);
  }
  prevButton.disabled = currentPage === 1;
  nextButton.disabled = currentPage === totalPages;
}

prevButton.addEventListener('click', () => {
  if (currentPage > 1) {
    currentPage--;
    renderPosts();
  }
});

nextButton.addEventListener('click', () => {
  if (currentPage < totalPages) {
    currentPage++;
    renderPosts();
  }
});

renderPosts();
```

À mesure que tu incorpores le défilement infini dans ton design, il y a quelques bonnes pratiques à garder en tête. La première considération est de fournir un bouton « Load More » qui charge le prochain ensemble de résultats quand l'utilisateur clique dessus. C'est une bonne façon de donner à l'utilisateur le contrôle sur quand il veut voir plus de contenu.

Une autre considération serait d'ajouter un bouton « Back ». Cela donne aux utilisateurs la capacité de revenir à la page précédente sans avoir à faire défiler tout le chemin vers le haut. Cela crée une meilleure expérience utilisateur et leur donne plus de contrôle sur leur expérience de navigation.

Parfois, tu verras des designs avec un bouton « Back to the top » qui ramène les utilisateurs en haut de la page de résultats. Une autre considération est de fournir un indicateur de chargement. Les utilisateurs devraient avoir une indication claire que plus de contenu est en cours de chargement ; autrement, ils pourraient penser que la page est cassée.

:::interactive_editor

```js
const container = document.querySelector('.infinite-scroll');
const loadingIndicator = document.querySelector('.loading-indicator');
const backToTopBtn = document.getElementById('back-to-top');

window.addEventListener('scroll', () => {
  if (window.scrollY > 400) {
    backToTopBtn.style.display = 'block';
  } else {
    backToTopBtn.style.display = 'none';
  }

  if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {
    loadMorePosts();
  }
});

backToTopBtn.addEventListener('click', () => {
  window.scrollTo({ top: 0, behavior: 'smooth' });
});

function loadMorePosts() {
  loadingIndicator.style.display = 'block';
  setTimeout(() => {
    for (let i = 0; i < 3; i++) {
      const post = document.createElement('div');
      post.className = 'post';
      post.textContent = `Post ${container.children.length + 1}`;
      container.appendChild(post);
    }
    loadingIndicator.style.display = 'none';
  }, 1000);
}
```

Une des dernières considérations serait de garder le pied de page accessible à l'utilisateur. Si le pied de page contient des informations importantes, alors il devrait être accessible à l'utilisateur à tout moment.

En conclusion, le défilement infini est une excellente façon d'afficher du contenu sur ton site web. Cependant, tu devrais garder en tête les bonnes pratiques quand tu conçois ton défilement infini pour que tu puisses fournir la meilleure expérience utilisateur possible.
