# --interactive--

La divulgation progressive (progressive disclosure) est un motif de design utilisé pour ne montrer aux utilisateurs que le contenu pertinent en fonction de leur activité actuelle et cacher le reste. C'est fait pour réduire la charge cognitive et rendre l'expérience utilisateur plus intuitive.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<div class="progressive-disclosure">
  <button id="show-details-btn" aria-expanded="false" aria-controls="extra-content">
    Show More Details
  </button>
  <div id="extra-content" class="hidden" tabindex="-1">
    <p>
      Here are additional details that are revealed only when you want to see them.
      This keeps the interface clean and reduces cognitive load.
    </p>
  </div>
</div>
<script src="index.js"></script>

```

```js
const btn = document.getElementById('show-details-btn');
const content = document.getElementById('extra-content');

btn.addEventListener('click', () => {
  const isHidden = content.classList.toggle('hidden');

  if (!isHidden) {
    content.focus();
  }
});

```

Par exemple, quand tu utilises la page de recherche de Google, la plupart de tes recherches seront simples. Donc avoir une interface utilisateur simple avec très peu d'options a du sens. Mais pour les quelques fois où tu auras besoin d'une recherche plus avancée, Google fournit une option de recherche avancée qui révélera plus d'options.

Tu peux trouver l'option de recherche avancée de Google depuis le menu des paramètres sur la page d'accueil de Google. Ou tu peux aller à `google.com/advanced_search`.

Un autre exemple de divulgation progressive serait le bouton « More details » sur les produits dans un site web e-commerce. Quand tu visites un site comme Amazon, tu verras une liste d'images de produits dans différentes catégories. Cela permet aux utilisateurs de faire leurs achats avec facilité et de ne pas être submergés par trop d'informations.

Si l'utilisateur est intéressé par un produit particulier, il peut cliquer sur l'image du produit pour révéler plus de détails sur le produit. Afficher toutes les informations immédiatement peut être submergeant pour l'utilisateur. Donc, en utilisant la divulgation progressive, l'utilisateur peut choisir de voir plus d'informations quand il est prêt.

document.querySelectorAll('.details-btn').forEach(button => {
  button.addEventListener('click', () => {
    const product = button.closest('.product');
    const details = product.querySelector('.product-details');
    const isHidden = details.classList.toggle('hidden');
    product.setAttribute('aria-expanded', !isHidden);
    if (!isHidden) {
      details.focus();
      button.textContent = 'Hide details';
    } else {
      button.textContent = 'More details';
    }
  });
});

Alors, quelles sont quelques bonnes pratiques pour concevoir avec la technique de divulgation progressive ? La première considération est de garder toutes les informations importantes visibles à tout moment. Les utilisateurs ne devraient pas avoir à lutter pour trouver l'information clé parce qu'elle est cachée par des fonctionnalités avancées supplémentaires.

Une autre considération est de fournir un point d'accès unique pour que les utilisateurs accèdent à des fonctionnalités ou des informations supplémentaires. Cela pourrait être un bouton ou un lien qui est toujours visible sur la page. Ajouter plusieurs points d'accès peut être déroutant pour les utilisateurs et peut conduire à une mauvaise expérience utilisateur.

:::interactive_editor
