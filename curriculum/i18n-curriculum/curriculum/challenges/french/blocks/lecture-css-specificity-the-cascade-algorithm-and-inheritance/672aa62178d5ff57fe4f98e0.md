# --interactive--

La spécificité CSS est un concept fondamental qui détermine quels styles sont appliqués à un élément quand plusieurs règles pourraient s'appliquer.

Comprendre la spécificité aide les développeurs à résoudre les conflits entre différentes règles CSS et garantit que les styles souhaités sont appliqués de manière cohérente.

La spécificité CSS est calculée en fonction du type de sélecteurs utilisés.

La plus haute spécificité est attribuée aux styles inline, qui sont appliqués directement à un élément via l'attribut `style`.

Dans cet exemple, le premier paragraphe sera rouge tandis que les autres éléments `p` seront bleus. C'est parce que les styles inline ont une spécificité plus élevée que les sélecteurs de type comme le sélecteur `p` montré dans le fichier `styles.css`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p style="color: red;">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
p {
  color: blue;
}
```

:::

Ensuite, les sélecteurs ID ont un haut niveau de spécificité.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Red paragraph</p>
<p>Other paragraph</p>
<p>Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

Ensuite, les sélecteurs de classe, les sélecteurs d'attribut et les pseudo-classes entrent en jeu.

Les exemples incluent les sélecteurs de classe comme `.container` et `.button`, les sélecteurs d'attribut comme `[type="text"]`, et les pseudo-classes comme `:hover`. Ceux-ci ont un niveau de spécificité modéré.

**NOTE** : Tu en apprendras plus sur les pseudo-classes dans les futures leçons.

Dans cet exemple, le premier paragraphe sera rouge parce qu'un sélecteur `id` a une spécificité plus élevée que les classes et les sélecteurs de type. Les éléments `.example-para` auront une couleur verte tandis que les éléments paragraphe restants auront une couleur bleue.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p class="example-para">Other paragraph</p>
<p class="example-para">Another paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

.example-para {
  color: green;
}

p {
  color: blue;
}
```

:::

Les sélecteurs de type, comme `div` et `h1`, et les pseudo-éléments comme `::before` et `::after`, ont une spécificité plus faible par rapport aux groupes précédents.

**NOTE** : Tu en apprendras plus sur les pseudo-éléments dans les futures leçons.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
#para {
  color: red; 
}

p {
  color: blue;
}
```

:::

Enfin, le sélecteur universel, représenté par un astérisque `*`, a la spécificité la plus faible de tous.

Voici un exemple de définition de la couleur pour tous les éléments en rouge en utilisant le sélecteur `*`. Tu ne verras cependant aucun élément rouge parce qu'il y a des sélecteurs ID et de type qui remplacent ces styles, ce qui souligne la faible spécificité du sélecteur `*`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p id="para">Example paragraph</p>
<p>Other paragraph</p>
<p>Yet another paragraph</p>
```

```css
* {
  color: red;
}

#para {
  color: green; 
}

p {
  color: blue;
}
```

:::

Les valeurs de spécificité sont calculées comme une valeur en quatre parties `(a, b, c, d)` :

- `a` : Styles inline (`1` ou `0`).
- `b` : Nombre de sélecteurs ID.
- `c` : Nombre de sélecteurs de classe, de sélecteurs d'attribut et de pseudo-classes.
- `d` : Nombre de sélecteurs de type, de pseudo-éléments et de sélecteurs universels.

Chaque partie de la valeur de spécificité a un poids différent :

- **Les styles inline** (`a`) ont le poids le plus élevé, contribuant à une valeur de `1` à la première partie de la valeur de spécificité.

- **Les sélecteurs ID** (`b`) portent le deuxième plus haut poids, avec chaque ID contribuant `1` à la deuxième partie de la valeur de spécificité.

- **Les sélecteurs de classe, les sélecteurs d'attribut et les pseudo-classes** (`c`) portent un poids modéré, avec chacun contribuant `1` à la troisième partie de la valeur de spécificité.

- **Les sélecteurs de type et les pseudo-éléments** (`d`) ont le poids le plus faible, avec chacun contribuant `1` à la quatrième partie de la valeur de spécificité.

- **Le sélecteur universel** (`*`) : Le sélecteur universel contribue `0` au calcul de spécificité et n'affecte pas la spécificité. Son inclusion dans un sélecteur ne change pas la valeur de spécificité.

Le CSS inline a la spécificité la plus élevée parce qu'il est appliqué directement à l'élément. Il remplace tout CSS interne ou externe. La valeur de spécificité pour les styles inline est `(1, 0, 0, 0)`.

Voici un autre exemple de style inline :

:::interactive_editor

```html
<p style="color: red;">This text is red.</p>
```

:::

Le CSS interne est défini dans un élément `style` dans la section `head` du document HTML. Il a une spécificité plus faible que les styles inline. Le CSS interne et externe partagent tous deux le même niveau de spécificité — la spécificité est déterminée par les sélecteurs utilisés, pas par l'endroit où le CSS est défini. Quand deux règles partagent une spécificité égale, celle qui apparaît plus tard dans le document gagne. Donc si une feuille de style externe est liée après un bloc `style` interne, les règles externes ont la priorité. Si elle est liée avant, les règles internes gagnent.

La valeur de spécificité pour les styles internes est déterminée par les sélecteurs utilisés. Par exemple, un sélecteur ID dans le CSS interne a une valeur de spécificité de `(0, 1, 0, 0)`.

:::interactive_editor

```html
<head>
  <style>
    #text {
      color: blue;
    }
  </style>
</head>
<body>
  <div id="text">This text is blue.</div>
</body>
```

:::

Dans cet exemple, le texte sera bleu sauf si un style inline ou un sélecteur plus spécifique est appliqué.

Le CSS externe est lié via un élément `link` dans la section `head` et est écrit dans des fichiers `.css` séparés. Comme le CSS interne, la spécificité des styles externes est déterminée par les sélecteurs utilisés. Le CSS externe offre la meilleure maintenabilité pour les gros projets.

La valeur de spécificité pour les styles externes est aussi déterminée par les sélecteurs utilisés. Par exemple, un sélecteur de classe dans le CSS externe a une valeur de spécificité de `(0, 0, 1, 0)`.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <div class="text">This text's color is defined in an external CSS file.</div>
</body>
```

```css
.text {
  color: purple;
}
```

:::

Dans cet exemple, la couleur du texte est définie dans le fichier `styles.css` et sera appliquée sauf si elle est remplacée par un sélecteur plus spécifique ou un style inline.
