# --interactive--

L'algorithme de cascade est le processus que le navigateur utilise pour décider quelles règles CSS appliquer quand il y a plusieurs styles ciblant le même élément. Il garantit que les styles les plus appropriés sont utilisés, en se basant sur un ensemble de règles bien définies.

Le processus commence par la **pertinence**. Le navigateur filtre d'abord toutes les règles CSS pour trouver celles qui s'appliquent réellement à l'élément en question. Cela inclut la correspondance des sélecteurs et la prise en compte des media queries qui pourraient être en vigueur.

Une media query est une technique CSS utilisée pour appliquer des styles basés sur les caractéristiques de l'appareil ou du viewport, comme sa largeur, sa hauteur ou son orientation.

Ensuite, l'algorithme considère l'**origine et l'importance**. Le CSS peut venir de différentes sources : les styles par défaut du navigateur (user-agent), les styles définis par l'utilisateur, et les styles écrits par l'auteur (toi).

Après avoir pris en compte l'origine, l'algorithme évalue ensuite l'importance de chaque règle, donnant la priorité aux règles marquées avec `!important`, qui remplacent les autres règles indépendamment de leur source.

Après avoir filtré par origine et importance, l'algorithme regarde la **spécificité**. Quand deux règles de la même origine et du même niveau d'importance s'appliquent, la règle avec la spécificité la plus élevée sera appliquée.

La spécificité est une mesure de la précision avec laquelle un sélecteur cible, les sélecteurs plus spécifiques ayant la priorité sur les plus généraux.

Enfin, si tout le reste est égal, l'**ordre d'apparition** entre en jeu. Quand deux règles ont la même spécificité, celle qui apparaît en dernier dans le CSS sera appliquée.

C'est pourquoi l'ordre dans lequel tu écris tes styles peut parfois affecter le résultat.

Regardons un exemple.

Dans le fichier HTML, il y a un seul élément paragraphe. Puis à l'intérieur du CSS, on a deux règles, chacune ciblant l'élément paragraphe.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<p>example paragraph</p>
```

```css
p {
  color: blue;
}

p {
  color: green; 
}
```

:::

La première règle définit tous les éléments paragraphe à la couleur de texte bleue tandis que la seconde règle définit tous les éléments paragraphe à la couleur de texte verte.

Alors quelle couleur sera appliquée ? La couleur verte sera appliquée aux éléments paragraphe.

En considérant la pertinence, l'origine et l'importance, la spécificité, la portée et l'ordre d'apparition, l'algorithme de cascade garantit que ton CSS se comporte de manière prévisible, te permettant de concevoir des pages web plus complexes et nuancées.
