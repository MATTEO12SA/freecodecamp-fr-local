## Outils de contraste des couleurs

- **Color Contrast Checker de WebAIM** : cet outil en ligne te permet de saisir les couleurs de premier plan et d'arrière-plan de ton design, puis de voir immédiatement si elles respectent les standards Web Content Accessibility Guidelines (WCAG).
- **TPGi Colour Contrast Analyzer** : c'est un vérificateur gratuit du contraste des couleurs qui te permet de vérifier si tes sites web et tes applications respectent les standards Web Content Accessibility Guidelines (WCAG). Cet outil inclut aussi une fonctionnalité de simulation du daltonisme qui te permet de voir à quoi ressemble ton site web ou ton application pour les personnes ayant des problèmes de perception des couleurs.

## Arbre d'accessibilité

L'arbre d'accessibilité est une structure utilisée par les technologies d'assistance, comme les lecteurs d'écran, pour interpréter le contenu d'une page web et interagir avec lui.

## Fonction `max()`

La fonction `max()` renvoie la plus grande valeur d'un ensemble de valeurs séparées par des virgules :

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <img src="https://cdn.freecodecamp.org/curriculum/css-photo-gallery/9.jpg" alt="A cat sitting on a table">
</body>
```

```css
img {
  width: max(250px, 25vw);
}
```

:::

Dans l'exemple ci-dessus, la largeur de l'image sera de `250px` si la largeur de la fenêtre d'affichage est inférieure à 1000 pixels. Si la largeur de la fenêtre d'affichage est supérieure à 1000 pixels, la largeur de l'image sera de `25vw`. En effet, `25vw` correspond à 25 % de la largeur de la fenêtre d'affichage.

## Bonnes pratiques avec CSS et l'accessibilité

- **`display: none;`** : utiliser `display: none;` signifie que les lecteurs d'écran et les autres technologies d'assistance ne pourront pas accéder à ce contenu, car il n'est pas inclus dans l'arbre d'accessibilité. Il est donc important d'utiliser cette méthode uniquement lorsque tu veux supprimer complètement le contenu à la fois de la présentation visuelle et de l'accessibilité.
- **`visibility: hidden;`** : cette propriété et cette valeur masquent le contenu visuellement, mais le gardent dans le flux du document, ce qui signifie qu'il occupe toujours de l'espace sur la page. Ces éléments ne seront également plus lus par les lecteurs d'écran, car ils auront été retirés de l'arbre d'accessibilité.
- **Classe CSS `.sr-only`** : c'est une technique courante utilisée pour masquer visuellement du contenu tout en le gardant accessible aux lecteurs d'écran.

:::interactive_editor

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <span>This span element can be read</span>
  <span class="sr-only">This span element can only be read by screen readers</span>
  <p style="display: none">
    This text is hidden with display: none and is NOT accessible to screen readers.
  </p>
  <p style="visibility: hidden">
    This text is hidden with visibility: hidden and is NOT read by screen readers,
    but it still takes up space.
  </p>
</body>

```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

```

:::

## Masquer du contenu avec des attributs HTML

- **Attribut `aria-hidden`** : utilisé pour masquer un élément aux personnes qui utilisent une technologie d'assistance comme les lecteurs d'écran. Par exemple, cela peut servir à masquer des images décoratives qui n'apportent aucun contenu significatif.
- **Attribut `hidden`** : cet attribut est pris en charge par la plupart des navigateurs modernes et masque le contenu à la fois visuellement et dans l'arbre d'accessibilité. Il peut être facilement activé ou désactivé avec JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
<p>This is a normal paragraph</p>

```

:::

## Problème d'accessibilité de l'attribut `placeholder`

Utiliser un texte d'espace réservé n'est pas idéal pour l'accessibilité. Trop souvent, les utilisateurs confondent le texte d'espace réservé avec une vraie valeur de champ : ils pensent qu'une valeur est déjà présente dans le champ.
