# --interactive--

En JavaScript, les variables agissent comme des conteneurs pour stocker des données auxquelles tu peux accéder et que tu peux modifier dans tout ton programme.

Tu peux imaginer les variables comme des boîtes qui contiennent des valeurs. Avec les variables, tu peux suivre des choses comme des nombres ou du texte, puis te référer à ces valeurs chaque fois que tu en as besoin dans ton programme.

Une façon de déclarer une variable en JavaScript consiste à utiliser le mot-clé `let`. Tu en apprendras plus sur le mot-clé `let`, ainsi que sur d'autres façons de déclarer des variables, dans les prochaines leçons.

Voici un exemple d'utilisation de `let` pour déclarer une variable appelée `age` :

```js
let age;
```

Pour l'instant, la variable `age` n'a aucune valeur affectée. Si tu essaies de l'utiliser, elle renverra `undefined`, ce qui signifie qu'elle n'a pas de valeur.

Voici un exemple.

**REMARQUE** : `console.log()` est une fonction qui affiche des informations dans la console, une partie de ton navigateur web utilisée pour déboguer du code. Tu en apprendras plus sur `console.log()` dans les prochaines leçons. Les symboles `//` servent aussi à ajouter des commentaires dans ton code. Les commentaires sont des notes pour toi-même ou pour d'autres programmeurs, et elles sont ignorées quand le code s'exécute.

:::interactive_editor

```js
let age;
console.log(age); // undefined
```

:::

Pour affecter une valeur à une variable, tu dois utiliser l'opérateur d'affectation comme ceci :

```js
let age = 25;
```

Maintenant, quand tu utilises la variable `age`, elle renvoie la valeur `25`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

L'opérateur d'affectation ressemble au signe égal (`=`), mais il ne vérifie pas l'égalité. Tu apprendras les bons opérateurs pour vérifier l'égalité dans de prochaines leçons.

L'opérateur d'affectation sert à affecter une valeur à une variable. Ce processus d'affectation d'une valeur à une variable s'appelle l'initialisation.

Un avantage du mot-clé `let` pour déclarer des variables est que tu peux leur réaffecter des valeurs. En programmation, la réaffectation signifie donner une nouvelle valeur à une variable qui en a déjà une.

Voici un exemple de réaffectation de la valeur de la variable `age`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

Maintenant, la variable `age` contient la valeur `30`. Remarque que le mot-clé `let` n'était pas nécessaire une deuxième fois, car la variable `age` avait déjà été déclarée. Il n'est donc pas nécessaire de la déclarer une seconde fois.

Quand tu utilises la réaffectation, tu n'as besoin de référencer que le nom de la variable. La réaffectation est utile parce qu'elle te permet de mettre à jour et de changer la valeur stockée dans une variable pendant l'exécution de ton programme. Un bon exemple serait la mise à jour des points dans un jeu.

Nommer les variables peut sembler simple, mais il existe quelques règles et bonnes pratiques pour garantir que ton code soit lisible et fonctionnel.

Les noms de tes variables doivent décrire ce que les données représentent. Par exemple, au lieu d'utiliser un nom comme `x`, un nom plus descriptif comme `age` ou `points` rend ton code plus facile à comprendre.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

Les variables en JavaScript doivent commencer par une lettre, un tiret bas (`_`) ou un signe dollar (`$`). Elles ne peuvent pas commencer par un nombre.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

Les noms de variables sont sensibles à la casse, ce qui signifie que le mot `age` entièrement en minuscules et le mot `Age` avec un `A` majuscule sont considérés comme deux variables différentes.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

C'est pourquoi il est important de garder une convention de nommage cohérente comme le camelCase. Le camelCase consiste à écrire le premier mot entièrement en minuscules, puis à commencer chaque mot suivant par une majuscule.

Voici un exemple d'utilisation de la convention camelCase pour une variable :

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

Certains mots-clés en JavaScript ne peuvent pas être utilisés comme noms de variables, comme `let`, `const`, `function` ou `return`, car ils sont réservés au langage lui-même.

Tu devrais aussi éviter les caractères spéciaux comme les points d'exclamation (`!`) ou les symboles arobase (`@`) dans tes noms de variables. Il vaut mieux garder des noms de variables lisibles en utilisant des lettres, des nombres, des tirets bas ou des signes dollar.

En suivant ces règles, ton code sera plus propre et plus facile à gérer à mesure qu'il gagne en complexité.
