# --interactive--

Quand tu travailles avec JavaScript, tu déclares souvent des variables pour stocker des données que tu prévois d'utiliser dans ton programme.

En JavaScript moderne, `let` et `const` sont les façons recommandées de déclarer des variables, mais elles diffèrent dans leur manière de gérer l'affectation et la réaffectation des valeurs.

Dans cette leçon, nous allons voir en quoi `let` et `const` diffèrent pour la déclaration, l'affectation et la réaffectation de variables.

Le mot-clé `let` te permet de déclarer des variables qui peuvent être mises à jour ou réaffectées plus tard. Tu peux voir `let` comme un conteneur flexible : une fois qu'une valeur y est stockée, tu peux la modifier selon les besoins dans ton programme.

Voici un exemple de déclaration et d'affectation d'une variable avec `let` :

```js
let score = 10;
```

Dans ce cas, la variable `score` est déclarée et reçoit la valeur `10`. Si tu veux mettre cette valeur à jour plus tard, tu peux le faire facilement :

:::interactive_editor

```js
let score = 10;
console.log(score); // 10
score = 20;
console.log(score); // 20
```

:::

Maintenant, `score` contient la valeur `20`. Cela rend `let` particulièrement utile quand tu sais que la valeur d'une variable changera pendant l'exécution de ton programme.

À l'inverse, `const` sert à déclarer des variables constantes. Une fois que tu affectes une valeur à une variable déclarée avec `const`, tu ne peux plus la réaffecter.

Cela rend `const` idéal pour les valeurs que tu ne veux pas modifier accidentellement pendant l'exécution de ton programme.

Voici un exemple de déclaration et d'affectation d'une variable avec `const` :

:::interactive_editor

```js
const maxScore = 100;
console.log(maxScore); // 100
```

:::

Une fois que `maxScore` a reçu la valeur `100`, il ne peut plus être modifié :

```js
maxScore = 200; // This will result in an error
```

Essayer de réaffecter une valeur à une variable `const` déclenchera une erreur dans ta console JavaScript, car les variables `const` sont immuables une fois qu'elles ont reçu une valeur.

Tu peux déclarer une variable `let` sans lui affecter immédiatement une valeur, puis lui affecter une valeur plus tard :

:::interactive_editor

```js
let age;
console.log(age); // undefined
age = 25;
console.log(age); // 25
```

:::

Les variables déclarées avec `const` doivent recevoir une valeur au moment de leur déclaration. Si tu essaies de déclarer une variable `const` sans lui affecter une valeur, tu obtiendras une erreur :

```js
const age; // Error: Missing initializer in const declaration
```

Tu devrais utiliser `let` quand tu dois déclarer des variables qui seront réaffectées plus tard. Par exemple, pour suivre un score qui change ou mettre à jour une valeur au fil du temps dans ton programme.

Utilise `const` quand tu veux déclarer des variables qui doivent rester constantes, comme des valeurs de configuration ou des paramètres qui ne doivent pas être modifiés accidentellement.

Tu peux aussi utiliser le mot-clé `var`, mais il n'est plus vraiment recommandé. `var` ressemble un peu à `let`, sauf qu'il a une portée plus large, ce qui est plus susceptible de causer des problèmes dans ton programme.
