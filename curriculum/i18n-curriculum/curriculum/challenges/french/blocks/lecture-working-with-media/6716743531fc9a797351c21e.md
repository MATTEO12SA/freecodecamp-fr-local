# --interactive--

Un élément remplacé est un élément dont le contenu est déterminé par une ressource externe plutôt que par le CSS lui-même. CSS, ou feuilles de style en cascade, sert à ajouter des styles à une page web. Les éléments `image`, `iframe` et `video` sont des exemples courants d'éléments remplacés.

Avec les éléments remplacés, tu peux contrôler la position ou la mise en page d'un élément. Mais ton CSS ne peut pas modifier directement le contenu de cet élément. Cela peut être plus simple à comprendre avec quelques exemples. Regarde l'élément image, qui intègre une image dans ta page web :

```html
<img src="example-img-url" alt="Descriptive text goes here">
```

L'élément lui-même est remplacé par l'objet externe : l'image. Ton CSS peut contrôler des choses comme le positionnement de l'image ou lui appliquer un filtre, mais tu ne peux pas modifier l'image elle-même. Un exemple plus complet serait l'élément `iframe`, qui intègre un site externe dans ta page web.

Voici un exemple d'utilisation de l'élément `iframe` pour une vidéo YouTube populaire de la chaîne freeCodeCamp. Si tu veux voir une autre vidéo dans la fenêtre d'aperçu, change la valeur de l'attribut `src` avec la vidéo de ton choix. Pour voir les aperçus, tu dois activer l'éditeur interactif.

**NOTE** : ne t'inquiète pas pour les attributs supplémentaires mentionnés dans l'exemple interactif, comme `referrerpolicy` et `allowfullscreen`. Tu en apprendras plus sur les éléments `iframe` dans un futur atelier.

:::interactive_editor

```html
<iframe width="400" height="200" src="https://www.youtube.com/embed/u43gJJrVa1I?si=BoDW_puFsy8OEr_Z" title="Professional Cloud Architect Certification Course – Pass the Exam! (YouTube video)" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen></iframe>
```

:::

Un autre exemple courant d'utilisation de l'élément `iframe` consiste à intégrer des cartes dans la page. Voici un exemple de carte intégrée.

Active l'éditeur interactif et essaie de manipuler la carte en zoomant et dézoomant.

:::interactive_editor

```html
<iframe
  title="Map of the Royal Observatory, Greenwich, London"
  width="300"
  height="200"
  src="https://www.openstreetmap.org/export/embed.html?bbox=-0.004017949104309083%2C51.47612752641776%2C0.00030577182769775396%2C51.478569861898606&amp;layer=mapnik">
</iframe>
```

:::

L'élément lui-même est remplacé par l'objet externe : le site. Ton CSS peut changer le positionnement du site intégré, mais tu ne peux pas modifier le contenu de ce site. Pour aller un peu plus loin, si le site intégré contient un élément `h1`, ton CSS ne pourra pas styliser cet élément `h1`. Tu ne peux pas changer sa taille, sa couleur de police, etc.

Il existe d'autres éléments remplacés, comme `video` et `embed`. Certains éléments se comportent aussi comme des éléments remplacés dans des circonstances précises. Voici un exemple d'élément `input` avec l'attribut `type` défini sur `image` :

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

Ce type d'`input` est considéré comme un élément remplacé, mais d'autres types d'`input`, comme `text` ou `email`, ne sont pas des éléments remplacés.
