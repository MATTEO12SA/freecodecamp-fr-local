# --interactive--

Un CSS reset est une feuille de style qui supprime tout ou partie du formatage par défaut que les navigateurs web appliquent aux éléments HTML. Par exemple, tu as peut-être remarqué que certains éléments, comme les paragraphes et les titres, ont déjà des margins par défaut, même si tu ne les définis pas explicitement dans tes feuilles de style personnalisées.

Tu verras aussi cela avec diverses propriétés sur une grande variété d'éléments HTML. Différents navigateurs peuvent également appliquer différents styles par défaut aux éléments HTML. Les styles par défaut dans Google Chrome peuvent ne pas être exactement les mêmes que dans Mozilla Firefox ou Microsoft Edge. Cela peut produire des styles incohérents d'un navigateur à l'autre, ce que tu devrais éviter autant que possible.

Pour gérer cela, un CSS reset supprime les styles par défaut. En supprimant tout ou partie des styles par défaut, tu peux obtenir une base cohérente pour ton design et réduire les incohérences potentielles entre les navigateurs et les appareils. Supprimer les styles par défaut peut aussi rendre le processus de stylisation plus facile, car tu ne verras que les styles que tu as appliqués explicitement dans tes feuilles de style personnalisées. Il existe deux grandes approches pour les CSS resets : tu peux soit définir des CSS resets personnalisés, soit utiliser des CSS resets tiers.

Les CSS resets personnalisés sont des feuilles de style que tu crées à partir de zéro pour répondre aux besoins de ton projet. De cette façon, tu peux contrôler les styles précis qui seront réinitialisés, avec beaucoup de flexibilité. Cependant, tu dois aussi investir du temps pour développer et maintenir les feuilles de style. Voici un exemple de règle CSS très courante pour réinitialiser la `margin` et le `padding` de tous les éléments HTML :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

Elle est généralement écrite en haut de la feuille de style CSS. Le sélecteur astérisque est un sélecteur joker qui correspond à tous les éléments HTML, donc ils auront une margin par défaut de zéro et un padding par défaut de zéro sur les quatre côtés. Cela te donne un point de départ, puis tu peux les personnaliser dans des sélecteurs CSS plus spécifiques plus bas dans la feuille de style.

Tu peux utiliser cette approche pour sélectionner n'importe quel élément HTML et réinitialiser ses styles par défaut. Crée simplement une feuille de style personnalisée et utilise les sélecteurs CSS appropriés pour cibler les éléments et définir les styles.

Mais ce processus peut prendre beaucoup de temps. Si tu veux gagner du temps, tu peux aussi utiliser un CSS reset tiers. Ces feuilles de style sont déjà préconstruites, donc tu peux simplement les télécharger et les ajouter directement à ton projet. Un excellent exemple de CSS reset tiers est `Normalize.css`. Cette feuille de style normalise les styles pour une grande variété d'éléments HTML tout en conservant certains styles par défaut utiles, surtout ceux qui sont importants pour l'accessibilité. Elle corrige aussi des bugs courants et des incohérences de style.

Une autre option est `sanitize.css`. C'est une bibliothèque CSS que tu peux utiliser pour garantir que les styles par défaut seront cohérents sur tous les principaux navigateurs modernes. Cette bibliothèque est développée aux côtés de `Normalize.css`, donc elles évoluent ensemble. Elle propose aussi des feuilles de style individuelles que tu peux télécharger pour des besoins spécifiques, comme la normalisation des formulaires et de la typographie.

Il existe beaucoup d'options, mais tu devrais choisir celles qui correspondent le mieux aux besoins de ton projet. Tu peux aussi combiner les deux approches en utilisant des CSS resets tiers avec des resets personnalisés.

Quand tu travailles avec des CSS resets, il est aussi important de prendre l'accessibilité en compte. Ton application web devrait être accessible à tout le monde. Tu ne devrais donc pas réinitialiser des styles qui pourraient être utiles aux lecteurs d'écran ou à d'autres technologies d'assistance. Tu devrais aussi tenir compte de l'impact que ces feuilles de style supplémentaires peuvent avoir sur les performances de ton application, car elles doivent être téléchargées avant que les styles personnalisés soient appliqués.

En supprimant les styles par défaut, les CSS resets te donnent un point de départ vierge pour implémenter ton design. Cela produit une expérience utilisateur plus uniforme et cohérente entre les navigateurs et entre les appareils.
