# --interactive--

Les styles par défaut des liens jouent un rôle essentiel dans l'amélioration de l'utilisabilité et de l'accessibilité sur le web.

Ces styles, généralement bleus pour les liens non visités et violets pour les liens visités, sont devenus un standard que les utilisateurs s'attendent à voir et sur lequel ils comptent quand ils naviguent sur des sites web.

Le but principal des styles par défaut des liens est de fournir des indices visuels clairs qui aident les utilisateurs à distinguer entre les éléments interactifs et non interactifs sur une page web.

Cette distinction est fondamentale pour créer une expérience de navigation intuitive et conviviale.

Considérons les styles par défaut de base pour les liens :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: underline;
}

a:visited {
  color: purple;
}
```

:::

Ces styles servent plusieurs fonctions importantes.

D'abord, la couleur bleue pour les liens non visités se distingue de la plupart des couleurs d'arrière-plan et de texte, rendant les liens facilement identifiables. Ce contraste est essentiel pour que les utilisateurs scannent rapidement une page et trouvent les éléments de navigation ou les informations importantes.

Le soulignement souligne davantage que le texte est cliquable, fournissant un indice visuel supplémentaire. C'est particulièrement utile pour les utilisateurs qui peuvent être daltoniens ou avoir des difficultés à distinguer les couleurs.

Le changement de couleur pour les liens visités (généralement en violet) aide les utilisateurs à garder une trace de là où ils sont allés. Cette fonctionnalité est inestimable pour naviguer sur de grands sites web ou faire des recherches, car elle empêche les utilisateurs de revisiter par inadvertance les mêmes pages.

Considère cet exemple HTML :

:::interactive_editor

```html
<p>Learn more about <a href="https://www.example.com/cats">cats</a> and <a href="https://www.example.com/dogs">dogs</a>.</p>
```

:::

Sans CSS personnalisé, la plupart des navigateurs afficheront ces liens en bleu avec un soulignement. Après avoir cliqué sur l'un des liens, sa couleur passera au violet, fournissant un retour immédiat à l'utilisateur sur son historique de navigation.

Bien qu'il soit courant pour les designers de modifier ces styles par défaut pour correspondre à l'esthétique d'un site web, il est essentiel de maintenir les principes de base derrière eux.

Si tu choisis de changer les styles par défaut, assure-toi que les liens sont toujours clairement distinguables du texte normal, qu'il y a une différence visible entre les liens visités et non visités, et que les couleurs choisies fournissent un contraste suffisant avec l'arrière-plan.

Par exemple, tu pourrais utiliser un style personnalisé comme ceci :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:link {
  color: blue;
  text-decoration: none;
  border-bottom: 1px solid blue;
}

a:visited {
  color: purple;
  border-bottom: 1px solid purple;
}
```

:::

Cela maintient le schéma de couleurs bleu et violet tout en remplaçant le soulignement par une bordure inférieure pour un look plus moderne.

Il est aussi important de considérer les différents états des liens. En plus des états par défaut et visités, les liens ont généralement les états hover et active :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<a href="/">Example link</a>
```

```css
a:hover {
  color: red;
}

a:active {
  color: darkorange;
}
```

:::

Ces états fournissent un retour immédiat aux utilisateurs quand ils interagissent avec les liens, améliorant l'utilisabilité globale du site.

En conclusion, bien qu'il soit possible de personnaliser les styles de liens, les principes derrière les styles par défaut devraient être maintenus.

Ils jouent un rôle essentiel dans la création d'une expérience web utilisable et accessible, aidant les utilisateurs à naviguer et interagir efficacement avec le contenu.

Priorise toujours la clarté et l'expérience utilisateur quand tu conçois des styles de liens pour tes sites web.
