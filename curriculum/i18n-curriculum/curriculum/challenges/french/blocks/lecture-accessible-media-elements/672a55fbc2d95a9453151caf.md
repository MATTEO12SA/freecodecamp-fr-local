# --interactive--

Beaucoup d'utilisateurs s'appuient sur des claviers plutôt que sur des souris à cause de handicaps physiques, de troubles musculo-squelettiques liés aux mouvements répétitifs, ou par préférence personnelle. Cela inclut les utilisateurs de lecteurs d'écran et ceux qui peuvent ne pas avoir de souris fonctionnelle.
L'accessibilité au clavier garantit que ces utilisateurs peuvent naviguer dans les applications web efficacement sans barrières.

Voyons quelques techniques pratiques que tu peux employer pour rendre les applications web accessibles au clavier.

Beaucoup d'utilisateurs s'appuient sur la touche `Tab` pour passer d'un élément interactif à un autre sur une page web. Par défaut, les navigateurs permettent aux utilisateurs de tabuler à travers des éléments comme les liens, les boutons et les champs de formulaire dans l'ordre où ils apparaissent dans le HTML. C'est ce qu'on appelle l'ordre de tabulation naturel.

Parfois, tu peux vouloir ajuster quels éléments peuvent recevoir le focus ou changer leur ordre de focus. L'attribut `tabindex` te permet de faire cela.

Voici la syntaxe de base :

```html
<element tabindex="number">Element Text</element>
```

La valeur de `tabindex` détermine comment l'élément se comporte dans la navigation au clavier :

`tabindex="0"` ajoute l'élément à l'ordre de tabulation naturel. 

La tabulation déplacera le focus du `button` au `div`, puis au lien, en suivant leur ordre dans le HTML.

Clique n'importe où dans l'espace blanc de la fenêtre d'aperçu. Puis utilise la touche `Tab` pour voir le focus se déplacer entre les différents éléments. Pour interagir avec l'exemple, tu devras activer l'éditeur interactif.

:::interactive_editor

```html
<button>First</button>
<div tabindex="0">Second</div>
<a href="#">Third</a>
```

:::

`tabindex="-1"` rend un élément focusable par programme. C'est utile pour gérer le focus dans des éléments qui ne sont pas normalement focusables, comme les titres, les conteneurs, les boîtes de dialogue ou les messages d'erreur :

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

Dans cet exemple, le paragraphe ne fait pas partie de l'ordre de tabulation normal, donc les utilisateurs ne peuvent pas l'atteindre en appuyant sur la touche `Tab`. Cependant, si tu mets le focus sur cet élément via un script, le message sera porté à l'attention de l'utilisateur. Tu en apprendras plus sur cette technique dans les leçons JavaScript.

Quand le `tabindex` est supérieur à `0`, il définit un ordre de tabulation personnalisé. Les éléments avec des valeurs positives plus basses reçoivent le focus en premier.

Dans cet exemple, la tabulation mettra d'abord le focus sur l'`input` avec `tabindex="1"`, puis `2`, puis `3`, quel que soit leur ordre dans le HTML.

```html
<input tabindex="2">
<input tabindex="1">
<input tabindex="3">
```

Des valeurs positives personnalisées sont parfois utilisées dans des widgets complexes, comme une barre d'outils où tu veux un ordre de navigation spécifique. Cependant, cette approche est déconseillée car elle peut rendre la navigation confuse et difficile à maintenir, surtout quand la page grandit ou change.

`accesskey` est un autre attribut que tu peux utiliser pour rendre ton projet web accessible au clavier. Il te permet de définir une touche qui met le focus sur ou active un élément particulier.

Voici un exemple interactif que tu peux essayer en suivant les suggestions ci-dessous (active d'abord l'éditeur interactif) :

- `accesskey="s"` assigne la touche `S` au bouton `Save`. Sur la plupart des navigateurs, appuyer sur `ALT + S` (sur Windows) et `CTRL + Option + S` (sur Mac) activera ce bouton.
- `accesskey="c"` définit la touche `C` pour le bouton `Cancel`, permettant aux utilisateurs de l'activer en utilisant `ALT + C` (Windows) et `CTRL + Option + C` (Mac).
- `accesskey="h"` assigne la touche `H` au lien `Home`, permettant aux utilisateurs de naviguer vers la page d'accueil en utilisant `ALT + H` (Windows) et `CTRL + Option + H` (Mac).

Note que la combinaison exacte de touches pour activer l'`accesskey` peut varier selon le navigateur et le système d'exploitation. C'est typiquement `ALT + Touche spécifiée` sur Windows et `CTRL + Option + Touche spécifiée` sur Mac.

L'attribut `accesskey` peut être problématique car il peut entrer en conflit avec des raccourcis existants ou se comporter de manière incohérente selon les appareils et les claviers, et n'est souvent pas intuitif à découvrir ou retenir pour les utilisateurs.

Si tu choisis d'utiliser `accesskey`, cela devrait être fait avec des indications claires aux utilisateurs, mais dans la plupart des cas il est préférable d'éviter `accesskey` dans les sites web et applications web.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

Une autre façon de rendre le clavier accessible dans tes applications est de t'assurer de fournir des indicateurs de focus clairs. Si tu trouves que l'indicateur de focus par défaut du navigateur n'est pas suffisant, tu peux le remplacer en ciblant l'état de focus de l'élément.

Voici un exemple de stylisation de l'état de focus pour un élément `button`. Clique n'importe où dans l'espace blanc de la fenêtre d'aperçu puis appuie sur la touche `Tab` pour mettre le focus sur le bouton. Pour interagir avec l'exemple, tu devras activer l'éditeur interactif.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">

<button>Example button</button>
```

```css
button:focus {
  outline: 2px solid #005fcc;
}
```

:::

La propriété `outline` est utilisée pour définir le contour autour de l'élément. Cet exemple définit le contour à une ligne bleue continue de 2 pixels d'épaisseur. L'indicateur de focus doit être stylisé d'une manière qui rend évident quel élément a actuellement le focus. Pour être accessible, l'indicateur doit avoir un contraste de couleur minimum d'au moins 3:1 avec la couleur de fond qu'il couvre. 

Tu devrais aussi éviter les pièges au clavier, qui se produisent quand un utilisateur ne peut pas déplacer le focus en dehors d'un certain élément dans des composants comme les modales et les popups.
