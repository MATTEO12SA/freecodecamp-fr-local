# --description--

Le texte alternatif, souvent abrégé en texte `alt`, est une brève description textuelle d'une image. Il fournit des informations essentielles sur l'image pour les utilisateurs qui ne peuvent pas la voir, comme les personnes qui utilisent des lecteurs d'écran et d'autres technologies d'assistance.

Il est essentiel pour rendre les sites web accessibles aux personnes avec des handicaps visuels.

Le texte alternatif est aussi utilisé par les moteurs de recherche pour comprendre les images. Certains navigateurs peuvent aussi l'afficher quand une image n'est pas chargée correctement. Cela peut arriver quand le fichier image est manquant ou quand l'utilisateur a des problèmes de connexion. Donc c'est utile pour de nombreux usages différents.

Une personne en situation de handicap visuel ne pourra pas savoir ce qu'il y a dans l'image à moins que ce soit décrit dans le texte alt. Un exemple de mauvais texte alt pour une image d'un chiot serait « Un chiot mignon. »

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/puppy.png" alt="A cute puppy." />

Ce texte n'est pas assez spécifique pour transmettre les détails importants de l'image, par exemple : à quoi ressemble le chiot ? Où est-il ? Y a-t-il des objets importants autour de lui ?

Améliorons cela. Un exemple de bon texte alternatif serait : « Un chiot noir et blanc avec un collier orange est couché sur le ventre dans le sable, regardant sur le côté. Une balle orange vif repose près de ses pattes avant. »

Voici comment cela apparaît en HTML avec un élément image, l'attribut `alt`, et une description plus détaillée :

```html
<img src="puppy.png" alt="A black and white puppy with an orange collar lies on its belly in the sand, looking off to the side. A bright orange ball rests near its front paws." />
```

Il est important de noter qu'il n'y a pas une seule bonne manière d'écrire un texte alt pour une image. Ce que tu inclus dans ta description dépendra du contexte dans lequel l'image est utilisée. Par exemple, si la photo du chiot était sur un site web sur les races de chiens, tu voudrais peut-être inclure plus de détails sur la description physique du chiot et peut-être ignorer le fait que le chiot est à la plage en train de jouer avec une balle orange. Au final, le texte `alt` que tu utilises pour une image doit refléter le but principal de l'inclusion de l'image sur la page, et les informations que tu fournis doivent donner aux personnes qui ne peuvent pas voir l'image les détails nécessaires pour comprendre ce but.

Voici un autre exemple d'un beau complexe touristique tropical. Décrivons-le.

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/resort.jpg" alt="Resort.">

Un exemple de mauvais texte `alt` pour cette image serait « Complexe touristique. »

C'est trop court et ça ne fournit pas assez d'informations sur l'image. Pour améliorer cette description, tu pourrais l'étendre pour inclure les éléments les plus importants de l'image :

« Complexe touristique tropical avec une piscine entourée de palmiers et de bungalows. »

Tu peux utiliser l'attribut alt en HTML comme ceci :

```html
<img src="resort.png" alt="Tropical resort featuring a swimming pool surrounded by palm trees and bungalows." />
```

Maintenant que tu sais ce qui fait un bon et un mauvais texte `alt`, voyons quelques bonnes pratiques.

- Tu devrais essayer de garder le texte `alt` court. Il doit être assez détaillé pour comprendre l'image mais pas si long qu'il devient confus.
- Tu ne devrais pas essayer de décrire chaque petit détail. Concentre-toi sur les aspects les plus importants de l'image.
- En général, tu n'as pas besoin de commencer par « image de » ou « photo de ». Tu peux juste commencer la description directement.
- Aussi, s'il y a du texte similaire autour de l'image, tu n'as pas besoin de l'écrire à nouveau.
- Il est généralement recommandé de terminer le texte alt par un point pour la cohérence.
- Si l'image est un lien vers une autre page, au lieu de décrire l'image elle-même, le texte `alt` doit décrire ce qui se passera si les utilisateurs cliquent dessus.

Par exemple, si ton site web a une icône de flèche droite qui amène l'utilisateur à la page suivante, au lieu d'écrire un texte alt qui dit seulement « flèche droite », comme dans cet exemple, où tu peux voir l'attribut alt avec cette description :

```html
<a href="about.html">
  <img src="arrow-right.png" alt="Right arrow." />
</a>
```

Tu devrais écrire quelque chose comme ça à la place, où le texte `alt` décrit ce qui se passera si les utilisateurs cliquent sur l'image. Ils iront à la page suivante.

```html
<a href="about.html">
  <img src="arrow-right.png" alt="Go to next page." />
</a>
```

Seules les images qui transmettent des informations importantes devraient avoir un texte `alt`. Si une image n'est utilisée qu'à des fins décoratives, elle devrait avoir un texte `alt` `null` (vide), pour qu'elle puisse être ignorée par les lecteurs d'écran et autres technologies d'assistance.

Voici un exemple où l'attribut `alt` est vide :

```html
<img src="decorative_image.jpg" alt="" />
```

Chaque image de ton site web devrait avoir un attribut `alt`, même s'il est vide. Si tu omets complètement l'attribut `alt`, certains lecteurs d'écran liront le nom du fichier à la place, ce qui peut être perturbant pour les personnes qui utilisent des technologies d'assistance, donc ce n'est pas recommandé.

Enfin, avant que ton site web soit en ligne, tu devrais tester soigneusement si les lecteurs d'écran peuvent lire le texte `alt` correctement.

Écrire un texte `alt` efficace est essentiel pour créer un contenu web accessible. En tant que développeur web, en fournissant des descriptions claires de tes images, tu peux t'assurer que tout le monde peut interagir avec ton site dans une expérience en ligne inclusive.
