# --description--

Quand tu commences à créer tes propres applications web, tu dois faire attention à la façon dont tu nommes tes fichiers.

Tu veux nommer tes fichiers de manière facile à comprendre et à maintenir.

Regardons des exemples de bons et mauvais noms de fichiers HTML. Note que l'extension `.html` dans ces exemples indique qu'il s'agit d'un fichier HTML.

Voici un exemple de mauvais nom de fichier :

```md
index1234.html
```

Rien qu'avec ce nom, on ne sait pas de quoi parle le fichier. Ça pourrait être pour une page de produits, une page de blog, ou tout autre type de page, mais on ne le sait pas à partir du nom de fichier.

Voici un exemple de meilleur nom pour un fichier HTML :

```md
about-us.html
```

En utilisant un nom plus descriptif comme `about-us.html`, on comprend clairement de quoi parle le fichier.

Tu remarqueras aussi qu'il n'y a pas d'espaces dans ce nom de fichier. À la place, tu utilises un tiret, `-`, pour séparer les mots.

Parfois, tu verras un nom de fichier appelé `index.html`. C'est un nom de fichier spécial utilisé pour représenter la page principale d'un site web. Quand tu visites un site web, le fichier `index.html` est le premier fichier chargé.

Une autre considération importante quand on nomme des fichiers, c'est l'utilisation de caractères spéciaux.

Mélanger des caractères spéciaux, des chiffres et des lettres peut rendre difficile la compréhension de ce que contient le fichier.

Voici un exemple de mauvais nom de fichier :

```md
file-1!@.html
```

Ce nom semble inutilement compliqué et ne nous donne aucune information sur le contenu du fichier.

Donc c'est mieux de t'en tenir uniquement aux lettres et aux tirets dans tes noms de fichiers.

Au fur et à mesure que tu apprendras d'autres langages, tu verras qu'il y a aussi des conventions courantes pour nommer les fichiers dans ces langages.

Dans ces situations, c'est toujours mieux de consulter la documentation officielle ou d'autres ressources pour comprendre les bonnes pratiques de nommage de fichiers dans ce langage.

Tu seras aussi parfois dans des situations où tu travailles en équipe et où tu devras suivre les conventions que l'équipe a établies pour nommer les fichiers.

Consulte toujours le guide de style de l'équipe pour t'assurer que tes noms de fichiers sont cohérents avec le reste de l'équipe.
