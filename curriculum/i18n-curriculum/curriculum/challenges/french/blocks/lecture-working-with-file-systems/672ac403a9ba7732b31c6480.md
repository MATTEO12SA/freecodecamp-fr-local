# --description--

Les fichiers sont classés selon leur contenu et leur structure. Le type de fichier détermine la façon dont un fichier est ouvert et traité par les programmes informatiques.

On identifie les types de fichiers grâce à leurs extensions. Une extension de fichier est un suffixe de trois ou quatre lettres à la fin du nom du fichier. Les trois principaux types de fichiers que tu trouveras dans les applications web sont HTML, CSS et JavaScript.

Les fichiers HTML contiennent la structure et le contenu de l'application web. Ils ont une extension de fichier `.html`.

Les fichiers CSS définissent les styles. Ils ont une extension de fichier `.css`.

Et les fichiers JavaScript ajoutent des fonctionnalités plus avancées et de l'interactivité. Ils ont une extension de fichier `.js`.

À mesure que tu développes des applications web, tu devras aussi inclure des images. Voici quelques-uns des types de fichiers d'image les plus utilisés.

JPEG, qui signifie « Joint Photographic Experts Group », est connu pour sa compression efficace. C'est super pour les photographies et les images. Cependant, la compression est avec perte, ce qui signifie qu'une partie de la qualité de l'image est perdue pour réduire la taille du fichier.

PNG, qui signifie « Portable Network Graphics », préserve la qualité de l'image, même après compression. Ça donne des fichiers plus volumineux. PNG prend en charge la transparence et c'est super pour les images avec des bords nets, comme les logos et les icônes.

Le format d'image GIF prend en charge à la fois l'animation et la transparence, mais il a une palette de couleurs limitée. GIF signifie « Graphics Interchange Format ».

Et enfin, on a SVG, qui signifie « Scalable Vector Graphics ». C'est un format d'image vectoriel. Ces images peuvent être redimensionnées autant que nécessaire sans perdre en qualité.

Super. Maintenant que tu connais les formats d'image, voyons quelques formats vidéo et audio.

MP3 est un format audio connu pour sa compression efficace. C'est un format avec perte, donc certaines données audio sont perdues pendant la compression pour rendre ces fichiers plus petits.

MP4 est l'un des formats vidéo les plus courants. Il offre une bonne compression et prend en charge plusieurs codecs audio et vidéo, sous-titres et métadonnées.

MOV, le format de fichier multimédia QuickTime développé par Apple, est principalement associé à QuickTime Player.

D'autres formats audio et vidéo populaires sont WAV, un format audio sans perte qui conserve la qualité originale de l'audio, et WebM, un format vidéo open-source de haute qualité.

Si tu as besoin de personnaliser les polices de ton application web, tu travailleras aussi avec des formats de police. Voici trois des plus utilisés.

TTF, qui signifie « TrueType Font », est un format de police compatible avec différents systèmes d'exploitation.

WOFF est un format de police moderne, spécifiquement conçu pour le développement web. Il signifie « Web Open Font Format ». Ces fichiers sont plus petits parce qu'ils ont une meilleure compression. Ils peuvent aussi stocker des métadonnées, y compris des informations de licence.

Et WOFF2 est le successeur de WOFF avec une compression et des performances encore plus efficaces.

Tu peux aussi créer des fichiers d'archive si tu as besoin de regrouper plusieurs dossiers et fichiers. Le format d'archive le plus utilisé est ZIP. ZIP offre une compression sans perte, donc aucune donnée n'est perdue pendant le processus. Il est largement pris en charge par les systèmes d'exploitation et les applications logicielles.

Et enfin, parlons de documentation. Dans les applications web, tu trouveras généralement un fichier appelé `README` qui contient des informations sur l'application, comme comment l'utiliser, comment l'installer, sa licence et comment contribuer.

Ils sont généralement écrits dans un format de fichier appelé Markdown. Avec Markdown, tu peux créer des documents structurés avec des titres, sous-titres, liens, images, listes et plus. Les fichiers Markdown ont une extension `.md` ou `.markdown`.

Tu peux trouver ici le fichier `README` du dépôt GitHub de freeCodeCamp : https://github.com/freeCodeCamp/freeCodeCamp/blob/main/README.md

Tu peux créer cette structure et ce format détaillés en utilisant Markdown.

Tu vas certainement lire et écrire beaucoup de fichiers `README` tout au long de ta carrière. Apprendre ces types de fichiers courants est essentiel pour le développement web.

Des blocs de construction de base que sont HTML, CSS et JavaScript aux formats d'image, vidéo et audio, chaque type de fichier joue un rôle important dans la création d'applications web interactives.
