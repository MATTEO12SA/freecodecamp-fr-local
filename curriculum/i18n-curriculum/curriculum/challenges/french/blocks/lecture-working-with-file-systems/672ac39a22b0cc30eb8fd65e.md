# --description--

Quand tu construis des applications web, il est important de réfléchir à l'organisation de tes fichiers et dossiers. Ça aidera à garder ton code organisé et plus facile à maintenir.

Regardons un exemple de structure de dossiers pour un projet HTML et CSS :

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

En haut de l'exemple, il y a un point unique, qui représente le répertoire racine.

Le répertoire racine est le répertoire de plus haut niveau dans un système de fichiers, il sert de point de départ pour tous les chemins de fichiers et contient tous les autres répertoires et fichiers. Un répertoire est un autre nom pour un dossier.

À l'intérieur du répertoire racine, le premier dossier affiché est le dossier `assets`. Le terme `assets` (ressources) fait référence à tous les fichiers utilisés dans le projet, comme les images, les polices ou d'autres ressources.

Dans cet exemple, il y a deux dossiers à l'intérieur du dossier `assets`, appelés `images` et `fonts`.

Le dossier suivant est le dossier `css`. Parfois, on l'appelle aussi un dossier de feuilles de style. C'est là que tu stockerais tous tes fichiers CSS.

Sous le dossier `css`, il y a les fichiers HTML. Le fichier `index.html` représente la page d'accueil, tandis que `about.html` et `contact.html` représentent les pages À propos et Contact, respectivement.

Enfin, il y a un fichier `README.md`. Les fichiers `README` sont couramment utilisés pour fournir des informations utiles sur le projet, comme ce qu'il fait, comment l'utiliser, et tout autre détail pertinent.

L'extension `.md` signifie Markdown, qui est un langage de balisage léger souvent utilisé pour écrire de la documentation.

En organisant ton projet en ressources, CSS et fichiers HTML, ça devient facile de trouver ce que tu cherches. Ça facilite aussi la contribution d'autres développeurs à ton projet.

Il est important de noter que ce n'est qu'un exemple parmi d'autres de comment organiser tes fichiers et dossiers. Il existe de nombreuses façons différentes de structurer un projet.
