# --interactive--

Il est important de s'assurer que tous les utilisateurs, y compris ceux en situation de handicap, peuvent accéder aux sites web sans problème.

Pour les personnes qui utilisent des lecteurs d'écran, les attributs `aria-label` et `aria-labelledby` fournissent des informations cruciales sur les éléments de la page qui pourraient être peu clairs ou invisibles.

Voyons ce que sont les attributs `aria-label` et `aria-labelledby` et les rôles qu'ils jouent pour rendre le web accessible aux personnes avec des déficiences visuelles et des handicaps associés.

Tu remarqueras que `aria-label` et `aria-labelledby` ont tous deux le préfixe aria. Qu'est-ce que cela signifie ? ARIA signifie Accessible Rich Internet Applications. C'est un ensemble d'attributs préfixés par `aria-`, qui permet aux développeurs de communiquer le rôle des éléments aux technologies d'assistance. L'attribut `aria-label` est un libellé invisible pour les éléments interactifs. Il ajoute à un élément un libellé texte que les lecteurs d'écran peuvent lire.

`aria-label` est particulièrement utile pour les éléments qui n'ont pas de texte visible mais qui doivent quand même être décrits par les lecteurs d'écran. Par exemple, les boutons avec seulement des icônes ont souvent besoin de `aria-label` pour transmettre leur rôle.

Voici un exemple :

```html
<button aria-label="Search">
  <i class="fa-solid fa-magnifying-glass"></i>
</button>
```

Dans ce cas, un lecteur d'écran peut annoncer ce bouton comme `Search, button`, même si le bouton ne contient qu'une icône. L'attribut `aria-label` indique aux lecteurs d'écran quel texte utiliser à la place de l'icône.

Si le bouton contenait le texte "Search" au lieu d'une icône, alors il n'y aurait pas besoin de l'attribut `aria-label`, car le texte fournirait le libellé pour le bouton.

Pour les éléments de saisie, l'attribut `aria-label` fournit directement un libellé s'il n'y a pas de libellé visible associé au champ de saisie.

L'attribut `aria-labelledby` fait exactement la même chose que l'attribut `aria-label`, mais au lieu de définir le texte directement dans l'attribut, tu utilises une référence vers un texte qui existe déjà sur la page. Le texte existant doit avoir un attribut `id`, qui sera utilisé comme valeur de référence dans l'attribut `aria-labelledby`.

Voici un exemple :

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

Dans ce cas, le texte du bouton est utilisé comme libellé pour le champ de saisie de recherche. Les lecteurs d'écran annonceront le champ de saisie comme quelque chose comme `Search, edit`. Si tu décides plus tard de changer le texte du bouton en `Find`, le libellé du champ de saisie sera automatiquement mis à jour vers le nouveau texte puisqu'il référence le texte du bouton.
Combiner plusieurs valeurs d'`id` dans une seule valeur d'attribut `aria-labelledby` est aussi possible. Voici comment ça fonctionne :

:::interactive_editor

```html
<div>
  <span id="volume-label">Volume</span>
  <span id="volume-details">Adjust the volume level</span>
  <input
    type="range"
    min="0"
    max="100"
    value="30"
    aria-labelledby="volume-label volume-details">
</div>
```

:::

Pour le curseur, le lecteur d'écran ira chercher le contenu des éléments `volume-label` et `volume-details` et annoncera `Volume Adjust the volume level`.

Tu as vu que les attributs `aria-label` et `aria-labelledby` aident tous deux les lecteurs d'écran à comprendre ce que font les éléments. Alors, lequel devrais-tu utiliser ? Comme ils fournissent la même fonctionnalité, l'un ou l'autre peut être utilisé, mais il peut y avoir quelques avantages à utiliser `aria-labelledby` plutôt que `aria-label` :

- Si quelqu'un utilise un service de traduction pour traduire le contenu de ta page, le texte dans un attribut `aria-label` peut ne pas toujours être traduit. 
- Utiliser `aria-labelledby` peut aussi aider à éviter un décalage entre le texte du libellé visible et le libellé invisible pour les utilisateurs de lecteurs d'écran, puisque la mise à jour du texte visible mettra automatiquement à jour le libellé invisible. 
- `aria-labelledby` peut rendre beaucoup plus facile la création programmatique de libellés invisibles complexes composés de plusieurs sources de texte.

Une dernière note : n'utilise pas `aria-label` et `aria-labelledby` sur un même élément en même temps. Dans ce cas, le libellé invisible pour les lecteurs d'écran sera toujours déterminé par `aria-labelledby` et `aria-label` sera complètement ignoré.
