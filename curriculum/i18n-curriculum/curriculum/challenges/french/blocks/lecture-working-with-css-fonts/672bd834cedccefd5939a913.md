# --description--

Avant de commencer, tu dois savoir ce qu'est une règle at en CSS. Les règles at sont des instructions données au navigateur. Tu peux les utiliser pour définir différents aspects de la feuille de style, comme les requêtes média, les keyframes, les polices, et plus encore. Tu en apprendras plus sur des concepts comme les requêtes média et les keyframes dans de futures leçons. Cette fois, tu vas découvrir la règle at `@font-face`.

Avec `@font-face`, tu peux définir une police personnalisée en précisant le fichier de police, son format et d'autres propriétés importantes, comme l'épaisseur et le style. Voici la syntaxe de base. Tu peux voir `@font-face` suivi d'un bloc entouré d'accolades :

```css
@font-face {
  /* Descriptors */
}
```

Dans les accolades, tu devras inclure des descripteurs pour personnaliser ta police. Voyons quelques-uns des plus couramment utilisés. Le descripteur `font-family` précise le nom que tu utiliseras dans toute la feuille de style pour faire référence à cette police. Par exemple, imaginons que tu définisses cette règle `@font-face`. Elle contient le descripteur `font-family`, et sa valeur est `MyCustomFont` :

```css
@font-face {
  font-family: "MyCustomFont";
}
```

Dans ce cas, tu devras utiliser ce nom dans ta feuille de style partout où tu veux attribuer cette famille de polices personnalisée à tous les éléments correspondant à un sélecteur CSS. Par exemple, tu l'utiliserais comme valeur de `font-family` :

```css
body {
  font-family: "MyCustomFont";
}
```

Mais pour que la règle at `@font-face` soit valide, tu dois aussi préciser `src`. Elle contient les références aux ressources de police. C'est essentiellement une liste de références externes ou de noms de polices installées localement, séparés par des virgules. Elle peut aussi inclure des indications sur le format et la technologie des ressources de police.

Dans le code ci-dessous, `src` est définie. La valeur est une liste d'URL séparées par des virgules et placées sur des lignes séparées pour améliorer la lisibilité :

```css
@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2"),
    url("path/to/font.woff"),
    url("path/to/font.otf");
}
```

Tu peux appeler la fonction `url()` pour inclure un fichier dans ta feuille de style. Dans ce cas, nous incluons les fichiers de police. Tu devras écrire le chemin du fichier entre parenthèses et entre guillemets, avec l'extension du fichier. Pour améliorer la lisibilité, tu peux écrire chaque ressource sur une ligne différente, mais la dernière doit se terminer par un point-virgule.

Pour chaque ressource de police, tu peux aussi préciser le format. C'est facultatif. C'est une indication pour le navigateur sur le format de la police. Si le format est omis, la ressource sera téléchargée et le format sera détecté après le téléchargement. Si le format est invalide, la ressource ne sera pas téléchargée.

Les formats de police possibles incluent `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` et `woff2`.

Voici un exemple avec des formats de police. Remarque que nous écrivons le format précis entre parenthèses et entre guillemets :

```css
@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype"),
    url("path/to/font.woff") format("woff");
}
```

Dans cet exemple, tu peux voir que nous précisons le format WOFF2, le format OpenType et le format WOFF.

woff signifie « Web Open Font Format ». La différence entre WOFF et WOFF2 est l'algorithme utilisé pour compresser les données. OpenType est un format de polices informatiques extensibles développé par Microsoft et Adobe qui permet aux utilisateurs d'accéder à des fonctionnalités supplémentaires dans une police. Il est largement utilisé sur les principaux systèmes d'exploitation.

En plus de préciser le format, tu peux aussi préciser la technologie de la ressource de police. C'est également facultatif. Voici un exemple où nous précisons la technologie de la deuxième ressource de police.

```css
@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2") format("woff2"),
    url("path/to/font.otf") format("opentype") tech(color-COLRv1),
    url("path/to/font.woff") format("woff");
}
```

Ce sont les fondamentaux de la règle `@font-face`. Avec cette règle at, tu peux préciser le fichier de police pour définir des polices personnalisées adaptées à tes designs uniques.
