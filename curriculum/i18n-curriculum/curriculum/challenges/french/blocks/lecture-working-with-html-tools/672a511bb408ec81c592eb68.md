# --description--

Quand tu construis tes projets, tu rencontreras souvent des problèmes où tes programmes ne fonctionnent pas comme prévu.

Les programmeurs appellent souvent ces problèmes des bugs. Le processus qui consiste à trouver et corriger ces bugs s’appelle le débogage.

Pour déboguer ton code, tu devras utiliser certains outils fournis par ton navigateur.

Deux outils importants sont l’inspecteur DOM et les outils de développement.

L’inspecteur DOM te permet d’inspecter la structure HTML de la page sur laquelle tu te trouves.

DOM signifie Document Object Model. C’est une structure en forme d’arbre qui représente les éléments d’une page. Tu en apprendras plus sur le DOM dans des modules plus avancés.

Les outils de développement te permettent d’inspecter le HTML, le CSS et le JavaScript de la page sur laquelle tu te trouves.

Regardons un exemple HTML qui contient un petit bug dans l’élément d’ancre :

```html
<a href="https://www.freecodecamp.org/larn/">freeCodeCamp curriculum</a>
```

Quand tu cliques sur le lien, il mène à une page 404. Une page 404 est une page d’erreur qui apparaît quand un utilisateur essaie d’accéder à une page web qui n’existe pas sur le serveur.

L’intention est que le lien mène au curriculum freeCodeCamp.

Pour voir quel peut être le problème, tu peux utiliser les outils de développement.

Pour ouvrir les outils de développement dans ton navigateur, tu peux faire un clic droit sur la page et sélectionner `Inspect`.

Tu peux aussi utiliser `Control Shift I` sur un clavier PC ou `Command Option I` sur Mac.

Quand tu ouvres les outils de développement dans Google Chrome, tu vois plusieurs onglets. Le premier s’appelle `Elements`. Cet onglet montre la structure HTML de la page sur laquelle tu te trouves.

Le deuxième onglet s’appelle `Console`. Il montre les erreurs qui peuvent se produire sur la page.

Si tu as un lien cassé, tu peux consulter la console pour voir les messages d’erreur liés à ce lien. Le message courant qui continue de s’afficher pour ce lien est l’erreur 404. Elle indique que la page est introuvable.

Cela indique que le problème vient de l’URL dans l’élément d’ancre. Quand tu inspectes la valeur `href`, tu vois qu’il y a une faute de frappe.

Pour l’instant, le message de la console montre `/larn` avec une 404, mais l’URL correcte doit être `/learn`. Quand le lien est corrigé, il fonctionne comme prévu.

Tu en apprendras plus sur les outils de développement tout au long de la certification, mais cet exemple montre déjà comment ils peuvent t’aider à déboguer ton code.
