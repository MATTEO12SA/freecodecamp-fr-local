# --description--

HTML est un langage très tolérant : les éléments s’affichent quand même lorsque tu fais des erreurs, par exemple si tu oublies une balise fermante.

Disons que tu as un élément `h2` sans balise fermante :

```html
<h1>Article Topic</h1>
<h2>Subheading 1 </h2>
<h2>Subheading 2 </h2>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
```

Le `h2` sans balise fermante s’affichera quand même correctement. Cela arrive parce que les navigateurs utilisent un algorithme d’analyse qui gère les erreurs courantes et essaie d’afficher le HTML aussi près que possible de l’intention de l’auteur.

Mais cela peut parfois produire un résultat inattendu. Ajoutons quelques paragraphes sous les balises de titre de niveau 2 existantes dans le code :

```html
<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.</p>

<h2>Subheading 1 </h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.</p>

<h2>Subheading 2 </h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.</p>

<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.</p>
```

Résultat : l’élément de paragraphe sous le `h2` sans balise fermante `h2` s’affiche comme un titre de niveau 2. C’est pour cela que tu as besoin d’un validateur HTML.

Un validateur HTML est un outil qui vérifie la validité de ton code HTML par rapport aux spécifications HTML standard. Il t’aide à repérer les erreurs et avertissements dans ton code HTML, pour t’assurer que tes pages web sont correctement structurées et conformes aux standards du web.

Utiliser un validateur HTML t’aide toi, tes futures revues de code, mais aussi toute autre personne qui lira ton code, comme tes coéquipiers ou des contributeurs open source.

Il existe plusieurs validateurs HTML. Le plus reconnu est le service de validation de balisage `w3.org`.

Quand tu vas sur le site `validator.w3.org`, tu peux cliquer sur le bouton `Validate by Direct Input` et coller ton code HTML.

Quand tu cliques sur le bouton `Check`, une liste de résultats s’affiche avec les erreurs à corriger.

Un autre validateur HTML que tu peux utiliser est `jsonformatter.org`.

Tu peux copier et coller ton code HTML dans le premier éditeur. Quand tu cliques sur le bouton `Validate`, il affiche les erreurs présentes dans ton code.
