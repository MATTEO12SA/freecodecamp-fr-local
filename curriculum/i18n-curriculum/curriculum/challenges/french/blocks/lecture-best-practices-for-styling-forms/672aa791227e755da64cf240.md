# --interactive--

Comme avec tous les éléments de texte, tu dois t'assurer que les styles que tu appliques à l'entrée de texte soient accessibles. Cela signifie que la police doit être de taille adéquate et que la couleur doit avoir un contraste suffisant avec l'arrière-plan.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="username">Username</label>
  <input type="text" id="username" name="username" placeholder="Enter your username">
  <button type="submit">Submit</button>
</form>

```

```css
body {
  background-color: #f9fafb;
  color: #222;
  padding: 2rem;
}

.accessible-form {
  max-width: 320px;
  margin: 0 auto;
}

label {
  display: block;
  font-weight: 600;
  margin-bottom: 0.5rem;
}

input[type="text"] {
  width: 100%;
  padding: 0.6rem 0.8rem;
  font-size: 1rem;
  border: 2px solid #555;
  border-radius: 4px;
  background-color: #fff;
  color: #111;
}

input[type="text"]:focus {
  outline: 3px solid #1e90ff;
  border-color: #1e90ff;
}

button {
  margin-top: 1rem;
  padding: 0.6rem 1rem;
  font-size: 1rem;
  background-color: #1e90ff;
  color: #fff;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

button:hover,
button:focus {
  background-color: #187bcd;
}

```

:::

Le placeholder, cependant, est souvent oublié. Il est important de te rappeler que c'est aussi du texte et que tu devras probablement changer le style pour t'assurer qu'il soit lisible.

Une autre chose à garder en tête est que tu devrais permettre à l'utilisateur de modifier l'entrée. Par exemple, s'il s'agit d'une textarea, tu ne devrais pas supprimer la capacité de la redimensionner. L'entrée devrait aussi s'adapter en conséquence quand l'utilisateur zoome la page.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="message">Your message</label>
  <textarea id="message" name="message" placeholder="Type your message here..."></textarea>
  <button type="submit">Send</button>
</form>

```

```css
body {
  background-color: #f9fafb;
  color: #222;
  padding: 2rem;
  line-height: 1.5;
}

.accessible-form {
  max-width: 480px;
  margin: 0 auto;
}

label {
  display: block;
  font-weight: 600;
  margin-bottom: 0.5rem;
}

textarea {
  width: 100%;
  min-height: 120px;
  padding: 0.8rem;
  font-size: 1rem;
  border: 2px solid #555;
  border-radius: 4px;
  background-color: #fff;
  color: #111;
  resize: both; 
  box-sizing: border-box; 
}

textarea::placeholder {
  color: #555;
  opacity: 1;
}

textarea:focus {
  outline: 3px solid #1e90ff;
  border-color: #1e90ff;
}

button {
  margin-top: 1rem;

  font-size: 1rem;
  background-color: #1e90ff;
  color: #fff;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

  background-color: #187bcd;
}

```

:::

Les éléments input sont aussi focalisables. Quand tu édites tes styles, tu devrais faire attention à préserver un indicateur visible quand l'élément a le focus, comme une bordure en gras.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="name">Full name</label>
  <input type="text" id="name" name="name" placeholder="Enter your full name">

  <label for="email">Email address</label>
  <input type="email" id="email" name="email" placeholder="you@example.com">

  <button type="submit">Submit</button>
</form>

```

```css

  color: #222;

  display: block;
  font-weight: 600;

  opacity: 1;
}
```

De plus, il y a l'état d'erreur à considérer. Quand le texte de l'utilisateur ne passe pas la validation de l'entrée, il y a un indicateur visuel qui montre qu'il y a quelque chose qui ne va pas.

Pour avoir un message d'erreur qui se met à jour dynamiquement, tu devrais utiliser JavaScript, dont tu apprendras plus dans les futures leçons.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form class="accessible-form">
  <label for="email">Email address</label>
  <input 
    type="email" 
    id="email" 
    name="email" 
    placeholder="you@example.com" 
    aria-describedby="email-error"
  >
  <p id="email-error" class="error-message">
    Please enter a valid email address.
  </p>

  <button type="submit">Submit</button>
</form>

<script src="index.js"></script>
```

```css
  background-color: #187bcd;

```

:::

Quand tu donnes ta touche personnelle à ton élément, tu veux t'assurer que l'état d'erreur reste perceptible et qu'il ne ressemble pas à l'indicateur de focus. Avec tout cela en tête, tu peux maintenant créer des formulaires utilisateur captivants.
