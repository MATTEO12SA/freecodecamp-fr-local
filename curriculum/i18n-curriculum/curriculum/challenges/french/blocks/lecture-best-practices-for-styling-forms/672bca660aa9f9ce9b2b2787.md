# --interactive--

Apprenons quelques-uns des problèmes courants quand on essaie de styliser des éléments d'entrée spéciaux comme les entrées `datetime-local` et `color`.

Ces types spéciaux d'entrées s'appuient sur des pseudo-éléments complexes pour créer des choses comme les sélecteurs de date et de couleur. Cela présente un défi important pour styliser ces entrées. Un défi est que, parce que le style par défaut dépend entièrement du navigateur, le CSS qui fait que le sélecteur a l'air correct dans un navigateur peut produire un résultat très différent dans un autre.

Voici un exemple d'entrée color :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="favorite-color">Pick your favorite color:</label>
  <input type="color" id="favorite-color" name="favorite-color">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="color"] {
  width: 60px;
  height: 40px;
  padding: 0;
  border: 2px solid #555;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Un autre peut être la complexité du pseudo-élément. Considère le sélecteur de date ; il y a beaucoup d'éléments mobiles ici et la structure complexe du pseudo-élément peut poser un défi important pour appliquer un style aux bonnes zones.

Voici un exemple d'entrée date :

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="birthdate">Select your birthdate:</label>
  <input type="date" id="birthdate" name="birthdate">
</form>
```

```css
input {
  padding: 8px 12px;
  margin: 8px 0;
  border-radius: 6px;
  border: 1px solid #ccc;
}

input[type="date"] {
  padding: 6px 10px;
  border: 2px solid #555;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

input[type="date"]::-webkit-calendar-picker-indicator {
  background-color: #4CAF50;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}

```

:::

Bien sûr, avec ces éléments complexes, tu cours aussi le risque de perdre accidentellement des fonctionnalités importantes quand tu les stylises manuellement. Tu pourrais non seulement perdre des indicateurs importants comme l'état de focus ou l'élément sélectionné, mais tu pourrais potentiellement casser le sélecteur entièrement.

Pour ces raisons, beaucoup de développeurs s'appuient sur des bibliothèques JavaScript ou des composants personnalisés entièrement au lieu d'utiliser les composants intégrés du navigateur.
