# --interactive--

Apprenons quand utiliser `appearance: none` pour résoudre les problèmes de stylisation des entrées de recherche, des cases à cocher et des boutons radio.

Les navigateurs appliquent un style par défaut à beaucoup d'éléments. Dans le cas des éléments input, ta capacité à les styliser avec CSS peut sembler plutôt restreinte. Donc, tu peux vouloir utiliser `appearance: none` pour cacher des aspects de l'élément par défaut et construire le tien à la place. Par exemple, cela cacherait les cases à cocher par défaut pour une entrée checkbox te permettant d'utiliser des indicateurs personnalisés comme une coche verte et un X rouge pour montrer l'état. Pour une entrée de recherche, les navigateurs basés sur WebKit afficheront une icône de recherche par défaut et un bouton d'annulation. Les cacher te permet de créer tes propres indicateurs qui apparaîtraient sur tous les navigateurs.

**NOTE** : Une partie du CSS dans cet exemple utilise des propriétés qui n'ont pas encore été couvertes. C'est juste pour te donner une idée de comment créer une case à cocher personnalisée. Tu apprendras comment tout cela fonctionne dans les futures leçons et ateliers.

```html
<link rel="stylesheet" href="styles.css" />

  <label>
    <input class="checkbox" type="checkbox" /> Agree

```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}
```

:::

WebKit est un moteur logiciel qui aide les navigateurs web à afficher les sites web. Les navigateurs comme Safari utilisent WebKit pour s'assurer que les pages web ont l'air et fonctionnent correctement. Cette propriété CSS `appearance: none` te donne un contrôle complet sur le style, mais elle vient avec quelques choses à surveiller. Les composants interactifs par défaut des éléments input incluent des fonctionnalités comme les indicateurs de focus et d'erreur que tu devras t'assurer de ne pas perdre.

Créer un style cohérent multi-plateforme est une excellente raison d'utiliser cette propriété. Tu peux aussi l'utiliser pour t'assurer que les cibles de tap sur un appareil mobile sont assez grandes ou que les couleurs d'une case à cocher ont assez de contraste.
