## Introduction à l'accessibilité

- **Web Content Accessibility Guidelines** : Les Web Content Accessibility Guidelines (WCAG) sont un ensemble de recommandations pour rendre le contenu web plus accessible aux personnes en situation de handicap. Les quatre principes des WCAG sont POUR, qui signifie Perceptible, Opérable, Compréhensible et Robuste.

## Technologies d'assistance pour l'accessibilité

- **Lecteurs d'écran** : des logiciels qui lisent à voix haute le contenu d'un écran d'ordinateur. Ils sont utilisés par les personnes aveugles ou malvoyantes pour accéder au web.
- **Claviers à gros caractères ou en braille** : utilisés par les personnes avec des déficiences visuelles pour accéder au web.
- **Loupes d'écran** : des logiciels qui agrandissent le contenu d'un écran d'ordinateur. Ils sont utilisés par les personnes avec une vision faible pour accéder au web.
- **Périphériques de pointage alternatifs** : utilisés par les personnes avec des handicaps moteurs pour accéder au web. Cela inclut des appareils comme les joysticks, les trackballs et les pavés tactiles.
- **Reconnaissance vocale** : utilisée par les personnes avec des handicaps moteurs pour accéder au web. Elle permet aux utilisateurs de contrôler un ordinateur avec leur voix.

## Outils d'audit d'accessibilité

- **Outils d'accessibilité courants** : Google Lighthouse, Wave, IBM Equal Accessibility Checker et axe DevTools sont quelques-uns des outils d'accessibilité courants utilisés pour auditer l'accessibilité d'un site web.

## Bonnes pratiques d'accessibilité

- **Bonne structure des niveaux de titres** : tu devrais utiliser les bons niveaux de titres pour créer une structure logique pour ton contenu. Cela aide les personnes qui utilisent des technologies d'assistance à comprendre le contenu de ton site.
- **Accessibilité et tableaux** : quand tu utilises des tableaux, tu devrais utiliser l'élément `th` pour définir les cellules d'en-tête et l'élément `td` pour définir les cellules de données. Cela aide les personnes qui utilisent des technologies d'assistance à comprendre la structure du tableau. Avec l'élément `caption`, tu peux écrire la légende (ou le titre) d'un tableau, pour que les utilisateurs, surtout ceux qui utilisent des technologies d'assistance, puissent comprendre rapidement le but et le contenu du tableau. Tu devrais placer l'élément `caption` immédiatement après la balise ouvrante de l'élément `table`. De cette façon, les lecteurs d'écran et autres technologies d'assistance peuvent fournir plus de contexte en annonçant la légende avant de lire le contenu.
- **Importance que les inputs aient un libellé associé** : tu devrais utiliser l'élément `label` pour associer des libellés aux champs de formulaire. Cela aide les personnes qui utilisent des technologies d'assistance à comprendre le rôle du champ.
- **Importance d'un bon texte `alt`** : tu devrais utiliser l'attribut `alt` pour fournir une alternative textuelle aux images. Cela aide les personnes qui utilisent des technologies d'assistance à comprendre le contenu de l'image.
- **Importance d'un bon texte de lien** : tu devrais utiliser un texte de lien descriptif pour aider les utilisateurs à comprendre le but du lien. Cela aide les personnes qui utilisent des technologies d'assistance à comprendre le but du lien.
- **Bonnes pratiques pour rendre les contenus audio et vidéo accessibles** : tu devrais fournir des sous-titres et des transcriptions pour les contenus audio et vidéo afin de les rendre accessibles aux personnes avec des déficiences auditives. Tu devrais aussi fournir des descriptions audio pour les contenus vidéo afin de les rendre accessibles aux personnes avec des déficiences visuelles.
- **Attribut `tabindex`** : utilisé pour rendre les éléments focusables et définir l'ordre relatif dans lequel ils doivent être parcourus au clavier. Il est important de ne jamais utiliser l'attribut `tabindex` avec une valeur supérieure à 0. À la place, tu devrais utiliser une valeur de 0 ou -1.

:::interactive_editor

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

:::

- **Attribut `accesskey`** : utilisé pour définir un raccourci clavier pour un élément. Cela peut aider les utilisateurs avec des handicaps moteurs à naviguer plus facilement sur le site.

:::interactive_editor

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

:::

## WAI-ARIA, rôles et attributs

- **WAI-ARIA** : signifie Web Accessibility Initiative - Accessible Rich Internet Applications. C'est un ensemble d'attributs qui peuvent être ajoutés aux éléments HTML pour améliorer l'accessibilité. Il fournit des informations supplémentaires aux technologies d'assistance sur le but et la structure du contenu.
- **Rôles ARIA** : un ensemble de rôles prédéfinis qui peuvent être ajoutés aux éléments HTML pour définir leur but. Cela aide les personnes qui utilisent des technologies d'assistance à comprendre le contenu du site. Des exemples incluent `role="tab"`, `role="menu"` et `role="alert"`.

Il y a six grandes catégories de rôles ARIA :

- **Rôles de structure de document** : ces rôles définissent la structure globale de la page web. Avec ces rôles, les technologies d'assistance peuvent comprendre les relations entre les différentes sections et aider les utilisateurs à naviguer dans le contenu.
- **Rôles de widget** : ces rôles définissent le but et la fonctionnalité des éléments interactifs, comme les barres de défilement.
- **Rôles de point de repère** : ces rôles classent et étiquettent les sections principales d'une page web. Les lecteurs d'écran les utilisent pour fournir une navigation pratique vers les sections importantes d'une page.
- **Rôles de région live** : ces rôles définissent les éléments dont le contenu changera dynamiquement. Ainsi, les lecteurs d'écran et autres technologies d'assistance peuvent annoncer les changements aux utilisateurs avec des handicaps visuels.
- **Rôles de fenêtre** : ces rôles définissent des sous-fenêtres, comme les boîtes de dialogue modales en pop-up. Ces rôles incluent `alertdialog` et `dialog`. 
- **Rôles abstraits** : ces rôles aident à organiser le document. Ils sont uniquement destinés à être utilisés en interne par le navigateur, pas par les développeurs. Tu devrais savoir qu'ils existent mais tu ne devrais pas les utiliser sur tes sites web ou applications web.

- **Attributs `aria-label` et `aria-labelledby`** : ces attributs servent à donner à un élément un nom programmatique (ou accessible), ce qui aide les personnes qui utilisent une technologie d'assistance (comme les lecteurs d'écran) à comprendre le rôle de l'élément. Ils sont souvent utilisés quand le libellé visuel d'un élément est une image ou un symbole plutôt que du texte. `aria-label` te permet de définir le nom directement dans l'attribut, alors que `aria-labelledby` te permet de référencer un texte existant sur la page.

```html
<button aria-label="Search">
    <i class="fas fa-search"></i>
</button>
```

:::interactive_editor

```html
<input type="text" aria-labelledby="search-btn">
<button type="button" id="search-btn">Search</button>
```

:::

- **Attribut `aria-hidden`** : utilisé pour cacher un élément aux technologies d'assistance comme les lecteurs d'écran. Par exemple, cela peut servir à cacher des images décoratives qui ne fournissent aucun contenu significatif.

:::interactive_editor

```html
<button>
    <i class="fa-solid fa-gear" aria-hidden="true"></i>
    <span class="label">Settings</span>
</button>
```

:::

- **Attribut `aria-describedby`** : utilisé pour fournir des informations supplémentaires sur un élément en l'associant à un autre élément qui contient l'information. Cela donne aux personnes qui utilisent des lecteurs d'écran un accès immédiat aux informations supplémentaires quand elles naviguent vers l'élément. Des usages courants incluent l'association d'instructions de formatage à un champ de saisie ou d'un message d'erreur à un champ après la soumission d'un formulaire invalide.

:::interactive_editor

```html
<form>
    <label for="password">Password:</label>
    <input type="password" id="password" aria-describedby="password-help" />
    <p id="password-help">Your password must be at least 8 characters long.</p>
</form>
```

:::
