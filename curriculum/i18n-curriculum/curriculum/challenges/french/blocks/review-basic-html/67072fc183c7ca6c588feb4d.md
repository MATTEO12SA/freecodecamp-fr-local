# --interactive--

## Bases de HTML

- **Rôle de HTML** : HTML représente le contenu et la structure de la page web.
- **Éléments HTML** : les éléments sont les blocs de construction d'un document HTML. Ils représentent des titres, des paragraphes, des liens, des images et plus encore. La plupart des éléments HTML se composent d'une balise ouvrante (`<elementName>`) et d'une balise fermante (`</elementName>`).

Voici la syntaxe de base :

```html
<elementName>Content goes here</elementName>
```

- **Éléments vides** : les éléments vides ne peuvent pas avoir de contenu et possèdent seulement une balise de début. Les éléments `img` et `meta` en sont des exemples.

```html
<img>
<meta>
```

Il est courant de voir certaines bases de code inclure un slash `/` dans l'élément vide. Ces deux écritures sont acceptables :

```html
<img>
<img/>
```

- **Attributs** : un attribut est une valeur placée dans la balise ouvrante d'un élément HTML. Les attributs donnent des informations supplémentaires sur l'élément ou précisent comment il doit se comporter. Voici la syntaxe de base d'un attribut :

```html
<element attribute="value"></element>
```

Un attribut booléen est un attribut qui peut être présent ou absent dans une balise HTML. S'il est présent, sa valeur est vraie, sinon elle est fausse. `disabled`, `readonly` et `required` sont des exemples d'attributs booléens.

- **Commentaires** : les commentaires servent en programmation à laisser des notes pour toi-même et pour les autres développeurs dans ton code. Voici la syntaxe d'un commentaire en HTML :

```html
<!--This is an HTML comment.-->
```

## Éléments HTML courants

- **Éléments de titre** : il existe six éléments de titre en HTML. Les éléments de titre de `h1` à `h6` servent à indiquer l'importance du contenu qui les suit. Plus le nombre est petit, plus l'importance est élevée, donc les éléments `h2` sont moins importants que les éléments `h1`.

:::interactive_editor

```html
<h1>most important heading element</h1>
<h2>second most important heading element</h2>
<h3>third most important heading element</h3>
<h4>fourth most important heading element</h4>
<h5>fifth most important heading element</h5>
<h6>least important heading element</h6>
```

:::

- **Éléments de paragraphe** : ils servent à créer des paragraphes sur une page web.

:::interactive_editor

```html
<p>This is a paragraph element.</p>
```

:::

- **Éléments `img`** : l'élément `img` sert à ajouter des images à la page web. L'attribut `src` sert à indiquer l'emplacement de cette image. Pour les éléments image, c'est une bonne pratique d'ajouter un autre attribut appelé `alt`. Voici un exemple d'élément `img` avec les attributs `src` et `alt` :

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/lasagna.jpg" alt="A slice of lasagna on a plate.">
```

:::

- **Élément `body`** : cet élément représente le contenu du document HTML.

:::interactive_editor

```html
<body>
  <h1>CatPhotoApp</h1>
  <p>This is a paragraph element.</p>
</body>
```

:::

- **Éléments `section`** : cet élément sert à diviser le contenu en sections plus petites.

:::interactive_editor

```html
<section>
  <h2>About Me</h2>
  <p>Hi, I am Jane Doe and I am a web developer.</p>
</section>
```

:::

- **Éléments `div`** : cet élément HTML générique n'a pas de signification sémantique. Il sert de conteneur général pour contenir d'autres éléments HTML.

:::interactive_editor

```html
<div>
  <h1>I am a heading</h1>
  <p>I am a paragraph</p>
</div>
```

:::

- **Éléments d'ancre (`a`)** : ces éléments servent à créer des liens sur une page web. L'attribut `href` sert à indiquer où le lien doit mener quand l'utilisateur clique dessus.

:::interactive_editor

```html
<a href="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg">cute cats</a>
```

:::

- **Éléments de liste non ordonnée (`ul`) et ordonnée (`ol`)** : pour créer une liste à puces, tu dois utiliser l'élément `ul` avec un ou plusieurs éléments `li` imbriqués dedans, comme ceci :

:::interactive_editor

```html
<ul>
  <li>catnip</li>
  <li>laser pointers</li>
  <li>lasagna</li>
</ul>
```

:::

Pour créer une liste ordonnée d'éléments, tu dois utiliser l'élément `ol` :

:::interactive_editor

```html
<ol>
  <li>flea treatment</li>
  <li>thunder</li>
  <li>other cats</li>
</ol>
```

:::

- **Élément d'emphase (`em`)** : il sert à mettre l'accent sur un morceau de texte.

:::interactive_editor

```html
<p>Cats <em>love</em> lasagna.</p>
```

:::

- **Élément d'importance forte (`strong`)** : cet élément sert à donner une forte importance à un texte pour indiquer une idée d'urgence ou de sérieux.

:::interactive_editor

```html
<p>
  <strong>Important:</strong> Before proceeding, make sure to wear your safety goggles.
</p>
```

:::

- **Éléments `figure` et `figcaption`** : l'élément `figure` sert à regrouper du contenu comme des images et des diagrammes. L'élément `figcaption` sert à représenter une légende pour ce contenu dans l'élément `figure`.

:::interactive_editor

```html
<figure>
  <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch.">
  <figcaption>Cats <strong>hate</strong> other cats.</figcaption>
</figure>
```

:::

- **Élément `main`** : cet élément représente le contenu principal d'une page web.
- **Élément `footer`** : cet élément est placé en bas du document HTML et contient généralement des informations de copyright et d'autres liens importants de la page.

:::interactive_editor

```html
<footer>
  <p>
    No Copyright - <a href="https://www.freecodecamp.org">freeCodeCamp.org</a>
  </p>
</footer>
```

:::

## Identifiants et regroupement

- **ID** : identifiants uniques pour les éléments HTML. Les noms d'ID ne doivent être utilisés qu'une seule fois par document HTML.

```html
<h1 id="title">Movie Review Page</h1>
```

Les noms d'ID ne peuvent pas contenir d'espaces. Si ton nom d'ID contient plusieurs mots, tu peux utiliser des tirets entre les mots, comme ceci :

```html
<div id="red-box"></div>
```

- **Classes** : les classes servent à regrouper des éléments pour le style et le comportement.

```html
<div class="box"></div>
```

Contrairement aux ID, tu peux réutiliser le même nom de classe dans tout le document HTML. La valeur de `class` peut aussi contenir des espaces, comme ceci :

```html
<div class="box red-box"></div>
<div class="box blue-box"></div>
```

## Caractères spéciaux et liens

- **Entités HTML** : une entité HTML, ou référence de caractère, est un ensemble de caractères utilisé pour représenter un caractère réservé en HTML. Le symbole esperluette (`&amp;`) et le symbole inférieur à (`&lt;`) en sont des exemples.

```html
<p>This is an &lt;img /&gt; element</p>
```

- **Élément `link`** : cet élément sert à lier des ressources externes comme des feuilles de style et des icônes de site. Voici la syntaxe de base pour utiliser l'élément `link` avec un fichier CSS externe :

```html
<link rel="stylesheet" href="./styles.css" />
```

L'attribut `rel` sert à indiquer la relation entre la ressource liée et le document HTML. L'attribut `href` sert à indiquer l'emplacement de l'URL de la ressource externe.

- **Élément `script`** : cet élément sert à intégrer du code exécutable.

```html
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>
```

Même si tu peux techniquement écrire tout ton code JavaScript dans les balises `script`, il est considéré comme une meilleure pratique de lier plutôt un fichier JavaScript externe. Voici un exemple d'utilisation de l'élément `script` pour lier un fichier JavaScript externe :

```html
<script src="path-to-javascript-file.js"></script>
```

L'attribut `src` sert ici à indiquer l'emplacement de ce fichier JavaScript externe.

## Boilerplate et encodage

- **Boilerplate HTML** : un boilerplate est un modèle qui inclut la structure de base et les éléments essentiels dont chaque document HTML a besoin.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <!--Headings, paragraphs, images, etc. go inside here-->
  </body>
</html>
```

:::

- **`DOCTYPE`** : il sert à indiquer aux navigateurs quelle version de HTML tu utilises.
- **Élément `html`** : il représente l'élément de plus haut niveau, ou la racine, d'un document HTML. Pour préciser la langue du document, tu dois utiliser l'attribut `lang`.
- **Élément `head`** : la section `head` contient des métadonnées importantes, c'est-à-dire des informations en arrière-plan nécessaires aux navigateurs et aux moteurs de recherche.
- **Éléments `meta`** : ces éléments représentent les métadonnées de ton site. Ils contiennent des détails sur l'encodage des caractères, la façon dont des sites comme Twitter doivent prévisualiser le lien de ta page, et plus encore.
- **Élément `title`** : cet élément sert à définir le texte qui apparaît dans l'onglet ou la fenêtre du navigateur.
- **Encodage de caractères UTF-8** : UTF-8, ou UCS Transformation Format 8, est un encodage de caractères standardisé très utilisé sur le web. L'encodage de caractères est la méthode utilisée par les ordinateurs pour stocker des caractères sous forme de données. L'attribut `charset` est utilisé dans un élément `meta` pour définir l'encodage des caractères sur UTF-8.

## SEO et partage social

- **SEO** : Search Engine Optimization est une pratique qui optimise les pages web pour qu'elles soient plus visibles et mieux classées dans les moteurs de recherche.
- **Élément Meta (`description`)** : il sert à fournir une courte description de la page web et influence le SEO.

```html
<meta
  name="description"
  content="Discover expert tips and techniques for gardening in small spaces, choosing the right plants, and maintaining a thriving garden."
/>
```

- **Balises Open Graph** : le protocole Open Graph te permet de contrôler la façon dont le contenu de ton site apparaît sur différentes plateformes de réseaux sociaux, comme Facebook, LinkedIn et beaucoup d'autres.
  En définissant ces propriétés Open Graph, tu peux donner envie aux utilisateurs de cliquer et d'interagir avec ton contenu.
  Tu peux définir ces propriétés avec une collection d'éléments `meta` dans la section `head` de ton HTML.
- **Propriété `og:title`** : elle sert à définir le titre affiché pour les publications sur les réseaux sociaux.

```html
<meta content="freeCodeCamp.org" property="og:title" />
```

- **Propriété `og:type`** : la propriété `type` sert à représenter le type de contenu partagé sur les réseaux sociaux. Les articles, sites web, vidéos ou musiques sont des exemples de ce contenu.

```html
<meta property="og:type" content="website" />
```

- **Propriété `og:image`** : elle sert à définir l'image affichée pour les publications sur les réseaux sociaux.

```html
<meta
  content="https://cdn.freecodecamp.org/platform/universal/fcc_meta_1920X1080-indigo.png"
  property="og:image"
/>
```

- **Propriété `og:url`** : elle sert à définir l'URL sur laquelle les utilisateurs cliqueront dans les publications sur les réseaux sociaux.

```html
<meta property="og:url" content="https://www.freecodecamp.org" />
```

## Éléments média et optimisation

- **Éléments remplacés** : un élément remplacé est un élément dont le contenu est déterminé par une ressource externe plutôt que par le CSS lui-même. Un élément `iframe` en est un exemple. `iframe` signifie inline frame. C'est un élément en ligne utilisé pour intégrer directement d'autre contenu HTML dans la page HTML.

```html
<iframe src="https://www.example.com" title="Example Site"></iframe>
```

Tu peux inclure l'attribut `allowfullscreen`, qui permet à l'utilisateur d'afficher l'iframe en mode plein écran.

```html
<iframe
  src="video-url"
  width="width-value"
  height="height-value"
  allowfullscreen
></iframe>
```

Pour intégrer une vidéo dans une `iframe`, tu peux la copier directement depuis des services vidéo populaires comme YouTube et Vimeo, ou la définir toi-même avec l'attribut `src` pointant vers l'URL de cette vidéo. Voici un exemple d'intégration d'un cours freeCodeCamp populaire depuis YouTube :

:::interactive_editor

```html
<h1>A freeCodeCamp YouTube Video Embedded with the iframe Element</h1>

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/PkZNo7MFNFg?si=-UBVIUNM3csdeiWF"
  title="YouTube video player"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerpolicy="strict-origin-when-cross-origin"
  allowfullscreen
></iframe>
```

:::

Il existe d'autres éléments remplacés, comme `video` et `embed`. Certains éléments se comportent aussi comme des éléments remplacés dans des circonstances précises. Voici un exemple d'élément `input` avec l'attribut `type` défini sur `image` :

```html
<input type="image" alt="Descriptive text goes here" src="example-img-url">
```

- **Optimiser les médias** : il y a trois choses à prendre en compte quand tu utilises des médias, comme des images, sur tes pages web : la taille, le format et la compression. Un algorithme de compression sert à réduire la taille de fichiers ou de données.
- **Formats d'image** : deux des formats les plus courants sont PNG et JPG, mais ce ne sont plus les formats les plus adaptés pour servir des images. Sauf si tu dois prendre en charge d'anciens navigateurs, tu devrais utiliser un format plus optimisé, comme WEBP ou AVIF.
- **Licences d'image** : une image dans le domaine public n'a pas de droit d'auteur attaché et peut être utilisée sans restriction. Les images placées spécifiquement sous licence Creative Commons 0 sont considérées comme appartenant au domaine public. Certaines images peuvent être publiées sous une licence permissive, comme une licence Creative Commons, ou la licence BSD utilisée par freeCodeCamp.
- **SVG** : les Scalable Vector Graphics suivent des données basées sur des chemins et des équations pour placer des points, des lignes et des courbes. En pratique, cela signifie qu'un graphique vectoriel, comme un SVG, peut être redimensionné à n'importe quelle taille sans perdre en qualité.

## Intégration multimédia

- **Éléments `audio` et `video`** : les éléments `audio` et `video` te permettent d'ajouter du son et du contenu vidéo à tes documents HTML. L'élément `audio` prend en charge des formats audio populaires comme mp3, wav et ogg. L'élément `video` prend en charge les formats mp4, ogg et webm.

```html
<audio src="CrystalizeThatInnerChild.mp3"></audio>
```

Si tu veux voir le lecteur audio sur la page, tu peux ajouter l'élément `audio` avec l'attribut `controls` :

:::interactive_editor

```html
<audio src="CrystalizeThatInnerChild.mp3" controls></audio>
```

:::

L'attribut `controls` permet aux utilisateurs de gérer la lecture audio, notamment d'ajuster le volume, de mettre en pause ou de reprendre la lecture. L'attribut `controls` est un attribut booléen qui peut être ajouté à un élément pour activer les contrôles de lecture intégrés. S'il est omis, aucun contrôle ne sera affiché.

- **Attribut `loop`** : l'attribut `loop` est un attribut booléen qui fait rejouer l'audio en continu.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
></audio>
```

:::

- **Attribut `muted`** : quand il est présent dans l'élément `audio`, l'attribut booléen `muted` démarre l'audio en mode muet.

:::interactive_editor

```html
<audio
  src="https://cdn.freecodecamp.org/curriculum/js-music-player/can't-stay-down.mp3"
  loop
  controls
  muted
></audio>
```

:::

- **Élément `source`** : pour les types de fichiers audio, les navigateurs ne prennent pas tous en charge les mêmes formats. Pour gérer cela, tu peux utiliser des éléments `source` dans l'élément `audio`, et le navigateur choisira la première source qu'il comprend. Voici un exemple d'utilisation de plusieurs éléments `source` pour un élément `audio` :

```html
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>
```

Tous les attributs que nous avons appris jusqu'ici sont aussi pris en charge par l'élément `video`. Voici un exemple d'élément `video` avec les attributs `loop`, `controls` et `muted` :

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
></video>
```

:::

- **Attribut `poster`** : si tu veux afficher une image pendant le téléchargement de la vidéo, tu peux utiliser l'attribut `poster`. Cet attribut n'est pas disponible pour les éléments `audio` et il est propre à l'élément `video`.

:::interactive_editor

```html
<video
  src="https://archive.org/download/BigBuckBunny_124/Content/big_buck_bunny_720p_surround.mp4"
  loop
  controls
  muted
  poster="https://peach.blender.org/wp-content/uploads/title_anouncement.jpg?x11217"
  width="620"
></video>
```

:::

## Types d'attribut target

- **Attribut `target`** : cet attribut indique au navigateur où ouvrir l'URL de l'élément d'ancre. Il existe quatre valeurs importantes possibles pour cet attribut : `_self`, `_blank`, `_parent` et `_top`. Il existe une cinquième valeur, appelée `_unfencedTop`, actuellement utilisée pour l'API expérimentale `FencedFrame`. Tu n'auras probablement pas encore de raison de l'utiliser.
- **Valeur `_self`** : c'est la valeur par défaut de l'attribut `target`. Elle ouvre le lien dans le contexte de navigation actuel. Dans la plupart des cas, ce sera l'onglet ou la fenêtre actuelle.

```html
<a href="https://freecodecamp.org" target="_self">Visit freeCodeCamp</a>
```

- **Valeur `_blank`** : elle ouvre le lien dans un nouveau contexte de navigation. Généralement, cela ouvre un nouvel onglet. Mais certains utilisateurs peuvent configurer leur navigateur pour ouvrir une nouvelle fenêtre à la place.

```html
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
```

- **Valeur `_parent`** : elle ouvre le lien dans le parent du contexte actuel. Par exemple, si ton site web contient une iframe, une valeur `_parent` dans cette iframe s'ouvrira dans l'onglet ou la fenêtre de ton site, pas dans le cadre intégré.

```html
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
```

- **Valeur `_top`** : elle ouvre le lien dans le contexte de navigation le plus haut, comme "le parent du parent". C'est similaire à `_parent`, mais le lien s'ouvrira toujours dans l'onglet ou la fenêtre complète du navigateur, même avec des cadres intégrés imbriqués.

```html
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>
```

## Chemins absolus et relatifs

- **Définition d'un chemin** : un chemin est une chaîne de caractères qui indique l'emplacement d'un fichier ou d'un dossier dans un système de fichiers. En développement web, les chemins permettent aux développeurs de lier des ressources comme des images, des feuilles de style, des scripts et d'autres pages web.
- **Syntaxe des chemins** : il y a trois syntaxes clés à connaître. La première est le slash, qui peut être un antislash (`\`) ou un slash (`/`) selon ton système d'exploitation. La deuxième est le point simple (`.`). Et enfin, il y a les deux points (`..`). Le slash est appelé "séparateur de chemin". Il sert à indiquer une séparation dans le texte entre les noms de dossiers ou de fichiers. Un point simple pointe vers le dossier courant, et deux points pointent vers le dossier parent.

```md
public/index.html
./favicon.ico
../src/index.css
```

- **Chemin absolu** : un chemin absolu est un lien complet vers une ressource. Il commence à la racine, inclut tous les autres dossiers, puis enfin le nom du fichier et son extension. Le "dossier racine" désigne le dossier ou répertoire le plus haut dans une hiérarchie. Un chemin absolu inclut aussi le protocole, qui peut être `http`, `https` ou `file`, ainsi que le nom de domaine si la ressource est sur le web. Voici un exemple de chemin absolu qui pointe vers le logo freeCodeCamp :

:::interactive_editor

```html
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">
  View fCC Logo
</a>
```

:::

- **Chemin relatif** : un chemin relatif indique l'emplacement d'un fichier par rapport au dossier du fichier actuel. Il n'inclut pas le protocole ni le nom de domaine, ce qui le rend plus court et plus flexible pour les liens internes d'un même site web. Voici un exemple de lien vers la page `about.html` depuis la page `contact.html`, les deux étant dans le même dossier :

```html
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>
```

## États des liens

- **`:link`** : c'est l'état par défaut. Cet état représente un lien que l'utilisateur n'a pas encore visité, cliqué ou utilisé. Tu peux voir cet état comme la base des styles pour tous les liens de ta page. Les autres états s'ajoutent par-dessus.
- **`:visited`** : il s'applique quand un utilisateur a déjà visité la page liée. Par défaut, cela rend le lien violet, mais tu peux utiliser CSS pour donner une indication visuelle différente à l'utilisateur.
- **`:hover`** : cet état s'applique quand un utilisateur place son curseur sur un lien. Il est utile pour attirer davantage l'attention sur un lien et s'assurer que l'utilisateur a vraiment l'intention de cliquer dessus.
- **`:focus`** : cet état s'applique quand on met le focus sur un lien.
- **`:active`** : cet état s'applique aux liens qui sont activés par l'utilisateur. En général, cela signifie cliquer sur le lien avec le bouton principal de la souris, le plus souvent par un clic gauche.
