# --interactive--

En programmation, comprendre comment les caractères sont représentés sous forme de nombres est fondamental. C'est là qu'intervient l'ASCII. L'ASCII, abréviation d'American Standard Code for Information Interchange, est une norme d'encodage de caractères utilisée par les ordinateurs pour représenter du texte. Elle attribue une valeur numérique à chaque caractère, valeur universellement reconnue par les machines.

Dans cette leçon, nous allons explorer ce qu'est l'ASCII, comment il fonctionne, et comment des méthodes JavaScript comme `charCodeAt()` et `fromCharCode()` sont liées à l'encodage des caractères. Même si les chaînes JavaScript utilisent Unicode (UTF-16) en interne, les valeurs ASCII correspondent aux 128 premiers caractères Unicode, ce qui explique pourquoi les exemples basés sur l'ASCII fonctionnent en JavaScript.

L'ASCII est un système qui encode des caractères tels que les lettres, les chiffres et les symboles en valeurs numériques. Chaque caractère est associé à un nombre précis.

Par exemple, la lettre majuscule `A` est représentée par le nombre `65` en ASCII, tandis que la minuscule `a` est représentée par `97`. Cet encodage permet aux ordinateurs de stocker et de manipuler du texte.

La norme ASCII couvre 128 caractères, dont :

- Les lettres anglaises majuscules et minuscules (A-Z, a-z).
- Les chiffres (0-9).
- Les signes de ponctuation et symboles courants (!, @, #, etc.).
- Les caractères de contrôle (comme le retour à la ligne et la tabulation).

En JavaScript, tu peux accéder au code numérique d'un caractère à l'aide de la méthode `charCodeAt()`. Cette méthode renvoie l'unité de code UTF-16 du caractère situé à un indice donné. Pour les 128 premiers caractères, cette valeur correspond au code ASCII.

Regardons un exemple :

:::interactive_editor

```js
let letter = "A";
console.log(letter.charCodeAt(0));  // 65
```

:::

Dans cet exemple, `A` est le premier caractère de la chaîne, et appeler `charCodeAt(0)` renvoie son code numérique (qui correspond à sa valeur ASCII pour les caractères latins de base), `65`.

Tu peux aussi utiliser cette méthode avec d'autres caractères pour trouver leurs valeurs de code numérique :

:::interactive_editor

```js
let symbol = "!";
console.log(symbol.charCodeAt(0));  // 33
```

:::

Ici, le code numérique du point d'exclamation `!` est renvoyé comme `33` (qui correspond à sa valeur ASCII).

Alors que `charCodeAt()` t'aide à récupérer le code numérique d'un caractère, la méthode `fromCharCode()` te permet de faire l'inverse : convertir une unité de code UTF-16 (qui correspond à l'ASCII pour les caractères de base) en son caractère correspondant.

Voyons cela en action :

:::interactive_editor

```js
let char = String.fromCharCode(65);
console.log(char);  //  A
```

:::

Dans cet exemple, `fromCharCode(65)` reconvertit le code numérique `65` (qui correspond à la valeur ASCII de `A`) en le caractère `A`.

Un autre exemple serait de convertir le nombre `97` en sa lettre minuscule correspondante :

:::interactive_editor

```js
let char = String.fromCharCode(97);
console.log(char);  // a
```

:::

Ces méthodes sont particulièrement utiles quand tu as besoin de manipuler ou de comparer des caractères en fonction de leurs valeurs de code numérique.

Par exemple, tu pourrais utiliser `charCodeAt()` pour vérifier si un caractère est une majuscule, une minuscule ou un chiffre en comparant sa valeur ASCII.

À l'inverse, `fromCharCode()` peut servir à générer dynamiquement des caractères à partir de leurs codes ASCII.
