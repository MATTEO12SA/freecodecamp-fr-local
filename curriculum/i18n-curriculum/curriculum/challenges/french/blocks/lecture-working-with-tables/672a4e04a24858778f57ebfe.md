# --interactive--

Les tableaux HTML sont moins utilisés aujourd'hui qu'avant. Mais, en tant que développeur front-end, tu dois quand même savoir les utiliser. Les tableaux font partie des premières méthodes que les développeurs avaient pour afficher des données dans un navigateur dans les années 1990.

Voici un exemple de code utilisé pour générer un tableau à partir du Bureau of Labor Statistics des États-Unis :

:::interactive_editor

```html
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers, Quality Assurance Analysts, and Testers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>
        $130,160 per year
        <br>$62.58 per hour
      </td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
    <tr>
      <th>On-the-job Training</th>
      <td>None</td>
    </tr>
    <tr>
      <th>Number of Jobs, 2022</th>
      <td>1,795,300</td>
    </tr>
    <tr>
      <th>Job Outlook, 2022-32</th>
      <td>25% (Much faster than average)</td>
    </tr>
    <tr>
      <th>Employment Change, 2022-32</th>
      <td>451,200</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>If this table had a footer it would be here.</th>
    </tr>
  </tfoot>
</table>
```

:::

Comme tu peux le voir, il y a un élément principal `table` avec un `id` défini sur `quickfacts`. À l'intérieur, le tableau possède un élément d'en-tête `thead`, un élément de corps `tbody` et un élément de pied `tfoot`.

Les éléments d'en-tête, de corps et de pied du tableau peuvent chacun contenir plusieurs lignes de tableau, `tr`. Chaque ligne peut contenir un en-tête de tableau `th`, qui étiquette les données de cette ligne. Elle peut aussi contenir plusieurs cellules de données individuelles, appelées données de tableau, `td`.

Cela fait beaucoup d'éléments HTML, mais ne te laisse pas impressionner : ils suivent une hiérarchie simple.

Voici le tableau le plus simple que l'on puisse créer avec tous ces éléments :

:::interactive_editor

```html
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>
        First Data Cell
      </td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>
        Second Data Cell
      </td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>The footer of this table, which might contain date of publication, author credits, or other meta information.</th>
    </tr>
  </tfoot>
</table>
```

:::

Comme tu peux le voir, les données sont toujours dans un `tr`. Dans cet élément `tr`, il y a un élément `th` avec un en-tête et un élément `td` avec les données.

Certains sites choisissent d'utiliser des `div` pour construire leurs propres tableaux au lieu d'utiliser l'élément `table`, qui est plus approprié.

Même s'il est possible d'afficher des données tabulaires avec des éléments génériques `div`, il vaut mieux utiliser l'élément `table`.

Il y a longtemps, certains développeurs utilisaient un `table` pour placer des éléments qui n'étaient pas des données sur une page web. Cela n'a jamais été considéré comme une bonne pratique, mais tu peux rencontrer des bases de code où les tableaux sont encore utilisés comme cela.

Aujourd'hui, les développeurs utilisent CSS flexbox et grid pour mettre en page leurs designs. freeCodeCamp couvrira ces outils plus en détail plus tard.

Pour l'instant, utilise les tableaux HTML uniquement pour leur objectif d'origine : afficher des données tabulaires.
