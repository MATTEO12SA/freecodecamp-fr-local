# --interactive--

Quand tu travailles avec des chaînes en JavaScript, il arrive que tu aies besoin d'inclure des caractères spéciaux que le moteur JavaScript pourrait sinon mal interpréter.

Deux tâches courantes consistent à créer un retour à la ligne dans une chaîne et à échapper certains caractères (comme les guillemets) pour s'assurer qu'ils s'affichent correctement.

Dans de nombreux langages de programmation, dont JavaScript, tu peux créer un retour à la ligne dans une chaîne à l'aide d'un caractère spécial appelé séquence d'échappement. La séquence d'échappement la plus courante pour les retours à la ligne est `\n`.

Par exemple, si tu veux découper une chaîne sur plusieurs lignes, tu utiliserais `\n` à l'endroit où tu veux que la nouvelle ligne commence :

:::interactive_editor

```js
let poem = "Roses are red,\nViolets are blue,\nJavaScript is fun,\nAnd so are you.";
console.log(poem);
```

:::

La séquence d'échappement `\n` indique à JavaScript d'insérer un retour à la ligne à cet endroit, ce qui fait que la chaîne s'affiche sur plusieurs lignes.

Un autre concept important lorsqu'on travaille avec les chaînes est l'échappement des caractères. Parfois, tu as besoin d'inclure dans ta chaîne des caractères que JavaScript utilise normalement pour autre chose, comme les guillemets.

Si tu utilises simplement des guillemets à l'intérieur d'une chaîne sans les échapper, cela peut provoquer une erreur, car JavaScript pensera que tu essaies de terminer la chaîne.

Par exemple, ceci provoquera une erreur :

```js
let statement = "She said, "Hello!"";
```

JavaScript est désorienté car il pense que la chaîne se termine après le mot `"said,"`, alors que tu veux que les guillemets autour de `"Hello!"` fassent partie de la chaîne.

Pour corriger cela, tu peux échapper les guillemets intérieurs en plaçant une barre oblique inverse (`\`) devant eux :

:::interactive_editor

```js
let statement = "She said, \"Hello!\"";
console.log(statement); // She said, "Hello!"
```

:::

La barre oblique inverse indique à JavaScript de traiter les guillemets comme des caractères littéraux, de sorte qu'ils s'affichent correctement dans la sortie.

Tu peux aussi échapper d'autres caractères spéciaux, comme la barre oblique inverse elle-même (`\\`), ou les apostrophes simples à l'intérieur d'une chaîne entourée d'apostrophes simples (`\'`).

Voici un autre exemple utilisant des apostrophes simples :

:::interactive_editor

```js
let quote = 'It\'s a beautiful day!';
console.log(quote); // It's a beautiful day!
```

:::

En échappant l'apostrophe simple avec `\'`, JavaScript sait qu'il doit l'inclure dans la chaîne plutôt que de terminer la chaîne prématurément.

L'échappement et la création de retours à la ligne sont essentiels quand tu mets en forme une sortie ou que tu gères des caractères spéciaux dans des chaînes. Ces techniques t'aident à éviter les erreurs et à garantir que ton texte s'affiche exactement comme prévu.
