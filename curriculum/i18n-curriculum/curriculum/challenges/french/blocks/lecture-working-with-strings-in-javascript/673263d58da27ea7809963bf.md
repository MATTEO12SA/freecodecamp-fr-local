# --interactive--

En JavaScript, les littéraux de modèle sont une façon puissante et souple de travailler avec les chaînes. Contrairement aux chaînes classiques, qui utilisent des apostrophes simples (`'`) ou des guillemets doubles (`"`), les littéraux de modèle se définissent avec des accents graves (`` ` ``).

Ils facilitent la manipulation des chaînes, notamment en permettant d'intégrer des variables directement à l'intérieur d'une chaîne, une fonctionnalité appelée interpolation de chaîne.

Les littéraux de modèle facilitent la création de chaînes qui s'étendent sur plusieurs lignes ou qui incluent des expressions (comme des variables, voire du code JavaScript) directement dans la chaîne.

Voici un exemple de littéral de modèle :

:::interactive_editor

```js
const name = "Alice";
const greeting = `Hello, ${name}!`;

console.log(greeting);
```

:::

Remarque l'utilisation des accents graves à la place des apostrophes simples ou des guillemets doubles. La syntaxe `${name}` est un exemple d'interpolation de chaîne, où la valeur de la variable `name` est insérée directement dans la chaîne.

L'interpolation de chaîne te permet d'intégrer des variables et des expressions dans une chaîne. C'est une nette amélioration par rapport à l'ancienne méthode, où l'on concaténait chaînes et variables à l'aide de l'opérateur `+`.

Voici un exemple utilisant la concaténation de chaînes et l'opérateur plus (`+`) :

:::interactive_editor

```js
const name = "Alice";
const age = 25;
const message = "My name is " + name + " and I am " + age + " years old.";
console.log(message);
```

:::

Et voici un exemple utilisant les littéraux de modèle et l'interpolation de chaîne :

:::interactive_editor

```js
const name = "Alice";
const age = 25;
const message = `My name is ${name} and I am ${age} years old.`;
console.log(message);
```

:::

Comme tu peux le voir, l'interpolation de chaîne avec les littéraux de modèle est bien plus claire et plus facile à lire, surtout quand tu travailles avec plusieurs variables.

Une autre excellente fonctionnalité des littéraux de modèle est qu'ils prennent en charge les chaînes multilignes. Avec les chaînes classiques, tu devrais utiliser des caractères d'échappement (`\n`) pour créer des retours à la ligne. Avec les littéraux de modèle, tu peux simplement écrire la chaîne sur plusieurs lignes, et la mise en forme est conservée :

:::interactive_editor

```js
let poem = `Roses are red,
Violets are blue,
JavaScript is fun,
And so are you.`;

console.log(poem);
```

:::

Une autre fonctionnalité des littéraux de modèle est qu'ils te permettent d'intégrer des expressions JavaScript directement dans la chaîne, comme dans cet exemple :

:::interactive_editor

```js
const song = "Bohemian Rhapsody";
const score = 9.5;
const highestScore = 10;
const output = `One of my favorite songs is "${song}". I rated it ${
  (score / highestScore) * 100
}%.`;
console.log(output);
```

:::

Les littéraux de modèle sont particulièrement utiles quand tu as besoin d'inclure des variables ou des expressions dans des chaînes, de mettre en forme des chaînes complexes ou de travailler avec du texte multiligne. Ils sont plus concis et plus lisibles que la concaténation de chaînes traditionnelle.
