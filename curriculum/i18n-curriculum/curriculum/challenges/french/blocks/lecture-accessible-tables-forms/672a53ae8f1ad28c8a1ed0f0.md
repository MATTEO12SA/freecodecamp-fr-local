# --interactive--

Les labels sur les champs input peuvent sembler être de petits détails, mais ils jouent un rôle crucial pour rendre les formulaires plus faciles à utiliser pour tout le monde.

Les labels aident les personnes handicapées, rendent les formulaires plus conviviaux pour tout le monde, et améliorent finalement les taux de conversion des formulaires.

Plongeons dans pourquoi ajouter des labels aux éléments input est important pour l'accessibilité et l'usabilité.

Les lecteurs d'écran s'appuient souvent sur les labels pour décrire le but des champs input. Pour que cela fonctionne correctement, le label doit être associé par programme à l'`input`. Bien qu'il y ait plusieurs façons de le faire, la plus courante est d'utiliser l'élément HTML `label`.

Voici comment associer un `label` avec un `input` :

:::interactive_editor

```html
<form>
   <label for="name">Your Name</label>
   <input type="text" id="name" />
</form>
```

:::

Dans cet exemple, l'attribut `for` de l'élément `label` est associé à l'`id` de l'élément `input`. Cette connexion permet aux lecteurs d'écran d'annoncer le `label` quand l'`input` est en état de focus, ce qui permet aux utilisateurs de lecteurs d'écran de comprendre le but de l'`input`.

Associer les labels à tous les champs input bénéficie aussi au SEO. Avec des labels appropriés, les moteurs de recherche comprendront mieux le contenu de la page, ce qui aura un impact positif sur le classement de la page dans les recherches.

Mais il est important de comprendre que tu fais du contenu pour des humains, pas pour des moteurs de recherche. Donc tu devrais t'assurer que les labels sont aussi clairs et descriptifs que possible.
