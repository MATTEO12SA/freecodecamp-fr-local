# --interactive--

Les libellés des champs de saisie peuvent sembler être de petits détails, mais ils jouent un rôle essentiel pour rendre les formulaires plus faciles à utiliser pour tout le monde.

Les libellés aident les personnes en situation de handicap, rendent les formulaires plus conviviaux pour tout le monde et améliorent au final les taux de conversion des formulaires.

Voyons pourquoi il est important d'ajouter des libellés aux éléments de saisie pour l'accessibilité et l'utilisabilité.

Les lecteurs d'écran s'appuient souvent sur les libellés pour décrire le rôle des champs de saisie. Pour que cela fonctionne correctement, le libellé doit être associé programmatiquement à l'`input`. Il existe plusieurs façons de le faire, mais la plus courante consiste à utiliser l'élément HTML `label`.

Voici comment associer un `label` à un `input` :

:::interactive_editor

```html
<form>
   <label for="name">Ton nom</label>
   <input type="text" id="name" />
</form>
```

:::

Dans cet exemple, l'attribut `for` de l'élément `label` est associé à l'`id` de l'élément `input`. Cette connexion permet aux lecteurs d'écran d'annoncer le `label` quand l'`input` reçoit le focus, ce qui aide les utilisateurs de lecteurs d'écran à comprendre le rôle de l'`input`.

Associer des libellés à tous les champs de saisie est aussi bénéfique pour le SEO. Avec des libellés appropriés, les moteurs de recherche comprendront mieux le contenu de la page, ce qui peut avoir un impact positif sur le classement de la page dans les résultats de recherche.

Mais il est important de comprendre que tu crées du contenu pour des humains, pas pour des moteurs de recherche. Tu devrais donc t'assurer que les libellés sont aussi clairs et descriptifs que possible.
