# --interactive--

L'opérateur `typeof` en JavaScript est un outil simple mais puissant qui te permet de voir le type de données d'une variable ou d'une valeur. Il renvoie toujours une chaîne indiquant le type.

Jetons un œil à quelques exemples :

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

Dans ce premier exemple, nous avons créé une variable appelée `num` et nous lui avons affecté le nombre `42`. Quand tu utilises l'opérateur `typeof` sur la variable nommée `num`, il renvoie la chaîne `number`.

Voici un autre exemple d'utilisation de l'opérateur `typeof` sur une variable appelée `isUserLoggedIn` :

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

Quand tu utilises l'opérateur `typeof` sur la variable `isUserLoggedIn`, il renvoie la chaîne `boolean` parce que le booléen `true` a été affecté à la variable.

Utiliser l'opérateur `typeof` peut être particulièrement utile quand tu débogues ou que tu essaies de comprendre quel type de données tu manipules dans ton code.

Cependant, il existe une bizarrerie bien connue en JavaScript en ce qui concerne `null`.

Regardons un exemple :

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

Dans cet exemple, nous avons une variable appelée `exampleVariable` et nous lui avons affecté la valeur `null`. Mais quand nous utilisons l'opérateur `typeof`, il renvoie la chaîne `object`.

C'est largement considéré comme un bug de JavaScript, qui remonte à ses tout débuts. La raison de ce comportement tient à la façon dont JavaScript a été conçu à l'origine.

Lors de la première implémentation du langage, des valeurs comme `null` étaient représentées comme un type particulier d'objet, ce qui aboutit à ce résultat inattendu.

Malheureusement, c'est devenu une partie intégrante du langage et, même si c'est déroutant, c'est quelque chose dont tu devras être conscient.
