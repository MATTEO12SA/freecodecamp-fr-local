# --description--

JavaScript est un langage à typage dynamique, ce qui signifie que tu n'as pas besoin de préciser le type de données d'une variable quand tu la déclares. À la place, le type est déterminé en fonction de la valeur affectée à la variable pendant l'exécution du programme. Cela te permet de changer le type d'une variable tout au long du programme.

Regardons un exemple :

```js
let example = "Hello";
example = 42;
```

Dans cet exemple, nous avons une variable appelée `example` dont le type de données est une chaîne. Mais ensuite, nous mettons à jour sa valeur pour en faire un nombre.

La souplesse du typage dynamique rend JavaScript plus permissif et facile à utiliser pour écrire rapidement des scripts, mais elle peut aussi introduire des bugs plus difficiles à repérer, surtout à mesure que ton programme grandit.

Dans les langages à typage statique comme C# ou C++, tu dois déclarer le type de données d'une variable quand tu la crées, et ce type ne peut pas changer.

Par exemple, si tu déclares une variable comme `integer`, tu ne peux lui affecter que des valeurs entières. Si tu essaies de lui affecter un type différent, le programme générera une erreur.

Voici un exemple dans le langage C# :

```csharp
int data = 42; // data must always be an integer
data = "Hello"; // This would cause an error in C#
```

La différence entre le typage dynamique et le typage statique réside dans la souplesse face à la sûreté de ton code. Les langages à typage dynamique offrent de la souplesse, mais au prix d'éventuelles erreurs à l'exécution.

Les langages à typage statique imposent des règles plus strictes qui peuvent éviter certaines erreurs, mais ils demandent davantage de déclarations en amont et offrent moins de souplesse pour changer les types.

Voici un autre exemple où l'on crée une variable de type `number`, puis où on la change ensuite pour qu'elle soit de type `string` :

```js
let data = 100;  // Initially a number
data = "New data";  // Dynamically changes to a string
```

Dans un langage à typage statique, ce genre de changement ne serait pas autorisé, car le type de données serait figé.

En conclusion, le typage dynamique de JavaScript permet aux variables de changer de type librement, ce qui offre de la souplesse mais peut entraîner des erreurs inattendues pendant l'exécution.

Les langages à typage statique comme Java t'obligent à préciser les types des variables en amont, ce qui aide à détecter les erreurs avant l'exécution du programme mais offre moins de souplesse.
