# --interactive--

HTML, qui signifie HyperText Markup Language, est un langage de balisage utilisé pour créer des pages web. Quand tu visites un site et que tu vois du contenu comme des paragraphes, des titres, des liens, des images et des vidéos, c'est du HTML.

Voici un petit exemple qui utilise des éléments HTML. Essaie de modifier une partie du texte dans l'éditeur et observe les changements dans la fenêtre d'aperçu.

:::interactive_editor

```html
<h1>Main heading element</h1>

<p>I am a paragraph element.</p>
```

:::

HTML représente le contenu et la structure d'une page web grâce aux éléments. La plupart des éléments ont une balise ouvrante et une balise fermante. On appelle parfois ces balises des balises de début et de fin. Entre ces deux balises, tu places le contenu. Ce contenu peut être du texte ou d'autres éléments HTML.

Voici un autre exemple d'élément paragraphe. Change le texte dans l'éditeur pour écrire `I love coding!` et observe le résultat dans la fenêtre d'aperçu.

:::interactive_editor

```html
<p>I am a paragraph element.</p>
```

:::

Les balises ouvrantes et fermantes commencent toutes les deux par un chevron ouvrant (`<`) et se terminent par un chevron fermant (`>`), avec le nom de la balise placé entre les deux. Même si les noms de balises HTML ne sont pas sensibles à la casse, la convention la plus répandue et la bonne pratique consistent à les écrire en minuscules.

Voici de plus près les balises ouvrante et fermante d'un paragraphe :

```html
<p>
```

```html
</p>
```

Ce qui distingue une balise ouvrante d'une balise fermante, c'est la barre oblique (`/`) placée immédiatement après le chevron ouvrant dans une balise fermante. Certains éléments HTML n'ont pas de balise fermante. On les appelle des éléments vides.

Voici un exemple d'élément image, qui est un élément vide :

```html
<img>
```

Remarque que cet élément image n'a pas de balise fermante et ne contient aucun contenu. Les éléments vides ne peuvent pas contenir de contenu et n'ont qu'une balise de début.

Tu verras parfois des éléments vides qui utilisent un `/` avant le `>` comme ceci :

```html
<img />
```

Même si beaucoup de formateurs de code comme Prettier choisissent d'inclure le `/` dans les éléments vides, la spécification HTML indique que la présence du `/` « ne marque pas la balise de début comme auto-fermante, mais est plutôt inutile et n'a aucun effet ».

Dans le développement réel, tu verras les deux formes, donc il est important de connaître les deux.

Si tu veux afficher une image, tu dois inclure quelques attributs dans ton élément image. Un attribut est une valeur spéciale utilisée pour ajuster le comportement d'un élément HTML.

Voici un exemple d'élément image avec un attribut `src`. Mets à jour la valeur de l'attribut `src` avec `"https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"` et tu verras l'image changer pour afficher deux chats qui dorment paisiblement.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg" />
```

:::

L'attribut `src` sert à indiquer l'emplacement de cette image. Pour les éléments image, il est recommandé d'inclure un autre attribut appelé attribut `alt`. L'attribut `alt` sert à fournir un texte court et descriptif pour les images.

Voici un exemple d'élément image avec les attributs `src` et `alt`. Essaie de casser l'image en mettant à jour la valeur de `src` avec `"https://.freecodecamp.org/curriculum/cat-photo-app/cats.jpg"`. Tu verras l'image disparaître et seul le texte `alt` s'afficher.

:::interactive_editor

```html
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch." />
```

:::

Tu te demandes peut-être si HTML seul suffit pour construire un site web. La réponse est : ça dépend. Si tu construis un petit projet d'entraînement qui affiche seulement du texte et des images, HTML seul peut suffire. En revanche, si tu crées un site web professionnel moderne, tu auras besoin de HTML, CSS et JavaScript.

HTML sert au contenu et à la structure. CSS sert au style. JavaScript sert à ajouter de l'interactivité à tes pages web. Une bonne analogie consiste à comparer HTML, CSS et JavaScript à un bâtiment complet.

HTML représente les blocs, le béton et les fers qui composent les murs. C'est la fondation qui rend le bâtiment solide. CSS représente le design intérieur et extérieur qui rend la maison belle. JavaScript représente le système électrique et l'installation d'eau qui assurent un accès continu à l'eau et à l'électricité.
