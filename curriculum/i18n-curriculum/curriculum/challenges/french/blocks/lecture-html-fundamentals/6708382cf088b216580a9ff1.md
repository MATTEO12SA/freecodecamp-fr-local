# --interactive--

L'attribut `id` ajoute un identifiant unique à un élément HTML.

Voici un exemple d'élément `h1` avec un `id` de `title`.

Sous l'élément `h1`, ajoute un élément `h2` avec un `id` défini sur `"subtitle"`. Tu peux écrire le texte que tu veux pour le `h2` et tu verras les changements dans la fenêtre d'aperçu. Pour interagir avec l'exemple, tu dois activer l'éditeur interactif.

:::interactive_editor

```html
<h1 id="title">Movie Review Page</h1>

```

:::

Tu peux faire référence au nom d'`id` `title` dans ton JavaScript ou ton CSS. Voici un exemple CSS qui cible l'`id` `title` pour changer la `color` du texte en `red`.

**NOTE :** du CSS t'est fourni dans cet exemple interactif. Ne t'inquiète pas si tu ne comprends pas encore le code CSS, tu en apprendras plus dans de futures leçons. Mais si tu veux voir la couleur du texte passer au bleu, active l'éditeur interactif, clique sur l'onglet `styles.css` et remplace `color: red;` par `color: blue;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Review page Example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <h1 id="title">Movie Review Page</h1>
  </body>
</html>
```

```css
#title {
  color: red;
}
```

:::

Le symbole dièse (`#`) placé devant `title` indique à l'ordinateur que tu veux cibler un `id` avec cette valeur. Les noms d'`id` ne doivent pas être utilisés plus d'une fois, et ils doivent toujours être uniques. Autre point important : les valeurs d'`id` ne peuvent pas contenir d'espaces. Voici un exemple où les mots `main` et `heading` sont utilisés comme valeur d'attribut `id` :

```html
<h1 id="main heading">Main heading</h1>
```

Les navigateurs considèrent cet espace comme une partie de l'`id`, ce qui peut provoquer des problèmes lors du style ou des scripts. Les valeurs d'attribut `id` ne doivent contenir que des lettres, des chiffres, des underscores et des tirets.

Contrairement à l'attribut `id`, la valeur de l'attribut `class` n'a pas besoin d'être unique et peut contenir des espaces.

Voici un exemple où une classe appelée `box` est appliquée à un élément `div`.

```html
<div class="box"></div>
```

Si tu veux ajouter plusieurs noms de classes à un élément, tu peux les séparer par un espace. Voici une version mise à jour qui applique plusieurs classes à un élément `div`.

```html
<div class="box red-box"></div>
```

Voici un autre exemple où plusieurs classes sont appliquées à plusieurs éléments `div`.

**NOTE :** du CSS t'est fourni dans cet exemple interactif. Ne t'inquiète pas si tu ne comprends pas encore le code CSS, tu en apprendras plus dans de futures leçons. Mais si tu veux changer la couleur de la première et de la troisième boîte, active l'éditeur interactif, clique sur l'onglet `styles.css` et remplace `background-color: red;` par `background-color: black;`.

:::interactive_editor

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta
       name="viewport"
       content="width=device-width, initial-scale=1.0" />
    <title>Colored boxes example</title>
    <link rel="stylesheet" href="./styles.css" />
  </head>
  <body>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
    <div class="box red-box"></div>
    <div class="box blue-box"></div>
  </body>
</html>
```

```css
.box {
  width: 100px;
  height: 100px;
}

.red-box {
  background-color: red;
}

.blue-box {
  background-color: blue;
}
```

:::

Donc, pour résumer, quand faut-il utiliser un `id` plutôt qu'une `class` ? Les classes sont préférables quand tu veux appliquer un ensemble de styles à plusieurs éléments. Si tu veux cibler un élément précis, il vaut mieux utiliser `id`, car ces valeurs doivent être uniques.

## --text--

Quand dois-tu utiliser un `id` plutôt qu'une `class` ?

## --answers--

Utilise une `class` quand tu veux un identifiant unique qui ne doit s'appliquer qu'à un seul élément.

### --feedback--

---

---
