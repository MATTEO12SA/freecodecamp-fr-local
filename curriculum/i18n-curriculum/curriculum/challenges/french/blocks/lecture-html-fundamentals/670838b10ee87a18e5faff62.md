# --interactive--

L'élément `script` sert à intégrer du code exécutable. La plupart des développeurs l'utilisent pour exécuter du code JavaScript. JavaScript sert à ajouter de l'interactivité à tes pages web. Des exemples courants d'utilisation de JavaScript incluent les jeux interactifs, les carrousels d'images et les formulaires dynamiques qui valident les saisies de l'utilisateur en temps réel.

Voici un exemple d'utilisation de l'élément `script` dans un document HTML. Supprime les `//` devant `alert("Welcome to freeCodeCamp");` et tu devrais voir apparaître une alerte. Pour voir les aperçus, tu dois activer l'éditeur interactif.

:::interactive_editor

```html
<body>
  <script>
    // alert("Welcome to freeCodeCamp");
  </script>
</body>
```

:::

Même si tu peux techniquement écrire tout ton code JavaScript à l'intérieur des balises `script`, la bonne pratique est plutôt de lier un fichier JavaScript externe. Voici un exemple d'utilisation de l'élément `script` pour lier un fichier JavaScript externe :

```html
<script src="path-to-javascript-file.js"></script>
```

L'attribut `src` est utilisé ici pour indiquer l'emplacement de ce fichier JavaScript externe. `src` signifie "source". Il n'est pas recommandé de placer tout ton JavaScript dans le document HTML à cause de la séparation des responsabilités. La séparation des responsabilités est un principe de conception où tu sépares tes programmes en sections distinctes, chacune chargée d'une responsabilité différente. Dans ce cas, on veut séparer le code JavaScript du code HTML.
