# --interactive--

## Design web responsive

- **Définition** : Le principe central du design responsive est l'adaptabilité : la capacité d'un site web à ajuster sa mise en page et son contenu selon la taille de l'écran et les capacités de l'appareil utilisé pour le consulter.
- **Grilles fluides** : Elles utilisent des unités relatives comme les pourcentages au lieu d'unités fixes comme les pixels, ce qui permet au contenu de se redimensionner et de se redistribuer selon la taille de l'écran.
- **Images flexibles** : Elles sont configurées pour se redimensionner à l'intérieur de leurs éléments conteneurs, ce qui évite qu'elles dépassent de leurs conteneurs sur les petits écrans.

## Media queries

- **Définition** : Elles permettent aux développeurs d'appliquer différents styles selon les caractéristiques de l'appareil, principalement la largeur du viewport.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="mq-example">The background of this paragraph will change when the viewport is at least 768px wide.</p>
```

```css
.mq-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) {
  /* Styles for screens at least 768px wide */
  .mq-example {
    background-color: lightblue;
  }
}
```

:::

- **Type de média `all`** : Il convient à tous les appareils. C'est la valeur par défaut si aucun type de média n'est précisé.
- **Types de média `print`** : Ils sont destinés aux documents paginés et aux documents affichés à l'écran en mode aperçu avant impression.
- **Types de média `screen`** : Ils sont principalement destinés aux écrans.
- **`aspect-ratio`** : Cela décrit le ratio entre la largeur et la hauteur du viewport.

```css
@media screen and (aspect-ratio: 16/9) {
  /* Styles for screens with a 16:9 aspect ratio */
}
```

- **`orientation`** : Cela sert à indiquer si l'appareil est en orientation paysage ou portrait.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="orientation-example">This paragraph's background will change in landscape orientation.</p>
```

```css
.orientation-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (orientation: landscape) {
  /* Styles for landscape orientation */
  .orientation-example {
    background-color: lightgreen;
  }
}
```

:::

- **`resolution`** : Cela sert à décrire la résolution du périphérique de sortie en points par pouce (dpi) ou en points par centimètre (dpcm).

```css
@media screen and (min-resolution: 300dpi) {
  /* Styles for high-resolution screens */
}
```

- **`hover`** : Cela sert à tester si le mécanisme d'entrée principal peut survoler des éléments.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button>Hover over me</button>
```

```css
button {
  padding: 10px 20px;
  font-size: 1rem;
  background-color: lightgray;
  border: 1px solid #999;
  cursor: pointer;
}

@media (hover: hover) {
  /* Styles for devices that support hover */
  button:hover {
    background-color: yellow;
  }
}
```

:::

- **`prefers-color-scheme`** : Cela sert à détecter si l'utilisateur a demandé un thème de couleur clair ou sombre.
- **Media queries et opérateurs logiques** : L'opérateur `and` sert à combiner plusieurs fonctionnalités media, tandis que `not` et `only` peuvent servir à inverser ou isoler des media queries.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="logical-example">This paragraph's background will change when the screen is at least 768px wide AND in landscape orientation.</p>
```

```css
.logical-example {
  padding: 10px;
  border: 1px solid #ccc;
  background-color: lightyellow;
}

@media screen and (min-width: 768px) and (orientation: landscape) {
  /* Styles for landscape screens at least 768px wide */
  .logical-example {
    background-color: lightcoral;
  }
}
```

:::

## Breakpoints media courants

- **Définition** : Les breakpoints media sont des points précis dans le design d'un site web où la mise en page et le contenu s'ajustent pour s'adapter à différentes tailles d'écran. Il existe des breakpoints généraux que tu peux utiliser pour cibler les téléphones, les tablettes et les écrans de bureau. Mais il n'est pas judicieux d'essayer de cibler chaque taille d'écran possible pour chaque appareil.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="breakpoint-example">This text will change size when the viewport width is at least 768px.</p>
```

```css
.breakpoint-example {
  font-size: 1rem;
  padding: 10px;
  border: 1px solid #ccc;
}

/* Styles for screens wider than 768px */
@media screen and (min-width: 768px) {
  .breakpoint-example {
    font-size: 1.125rem;
    background-color: lightblue;
  }
}
```

:::

- **Petits appareils (smartphones)** : jusqu'à 640px
- **Appareils moyens (tablettes)** : de 641px à 1024px
- **Grands appareils (ordinateurs de bureau)** : 1025px et plus

## Approche mobile-first

- **Définition** : L'approche mobile-first est une philosophie de design et une stratégie de développement en design web responsive qui donne la priorité à la création de sites web pour les appareils mobiles avant la conception pour les écrans plus grands.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div class="container">
  <p>The width of this container changes based on the viewport size.</p>
</div>
```

```css
/* Base styles for mobile */
.container {
  width: 100%;
  padding: 10px;
  background-color: lightblue;
}

/* Styles for larger screens */
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
    padding: 20px;
  }
}

@media screen and (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

:::
